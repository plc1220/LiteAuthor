围绕角色当前处境的失衡点设计矛盾，再从常用矛盾模板里选型。**有效矛盾不是有人吵架，而是有人因为失衡必须采取行动。**

## 适用情况
- 明明安排了冲突，情节还是不动
- 误会、拉扯、对立写得生硬
- 不知道该用什么矛盾把一段戏撑起来
- 想扩篇幅，但现有冲突只够用一次
- 想给 CP 或主线加波折，却总像强行拧巴

## 输入
- 当前剧情阶段
- 参与人物
- 角色原本的稳定状态
- 想让这一段发生的变化
- 已有外部事件或隐藏心事
- 希望达到的效果：加压 / 拉扯 / 误会 / 期待 / 长线铺垫 等

## 步骤
### 1. 先写当前的平衡
先别急着造冲突，先写：
- 角色原本以为事情会怎样发展
- 此刻最想维持什么
- 当前的安稳处境是什么

没有平衡，就没有失衡。

### 2. 找出失衡点
句式参考：
`角色原本想____，但是____，于是不得不____。`

真正有效的失衡点，必须满足至少一项：
- 角色直接看见问题
- 角色被问题伤到
- 角色直觉到危险或不对劲
- 角色虽然没搞明白，但已经开始难受或焦躁

### 3. 写 A / B 各自最想恢复什么
分别写清：
- A 想恢复什么平衡
- B 想恢复什么平衡
- 这两个目标是否互相冲突

很多有劲的场景，不是谁单纯想赢，而是双方都想把局面拉回对自己有利的位置。

### 4. 判定外显冲突和暗冲突
- **外显冲突**：读者一眼能看见的问题，如争吵、打脸、离开、竞争
- **暗冲突**：表面平静，但目标、情绪、立场已经错位

暗冲突不是没有冲突，而是冲突暂时藏在人物反应和选择里。

### 5. 先定这场戏的结果
一场矛盾只落三种结果之一：
- **暂时解决**：平衡被拉回去，但代价留下了
- **进一步恶化**：新问题更大，局势更乱
- **暂时压下**：矛盾没解，只是被外力延后

没有结果变化的冲突，容易像原地打转。

### 6. 从常用模板里选型，可单用也可叠加
#### A. 延时矛盾
**定义**：本来当场解释清或见到人就能解决的问题，因为外部事故、时机错过、任务打断，被拖到后面，反而长出更大矛盾。

**适合**：
- 引出更大波折
- 制造“差一点就说开”的期待
- 把短矛盾拉成长矛盾

**检查**：
- 延迟理由必须具体合理，不是作者硬拖
- 延迟之后必须长出新 stakes，而不只是重复错过

#### B. 不解释矛盾
**定义**：有解释机会，却没有解释，导致误会继续发酵。

**只在这些理由下使用**：
- 角色在生气或赌气，但只适合短时使用
- 解释出来会受更大伤害
- 角色解释了，但没解释到点子上

**高危**：
- 一句正常沟通就能解决
- 角色既无顾虑，也无代价，只是作者不让说

#### C. 倒霉式矛盾
**定义**：一方让另一方倒霉，或在倒霉后出手帮忙；也可以两者叠加。

**适合**：
- 快速制造 CP 火花
- 让两人被迫绑定
- 用麻烦换互动，用补救换关系变化

**检查**：
- 倒霉必须改变利益、处境或关系
- 不能只是事故列表，写完什么都没变

#### D. 脑补式矛盾
**定义**：角色拿到半截信息，自行补完成错误结论，于是躲避、吃醋、误会、先入为主。

**适合**：
- 感情拉扯
- 误会升级
- 角色旧伤被触发

**检查**：
- 必须有足够证据让角色会这么想
- 最好和角色的旧伤、偏见、恐惧有关
- 不能无限拖，最终要让误会改局势或改关系

#### E. 三观不合矛盾
**定义**：双方价值观、信任对象、底线、立场不同，哪怕彼此在意，也很难一句话说平。

**适合**：
- 长线拉扯
- 事业线 / 阵营线 / 感情线并行
- 需要可持续冲突的主线

**检查**：
- 双方立场都要站得住
- 不是一方单纯犯蠢，另一方负责正确

#### F. 预言式矛盾
**定义**：提前告诉读者，角色眼下的选择会导致未来某种失去、后悔或翻车，用来吊住长线期待。

**适合**：
- 很长的铺垫段
- 大高潮前的远期悬念
- 让读者知道“这一步很重要”

**写法**：
- 可直接写“如果早知道……”
- 也可更隐一点，用旁人警告、对白反复否认、未来回声来做

**检查**：
- 预言必须后续回收
- 不要剧透到把具体过程全说死

### 7. 给模板补证据和代价
无论选哪种模板，都补齐四件事：
- 角色为什么会这样反应
- 如果不处理，会失去什么
- 误会 / 拖延 / 对立会怎样升级
- 这次冲突如何改下一步选择

### 8. 落到场景里
按这条链写：
`失衡点 → 情绪反应 → 动作 → 场面变化 → 余波`

如果只有“设定上有矛盾”，没有情绪和动作，场景还是起不来。

## 输出格式
1. 当前平衡
2. 失衡点
3. A / B 各自想恢复的平衡
4. 外显冲突 / 暗冲突
5. 选用的矛盾模板
6. 模板成立理由与代价
7. 场景推进链
8. 结果：解决 / 恶化 / 压下
9. 后续还能怎么加码
10. 降智或硬拖风险点

## 快速示例
### 输入
- 男主街头看见失散多年的女主，刚追过去，就有人冲出马路要被撞

### 输出
1. 当前平衡：男主以为终于能和女主重新见面
2. 失衡点：救人和追人无法同时完成
3. 模板：延时矛盾
4. 成立理由：男主不可能见死不救；这一停步让“相认”变成“再次错过”
5. 场景推进链：看见女主 → 冲过去 → 被事故打断 → 先救人 → 回头找不到人
6. 结果：矛盾没解，反而升级成更大的寻找与误会

## 注意事项
- **不要**让“就不解释”变成一句话能解决的硬拖
- **不要**让脑补只靠角色犯蠢
- **不要**让每个矛盾都只是换人名重复
- **不要**只有热闹，没有后果
- 好矛盾的标准不是吵得凶，而是**能逼人行动，还能继续往下长剧情**

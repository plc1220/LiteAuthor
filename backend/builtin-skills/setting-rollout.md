按“风声 → 验证 → 正式展开”把新设定放进正文。**设定不是说明书，也不是突然空降。让读者先隐约感觉到，再看见证据，最后真正用上，读起来才舒适。**

## 适用情况
- 写到中途要加新设定
- 隐藏副本、组织、规则、地图要出场
- 角色要新增能力、身份或特殊道具
- 设定一写就像硬塞，读者容易出戏

## 输入
- 要导入的新设定一句话
- 设定类型：世界规则 / 地点 / 身份 / 能力 / 道具 / 组织
- 它第一次正式生效的章节或场面
- 它要解决或放大的当前冲突
- 现有正文里可挂钩的元素

## 核心原则
1. **先给风声，再给说明。**
2. **世界设定提前 3—5 章放料最稳。** 先让读者闻到味，再慢慢摊开。
3. **人物新能力必须补诱因和过渡。** 没原因的升级最容易出戏。
4. **临时新增设定也要像早有准备。** 让读者感觉“原来如此”，不要感觉“现在才有”。
5. **每次导入都要服务当前剧情。** 不能停下来单独上课。

## 步骤
### 1. 先写清这项设定到底改变什么
填空：
`这个新设定进场后，会让____发生变化。`

常见变化：
- 打开新地图
- 抬高难度
- 解释旧谜团
- 给角色新手段
- 逼关系或冲突升级

### 2. 判断它属于哪类设定
#### A. 世界 / 地点 / 组织类
如隐藏副本、门派、势力、特殊职业、禁地规则。

#### B. 人物类
如新能力、新身份、新血脉、新武器、新契约。

#### C. 规则类
如隐藏代价、世界限制、特殊交易规则。

不同类型，导入节奏不同。

### 3. 世界类设定用“三段导入”
#### 第一段：风声
正式展开前，先用零碎信息放风：
- 传闻
- 异常现象
- 别人的只言片语
- 主角偶然碰到的边角痕迹
- 一个说不透的旧线索

目标：让读者先知道“这里有东西”。

#### 第二段：验证
让主角或重要角色碰到更具体的证据：
- 看到入口、见到相关人物、拿到碎片信息
- 某件小事证明传闻并非空穴来风
- 读者开始能猜到大概方向

目标：把“听说过”推进到“真的存在”。

#### 第三段：正式展开
在当前冲突最需要它的时候让设定真正落地：
- 进副本
- 见组织
- 启动规则
- 公开设定

目标：让设定一登场就改局势，而不是只负责解释。

### 4. 人物类设定用“诱因链”
新增能力或身份时，至少补齐这 5 件事：
1. **需求**：为什么现在非得要它
2. **诱因**：什么机缘、打击、任务或遭遇把它引出来
3. **代价 / 证明**：拿到它付了什么，或凭什么能拿到
4. **小试用**：先给一个小范围兑现
5. **正式常态化**：什么时候真正进入主打法

可直接套用：
`先遇到问题 → 被迫接触新设定 → 付出代价或完成考验 → 小范围证明 → 在更大场面正式使用`

### 5. 临时加设定时，先找“舒适锚点”
如果这项设定不是开篇就有，先补 1—3 个锚点：
- 前文已有相关名字或现象
- 已有角色和它沾边
- 已有冲突正好需要这项设定解释或放大
- 主角之前就对这类东西有过接触或认知

**锚点不用太重，但必须让它挂得住。**

### 6. 每次放料都留一个“未说完的问题”
每一轮只回答一部分，再留下一个更值钱的问题：
- 这地方到底是什么
- 为什么现在才开
- 代价是什么
- 谁早就知道
- 主角能不能用得住

这样设定才会和钩子一起往前滚。

### 7. 正式展开时，别一次性讲完
稳妥顺序：
- 先让角色用到 / 碰到
- 再在行动中解释必要规则
- 最后用后果补齐深层信息

**先体验，后说明。**

## 常用模板
### 模板一：隐藏副本
- 前 3—5 章：传闻、异常资源、失踪消息、有人提起旧地名
- 前 1—2 章：主角拿到入口线索或亲眼见到异常
- 正式章节：副本开启，规则落地，主角被迫进入或主动争夺

### 模板二：新增能力
- 先安排打击、任务或短板
- 再安排机缘：契约灵兽、获得传承、触发血脉、接触道具
- 给出代价或门槛
- 先小用一次，证明不是白送
- 再在关键冲突里正式兑现

## 输出格式
1. 新设定一句话
2. 设定类型
3. 它要改变的当前局势
4. 风声 / 验证 / 正式展开节点
5. 需要补的诱因或过渡
6. 可用的舒适锚点
7. 每轮放料留下的问题
8. 正式展开的场面设计
9. 出戏风险点

## 注意事项
- **不要**设定一出场就先讲半章说明书
- **不要**角色突然多出能力，却没有诱因、代价或过渡
- **不要**为了补漏洞临时塞设定，又和前文毫无挂钩
- **不要**提前把所有规则摊平，后面就没味了
- 设定导入真正要做到的，不是“读者懂了”，而是**读者先被勾住，再在用的时候自然懂了**。

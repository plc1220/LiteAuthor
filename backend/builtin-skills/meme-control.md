把梗当调味，不要当门槛。**正文里的梗至少要满足两件事：懂梗的人会笑，不懂梗的人也不至于被挡住。**

## 适用情况
- 一章里梗太密，读者像在刷弹幕
- 跨圈梗太多，不百度就看不懂
- 甜文、虐文、悬疑文被梗打断气氛
- 作者自己觉得很好笑，读者却只觉得出戏

## 输入
- 当前段落
- 想放的梗或引用
- 这段原本的气氛目标
- 目标读者大致画像

## 步骤
### 1. 先写这个梗到底是来干什么的
只能从下面选 1—2 项：
- 让角色说话更有个人味
- 让场面更轻松
- 提供一个短平快笑点
- 强化人物的上网感、年龄感、圈层感

如果答不出来，通常就是可以删的自嗨梗。

### 2. 过三道检查
#### A. 读者门槛检查
问：目标读者大概率懂吗？
- 大众梗：可用
- 圈内梗 / 小众梗：慎用
- 需要百度才懂：尽量别放正文核心位置

#### B. 人设语境检查
问：这个角色、这个世界，真的会这么说吗？
- 现代年轻角色：可适量上网感
- 古言 / 仙侠 / 严肃悬疑：更要警惕错位
- 不是所有角色都适合高频吐槽和玩梗

#### C. 气氛位置检查
问：这段原本是在发糖、推悬念、写泪点，还是写危险？
- 情绪越重，越少玩梗
- 重大反转、表白、哭点、凶险场面，优先保气氛

### 3. 能用场面表达，就别硬塞引用
如果同样效果可以通过下面方式达成，优先不用梗：
- 人物反应
- 场面错位
- 口是心非
- 观察式幽默
- 情境反差

**角色自己的好笑，比借别人现成的梗更稳。**

### 4. 如果要保留梗，让不懂的人也能看懂基本意思
做法：
- 前后文给出足够反应
- 让笑点落在场面和关系上，而不是只靠梗本体
- 不把关键剧情推进压在梗的理解门槛上

### 5. 控制密度
粗略规则：
- 一场戏有一个主梗就够了
- 同一段里不要连甩多个圈层梗
- 连续高强度玩梗后，给正文留喘气空间

### 6. 跨圈梗慎选
高风险组合：
- 非网游文硬玩网游圈梗
- 甜文硬塞政治梗
- 古言硬塞现代直播梗
- 名著梗、学科梗、黑话梗堆成识别门槛

**梗和题材、角色、读者圈层越远，翻车概率越高。**

### 7. 写完后做“删掉试验”
删掉这个梗再看：
- 这段是不是更顺了？
- 人物是不是没受影响？
- 气氛是不是反而更稳？

如果答案大多是“是”，那就删。

## 输出格式
1. 这段原本的气氛目标
2. 梗的功能
3. 门槛 / 人设 / 气氛三项检查
4. 保留 / 改写 / 删除结论
5. 若保留，怎样让不懂梗的人也能读懂
6. 若替换，改成什么场面幽默

## 快速示例
### 输入
甜文日常里突然插入大量直播圈黑话。

### 输出
- 气氛目标：轻松发糖
- 问题：目标读者未必懂黑话，且和当前情绪无关
- 结论：删掉黑话本体，保留“角色嘴贫”和“对方听不懂的错位感”，改成更直观的对话和动作笑点

## 注意事项
- **不要**把读者看不懂，当成“读者不上网”
- **不要**用梗替代真正的幽默和人物塑造
- **不要**在重要情绪场面里疯狂抛梗
- 真正耐看的梗，不是作者显得懂很多，而是角色和场面因此更鲜活

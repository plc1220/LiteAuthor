把剧情线为主的文的大纲列到能直接防卡文、防 bug、控节奏的程度。**越烧脑、越复杂、越多线，纲就要越细；节奏掌控力越弱的人，越不要裸奔。** 感情流可以只列节点，剧情流最好细到阶段、场次、线索和因果。

## 适用情况
- 写剧情流、悬疑、权谋、升级、群像等主打剧情推进的文
- 设定很多，写着写着开始卡文
- 中后期容易出 bug、忘伏笔、节奏失衡
- 总觉得有很多点子，但连不成一条顺线
- 情节复杂，担心提前或延后关键剧情

## 输入
- 一句话题材
- 一句话主线
- 预计篇幅
- 已有角色、势力、设定
- 目前想到的关键桥段和反转

## 核心原则
1. **剧情越复杂，纲越要细。**
2. 细纲不是束缚发挥，而是先把因果和节奏控住。
3. 至少要细到：这一段谁想做什么、为什么做、被什么拦、结果怎样、下一步怎么接。
4. 伏笔、误导、揭晓都要写进纲里，不要只存在脑子里。
5. 如果这篇明明是剧情主，就不要拿感情流那种纯节点大纲硬顶。

## 步骤
### 1. 先钉死主线和终点
分别写两句：
- `主角当前必须解决的是____`
- `这本书阶段性 / 最终要到达的是____`

这一步不清，后面的细纲只会越来越散。

### 2. 把全文拆成阶段
按 3—6 段去拆，每段都写清：
- 这一阶段的目标
- 这一阶段最大的阻力
- 这一阶段结束时，局势会怎么变

例如：
- 站稳脚跟
- 拿到资格
- 进入更高层对抗
- 接近核心真相
- 最终正面对撞

### 3. 每个阶段再拆成场次 / 章节任务
每一场至少写清 5 项：
1. 触发事件
2. 主角目标
3. 障碍 / 对手动作
4. 转折 / 判断
5. 结果与下一场钩子

不会列时，先用模板：
`因为____，主角必须____，但被____拦住，于是做出____，结果____。`

### 4. 单独列“线索表”
把关键线索分开整理，不要散在文档里。

建议至少列：
- 线索是什么
- 第一次出现在哪
- 当时谁看见 / 知道
- 会被怎么误解
- 真相何时揭晓
- 最终回收在哪

### 5. 再列“信息差表”
分别记清：
- 主角知道什么
- 对手知道什么
- 读者知道什么
- 哪一步会因为信息差产生误判 / 反转

**剧情流最怕作者自己知道，但纲里没写，写到一半就忘。**

### 6. 检查因果链
逐场检查：
- 这一场为什么会发生
- 它是不是由上一场推出来的
- 删掉这一场，后面还能不能成立
- 这一场有没有真正改变局势

如果删掉某段后主线几乎不受影响，这段多半只是热闹。

### 7. 检查节奏升级
每过一段至少升级一项：
- 风险更高
- 对手更强
- 牵连更广
- 真相更深
- 代价更重

**不要一直换事件，不升级难度。**

### 8. 给关键节点标“不能动”
把会影响全局的节点单独标记：
- 大反转
- 身份揭露
- 真相揭晓
- 决裂 / 联手
- 阶段 boss 战

这些点一改，全盘都要重查；所以要提前在纲里钉死。

### 9. 写完做 bug 排查
逐条问：
- 伏笔有没有回收
- 角色行为是否符合已知信息
- 设定有没有前后打架
- 反派手段有没有越级或降智
- 主角推进是不是一直靠巧合
- 关键剧情有没有提前或延后到失衡

## 输出格式
1. 一句话主线
2. 终点 / 结局条件
3. 阶段表
4. 每阶段的场次任务
5. 线索表
6. 信息差表
7. 节奏升级点
8. 不能动的关键节点
9. 当前最容易出 bug 的地方

## 快速示例
### 输入
- 主角要查清一场旧案
- 中间牵扯家族、同伴和更大阴谋

### 输出
- 阶段 1：拿到旧案入口线索
- 阶段 2：调查时被人误导，怀疑错对象
- 阶段 3：同伴背刺，真相方向彻底改变
- 线索表：旧照片、证词漏洞、时间线异常分别在哪出现、何时回收
- 信息差表：读者比主角先知道一条假线索有问题，用来制造悬念

## 注意事项
- **不要**只记名场面，不记因果连接
- **不要**把伏笔全放脑子里，写到后面容易忘
- **不要**剧情主文只列一个模糊大纲就开写
- **不要**为了自由发挥，把关键节点放成漂浮状态
- 细纲真正的作用，不是限制创作，而是**让复杂剧情也能稳稳落地。**

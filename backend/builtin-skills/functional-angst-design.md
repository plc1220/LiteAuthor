写虐点时，先定功能，再定痛法。**虐点的任务不是让读者难受，而是让读者更心疼角色、更相信关系、更想往下看。** 没有功能的虐，只会像故意给读者添堵。

## 适用情况
- 想写虐点，但总像为虐而虐
- 一虐就掉追更，读者嫌堵
- 甜文、正剧、BE 的力度拿不准
- 冲突有了，但心疼感不够
- 不知道该用外在虐点还是内在虐点

## 输入
- 文的定位与标签：甜文 / 正剧 / 虐恋 / 剧情向等
- 结局承诺：HE / 开放式 / BE
- 想让读者得到的感觉：心疼 / 后怕 / 怜爱 / 愤怒 / 期待回收
- 当前人物关系阶段
- 这次准备虐谁、伤到哪
- 这场虐点后想换来什么

## 步骤
### 1. 先写这次虐点的功能
从下面至少选 1 项，最好不超过 2 项：
- 加深对某个角色的怜爱
- 暴露角色真正的软肋
- 逼角色做选择 / 站队
- 给和好、反转、告白蓄力
- 放大人物成长代价
- 把剧情从平推进到高压

**答不上功能，就先别虐。**

### 2. 先对齐读者预期
- **甜文**：可以有虐点，但要短、准、快回收
- **正剧 / 剧情向**：可以更重，但必须推动人物和主线
- **BE**：要尽早在标签、文案、前期气质里给信号，别临门一脚背刺

如果标签承诺和虐点力度相反，最容易掉读者。

### 3. 选虐点类型：外在 or 内在
#### 外在虐点
生病、受伤、家庭阻力、任务失败、舆论打击、身份压迫、现实困境。
- 好处：容易写清楚，不容易显得角色作
- 适合：甜文、剧情文、中前期拉高心疼值

#### 内在虐点
误判、冷战、立场冲突、主动推开、旧伤复发、信任崩塌。
- 好处：更能打感情线
- 风险：动机不够就会显得矫情
- 适合：中后期、关系已经有基础时使用

### 4. 先打在最疼的地方
不要只写受伤本身，要写为什么这一下对他最疼：
- 自卑的人被公开否定
- 缺爱的人被误以为不重要
- 责任感重的人被迫看同伴受伤
- 很能扛的人第一次扛不住

**疼点必须贴人设，不要换谁都一样。**

### 5. 内在虐点必须有充足动机
重点检查：
- 这件事是不是一句话就能说清
- 角色为什么偏偏现在说不清 / 不能说 / 不敢说
- 不说的代价是什么
- 这份冲突能不能回收成更深的理解、站队或反转

如果只是作者不让他们说开，读者会很烦。

### 6. 虐完必须换来情绪补偿
至少回收 1 项：
- 更心疼角色
- 更确认两人的重要性
- 有人公开站队 / 护短
- 真相揭开
- 局势升级，读者更想看后续
- 后续反击更爽

**只痛不补，读者只会想跑。**

### 7. 给虐点安排后手
常用回收方式：
- 安抚：照顾、陪伴、道歉、坦白
- 反转：事实揭开、误会洗清、主动权翻回
- 打脸：让加害方付代价
- 成长：角色因此学会面对某个问题
- 绑定加深：以后再也没法把对方当普通人看

### 8. 控制强度
可按 1—5 级粗分：
1. 小委屈、小错位
2. 明显心疼，但不伤筋动骨
3. 真正破防，关系和主线都被改写
4. 重创，必须用长回收
5. 终局级失去 / 死亡 / BE 级代价

商业长篇里，重度虐点越重，前期预警和后续回收越要足。

### 9. 最后排雷
- 是不是只是为了虐而虐
- 是否和题材承诺冲突
- 是否只有痛，没有后果
- 是否伤害太重，回收却太轻
- 是否让角色显得无脑、矫情、故意误会

## 输出格式
1. 文的定位与结局承诺
2. 本次虐点功能
3. 虐点类型：外在 / 内在 / 混合
4. 被打中的角色软肋
5. 触发事件
6. 读者情绪目标
7. 回收方式
8. 强度等级
9. 风险点

## 快速示例
### 输入
- 正剧向，HE
- 想让读者更心疼女主，并逼男主站队

### 输出
- 功能：怜爱 + 逼选择
- 类型：外在虐点为主
- 软肋：女主最怕被当成可以随便牺牲的人
- 触发事件：任务失败后，女主替全队背锅，被公开质疑
- 读者情绪目标：心疼、愤怒、等男主表态
- 回收方式：男主在更大的场合公开站女主，顺带揭出真正责任人
- 强度：3 级
- 结果：不仅虐到了人，还改了关系和局势

## 注意事项
- **禁止**在甜文标签下长期高压狂虐
- **禁止**把一句话能解决的误会拖成大虐点
- **禁止**只让角色倒霉，不让加害和局势付代价
- **禁止**结局突然 BE，却前面完全没给信号
- 好的虐点不是好惨，而是**惨得有用，痛完更值**。

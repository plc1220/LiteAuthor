先把危险写成会一步步逼近的期限，不要写成模糊的“总有一天会出事”。**倒计时危机的核心，不是一直喊危险，而是让读者看见时间、条件、近失和代价不断收紧。**

## 适用情况
- 想让读者持续担心某人会死、失去、暴露、被带走
- 中段需要稳定追读感，但不能每章都靠大打大杀
- 感情线已经升温，需要一个长期悬在头上的大问题
- 有预言、规则、名单、宿命、期限类设定

## 输入
- 最终危机是什么
- 触发条件或期限
- 谁知道真相，谁只知道一部分
- 角色当前最珍惜的东西
- 预计要拉多久

## 核心原则
1. **期限要具体**：某天、某年龄、某条件达成、某物被取走。
2. **每次提醒都要更近一步**，不能原地复读。
3. **近失要升级**：预告 < 小事故 < 明确名单 / 规则 < 大事故 < 必须做选择。
4. **危机必须改关系和行动**，不能只改气氛。
5. **能救一次，不代表能一直救。** 每次救场都要证明问题更难拖了。

## 常见信息差
- 读者知道，主角不知道：最适合追读悬念
- 主角知道一半，另一半被瞒着：最适合情感拉扯
- 双方都知道有危险，但不知道具体何时爆：最适合共同应对

## 步骤
### 1. 先写死最终炸点
用一句话写：
`如果这个问题不解决，最后会发生____。`

例如：
- 某人会在限定时间内死亡
- 真相一旦触发，关系就会彻底改写
- 规则继续拖延，会让更重要的人付代价

### 2. 给危机一个明确时钟
可用：
- 倒数天数 / 生日 / 仪式日
- 某个动作一旦完成
- 某个印记、症状逐渐加重
- 名单、预言、生死簿已经写下结果
- 上位规则明确下最后通牒

### 3. 先抛第一层“模糊但可信”的预告
第一层只做两件事：
- 让读者知道这件事迟早会来
- 让角色开始在意，但还没完全相信

### 4. 第二层必须具体化
把危险从“好像会出事”推进到：
- 具体是谁
- 具体会失去什么
- 具体在什么条件下发生

**到了第二层，读者要能清楚说出：他们到底在怕什么。**

### 5. 第三层安排第一次近失
设计一次差点发生、但暂时被拦住的事故：
- 差点车祸、坠落、暴露、失控
- 差点触发规则
- 差点完成那个会导致终局的动作

作用：
- 证明危险不是嘴上说说
- 让角色意识到问题已经进现实

### 6. 第四层升级代价
不要只写“又差点出事”，要让 stakes 变重：
- 事故更频繁
- 保护者要付更大代价
- 拖延本身开始伤及别人
- 真相再不说就会更残酷
- 角色必须在“救对方”和“保自己”之间选

### 7. 把倒计时嵌进日常或甜戏
每隔 2—3 章至少点亮一次：
- 一个异常症状
- 一份新名单、新预告
- 一句没说出口的真相
- 一次因为保护而改变计划
- 一个看似温暖、实则更让人害怕失去的场面

**最稳的写法不是全程紧绷，而是让危险在正常生活里反复露头。**

### 8. 最后把危机变成“必须选”
终局前至少逼出一个硬选择：
- 说不说真相
- 救谁
- 完成命运还是违抗规则
- 留下还是离开

**倒计时的价值，不是最后真的死不死，而是它逼角色做了什么选择。**

## 输出格式
1. 最终危机
2. 期限 / 触发条件
3. 信息差分配
4. 预告—具体化—近失—升级—终局选择链
5. 每一层带来的关系 / 行动变化
6. 中段提醒点
7. 终局前最难的一选

## 快速示例
### 输入
- 男女主相爱后，女主一旦完成某个命定动作，男主就会消失

### 输出
- 时钟：命定动作 + 不断加重的异常事故
- 第一层预告：旁人提到“完成那一步，他就会不在了”
- 第二层具体化：明确消失的是谁、由谁触发
- 第一次近失：女主差点真的完成动作
- 代价升级：男主为了拖延不断救人改命，问题反而越来越大
- 终局选择：到底告诉她真相，还是继续瞒下去

## 注意事项
- **禁止**只反复说危险，却不让危险更具体
- **禁止**近失每次都一个模子
- **禁止**拖很久不提醒，读者会忘
- **禁止**明明有倒计时，角色和剧情却像没事人
- **倒计时危机最值钱的地方，是“越来越近”四个字。**

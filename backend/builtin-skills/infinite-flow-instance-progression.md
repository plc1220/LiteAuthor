把每个副本都写成主线的台阶，而不是一组可替换的小世界。**无限流能不能一直追下去，关键不在副本多，而在副本之间有没有明显递进和承接。**

## 适用情况
- 第二个或第三个副本开始掉追读
- 副本写完像 reset，主角又回到原地
- 读者觉得“这个副本可有可无”
- 每个副本都很热闹，但连起来没有主线感
- 不知道副本之间怎么下钩子

## 输入
- 全文主线目标
- 已有或计划中的 3—5 个副本
- 主角当前抓人点
- 每个副本准备给出的规则、资源、真相或情感变化

## 核心原则
1. **无限流必须有一条粗主线。**
2. 每个副本至少推进一项：`难度`、`资源`、`真相`、`队伍`、`关系`、`主角心态`
3. 副本结束后，主角和故事都不能回到原地
4. 副本之间必须留承接钩子
5. 如果删掉一个副本主线几乎不变，这个副本就太可有可无

## 步骤
### 1. 先写整本书的粗主线
填空：
`主角一直在朝____靠近。`

可以是：
- 找出真相
- 逃出系统
- 复活某人
- 杀到终局
- 找回身份
- 改变某种规则

没有这条粗线，副本再多也像散装快穿。

### 2. 给每个副本指定“递进任务”
每个副本至少明确一项不可逆变化：
- 难度升级
- 拿到关键道具
- 新规则曝光
- 队伍关系变化
- 主角能力突破
- 情感升温或伤痕加深
- 更接近终局敌人

写成：
`副本 A 结束后，主角比进入前多了____，少了____，离目标近了____。`

### 3. 让副本和副本之间有承接关系
承接不一定只有剧情，也可以是：
- 上一副本留下的伤
- 上一副本拿到的道具
- 上一副本种下的疑点
- 上一副本建立/破裂的人际关系
- 上一副本暴露的新身份或阵营

**读者要感觉到“下一本是上一本逼出来的”，而不是作者重新开了一局。**

### 4. 结尾必须下钩子
副本结尾至少留一类东西：
- 新真相只揭开一半
- 更高一级的规则被看见
- 新副本入口和上副本后果直接相关
- 主角刚拿到关键线索，却发现目标更远
- 队友 / 对手关系发生新变化

如果一个副本打完就彻底平收，读者很容易在这里下车。

### 5. 反查这个副本是否“可删”
逐条问：
- 删掉它，主线会不会少一步关键推进
- 删掉它，主角会不会少一层变化
- 删掉它，下个副本还能不能照常开始

如果三个问题都能轻松回答“能”，这个副本就需要重做。

### 6. 副本掉追读，先查两件事
通常高频原因只有两类：
1. **读者没跟住主角**：主角不够抓人，读者不在感情上接受他/她
2. **下一本钩子不够强**：读者判断“这个副本后面看不看都行”

诊断时先别只怀疑题材，要先看主角和承接。

### 7. 让主角在每个副本都更抓人一点
每个副本至少安排一次：
- 让读者替他/她捏一把汗
- 让读者看到他/她新的狠、惨、强、怪、可怜或可爱
- 让读者确认“我想继续看他/她往前闯”

副本主线之外，主角本身也要持续增值。

### 8. 设计一条清晰的递进轴
优先固定 1—2 条主轴，不要每个副本各玩各的。
常用主轴：
- 难度轴：越来越难
- 资源轴：越来越接近关键道具 / 身份
- 真相轴：一层层扒开世界底
- 情感轴：队友 / CP / 宿敌关系持续变化

## 输出格式
1. 整本粗主线
2. 各副本的递进任务
3. 各副本结束后的不可逆变化
4. 副本之间的承接点
5. 每个副本的结尾钩子
6. 可删风险最高的副本
7. 主角抓人点是否持续增强
8. 调整建议

## 快速示例
### 输入
- 无限流，第二个副本点击明显下滑

### 输出
- 粗主线：主角要找出系统为什么选中自己
- 副本 1：让主角活下来，并拿到第一条关于系统的异常线索
- 副本 2：不是重新闯关，而是让主角利用上一本得到的道具和经验，进一步接近真相
- 结尾钩子：第二本结束时，主角发现幕后人一直能看见自己的选择
- 诊断：如果第二本写完像和主线无关的小世界，读者会判断后面可看可不看

## 注意事项
- **不要**把副本写成可随便替换的支线
- **不要**每个副本都热闹，但主线一动不动
- **不要**副本结束就完全平收，不留下一轮期待
- **不要**只顾规则设计，忘了让读者继续在乎主角
- 无限流最稳的递进，不是本越来越花，而是**主角、真相和终局都在一步步逼近。**

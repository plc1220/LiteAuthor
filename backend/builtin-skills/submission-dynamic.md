写臣服关系时，不要只写一方很强、另一方很弱。**真正带感的是：在某个具体情境里，一方掌握判断权和解决权，另一方逐步把对方神话，从而放大其魅力、控制感和苏感。** 这种关系可以是明面身份差，也可以是隐性能力差。

## 适用情况
- 想写很苏的男主 / 女主 / 反派
- 需要支配与被支配、救赎与被救赎、领导与被领导等高张力关系
- 主仆、君臣、搭档、队长与队员等关系写不出魅力差
- 角色明明不弱，却写不出别人为什么会被他吸住

## 输入
- 关系双方的人设与地位
- 不对等来自哪里：身份 / 能力 / 知识 / 资源 / 情绪稳定 / 道德主动等
- 什么场景最能体现这个差值
- 跟随者最缺什么，最怕什么
- 是否需要后期反转主动权

## 步骤
### 1. 先选不对等轴
从下面至少选 1 条：
- 地位高低
- 实力高低
- 信息差 / 判断力
- 资源控制
- 生死救援
- 情绪掌控
- 救赎关系
- 领导 / 跟随关系

### 2. 把强写成情境里的强
不要写成全方位完人。先回答：
`在什么情境里，所有人都会下意识先看他？`

例如：
- 下墓时他最懂机关
- 战场上他最会指挥
- 谈判时他最能控场
- 崩溃时他最稳

**离开这个情境，他也可以有短板。情境里的王，比全能神更带感。**

### 3. 先补跟随者为什么会交出判断权
至少写清 2 项：
- 他自己解决不了什么
- 他为什么会被对方的方法击中
- 他跟着对方能得到什么：安全、方向、被看见、被拯救、被定义

### 4. 按神话对方的顺序推进
常用顺序：
`看不透 → 跟着走 → 被带出危机 → 默认对方是答案 → 主动替对方找理由 → 愿意为其破例 / 臣服`

可见信号：
- 遇事先看对方反应
- 别人质疑时先替对方辩护
- 明明知道对方也有问题，仍下意识把他放大
- 开始把普通举动解释成特殊意义

### 5. 用场面证明，不要用作者夸
少写：
- 他很强
- 他很有魅力

多写：
- 危机里只有他能做判断
- 别人乱时他能定局
- 跟随者在关键时刻先选他的方案
- 他一出手，局势就改了

**臣服关系最怕旁白硬夸。让别人跟着他，比夸他更有效。**

### 6. 明面臣服和暗面臣服都能用
#### 明面臣服
主仆、君臣、上下级、雇佣、护卫、队长与队员。

#### 暗面臣服
同学、搭档、恋人、竞争者之间也能成立：
- 臣服于能力
- 臣服于判断
- 臣服于稳定感
- 臣服于某种独特魅力或救赎力

### 7. 让神话放大苏感，但别抹掉人味
可以让角色很强，但要留一条真实的人味：
- 只在某人面前失控
- 有一个不合外壳的小弱点
- 在特定关系里也会被反向影响

这样魅力更稳，不像空壳。

### 8. 长篇里最好补一次主动权反转
后期可用：
- 看似被控制的人，才是情感上真正控场的人
- 明面领导者离不开那个一直跟随的人
- 臣服关系从单向，变成互相绑死

**反转不是推翻前面，而是让关系更深。**

### 9. 排雷
逐条检查：
- 跟随者是不是写成纯废物
- 强者是不是全靠作者吹，没有情境兑现
- 关系是不是只剩压迫，没有吸引和回报
- 是否把无理由的羞辱、虐待误当成支配魅力

## 输出格式
1. 关系模式
2. 不对等轴
3. 情境里的强点
4. 跟随者缺口
5. 神话推进链
6. 关键证明场面
7. 是否有主动权反转
8. 风险点

## 快速示例
### 输入
- 探险文搭档
- A 擅长规则判断，B 武力强但不懂机关
- 想把 A 写得很苏

### 输出
- 关系模式：领导 / 跟随
- 不对等轴：信息差 + 判断力
- 情境里的强点：每次进遗迹，A 都最先看出真正出口
- 跟随者缺口：B 能打，但一旦选错路全队都会死
- 神话推进链：B 起初觉得 A 古怪 → 危机里被 A 带着脱身 → 再遇险时先等 A 开口 → 别人怀疑 A 时先站 A
- 反转：后期 A 明面上仍掌控路线，但真正能让 A 失控的，只有 B 的去留

## 注意事项
- **禁止**把臣服写成单纯羞辱和工具化
- **禁止**让强者离开关键情境后只剩作者滤镜
- **禁止**跟随者全程没有主动性、没有价值
- 好看的臣服关系，不是一个人一直压另一个人，而是**一方先成为答案，另一方再把他神话**。

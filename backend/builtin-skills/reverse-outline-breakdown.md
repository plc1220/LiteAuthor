先把样本拆成“读者记住了什么、作者实际上怎么排的、这些排法为什么有效”。**逆推不是抄摘要，而是反推：主线怎么压成一句话、哪些场面为什么能留在脑子里、作者如何用章节功能和叙事顺序让它成立。**

## 适用情况
- 看完榜单文只会说好看，不会学
- 想系统扒大纲、节奏、伏笔、开头切点
- 只会记剧情，抓不住章节功能
- 想研究为什么某些场面特别黏人
- 自己的大纲容易散、没有中段支撑

## 输入
- 1—3 本同类型样本
- 本次研究重点：开篇 / 节奏 / 伏笔 / 感情线 / 打脸 / 大纲
- 研究范围：前 20 章 / 一卷 / 全文
- 自己当前项目的对应难点（可选）

## 核心原则
1. **先看情节和节奏，再看文笔。** 第一次拆时，先别沉迷句子。
2. **先凭记忆抓有效点，再回原文验证。** 能留在脑子里的，往往才是样本真正值钱的东西。
3. **少写剧情摘要，多写章节功能。**
4. **最后用大白话测试故事骨架。** 去掉漂亮文字后还觉得有意思，说明骨架是立得住的。

## 步骤
### 1. 先定研究目标和样本
优先选：
- 同题材
- 同频道
- 同篇幅层级
- 自己确实喜欢、愿意反复看的作品

本次只抓 1—2 个重点，不要一轮里什么都想学。

### 2. 第一遍快速看完，只盯情节和节奏
看时重点记：
- 主线一句话是什么
- 前几章从哪个切点进入
- 哪几处明显让自己停不下来
- 章尾通常怎么把人推到下一章

**这一遍先忽略文笔、修辞、金句。**

### 3. 合上原文，先做“记忆复述”
不翻原文，先完成 3 件事：
1. 用一句话概括故事主线
2. 凭记忆复述核心情节
3. 列出最先想起来的 5—10 个场面

再给每个场面补一句：
- 为什么只记住了这个
- 它给的是爽点、甜点、反转、情绪，还是节奏变化

**记不住的地方，不一定没用；但记得住的地方，一定值得重点拆。**

### 4. 回到原文，对比“记忆版”和“正文版”
逐条检查：
- 自己记住的场面，前面垫了什么
- 原文里有但自己没记住的段落，承担了什么功能
- 自己误记或漏记的部分，说明作者用什么手法把重点抬亮了

这样更容易看清：
- 哪些是真正有效的高光
- 哪些只是过渡
- 哪些铺垫虽不抢眼，但对后文必需

### 5. 拆章纲，不抄剧情
每章至少记 6 项：

```markdown
第 X 章
- 一句话内容：
- 本章功能：立人设 / 推剧情 / 推感情 / 埋伏笔 / 回收 / 卡点
- 关键看点：
- 伏笔或回收：
- 章尾引出：
- 本章为什么放在这里：
```

如果嫌太细，至少记：
- 人物
- 地点
- 发生了什么
- 这一章的作用

**重点不是“发生了什么”，而是“为什么非得这一章发生”。**

### 6. 再把章纲压成大情节
把连续章节压成几个大节点：
- 这一段要解决什么问题
- 这一段结束后局势怎么变
- 它如何顶出下一段

可直接写成：
- 第 1—6 章：开篇困局
- 第 7—15 章：第一次翻面
- 第 16—28 章：关系 / 局势升级

### 7. 拆“故事时间”和“叙事顺序”
分别整理：
- 原始时间线：故事里真实发生的先后
- 叙事时间线：作者实际写出来的先后

重点看：
- 为什么从这个时间点开写
- 哪些过去被后置
- 哪些信息被藏到后面才揭
- 哪些空白被作者直接跳过

### 8. 补拆人物、关系和看点供货
不要只记设定，记这些：
- 角色第一眼卖点是什么
- 后面靠什么不断加深
- 关系是通过什么事件发生变化的
- 爽点 / 甜点 / 虐点前面各垫了什么
- 章尾是空吊悬念，还是局势真的变了

### 9. 提炼可复用规则
每拆完一本，至少压出 3—5 条规则，格式尽量具体：
- 前 3 章先立什么，后补什么
- 一章通常承担几个功能
- 伏笔埋多深、多久回收
- 关系推进常靠哪类事件
- 章尾最常断在什么位置

### 10. 反用到自己的项目
把规则翻成自己的检查表：
- 自己的开头是不是切错时间点
- 自己哪几章只有信息，没有功能
- 自己的伏笔有没有情绪价值
- 自己的中段有没有“大情节节点”支撑

### 11. 最后做一次“大白话测试”
把自己的故事或某段大纲，先用最朴素的话写出来：
- 主线一句话
- 阶段节点 3—5 个
- 每个节点的功能
- 最想让读者记住的 3 个场面

**先别管文笔。**
如果这样的版本都觉得有意思，说明骨架成立；
如果一离开漂亮文字就发空，说明该先补情节和节奏。

## 输出格式
1. 样本与研究重点
2. 一句话主线
3. 记忆复述版核心情节
4. 最强记忆场面清单
5. 章纲表
6. 大情节节点表
7. 原始时间线 vs 叙事时间线
8. 人物 / 关系 / 看点分析
9. 可迁移规则 3—5 条
10. 反用到自己项目的修改建议
11. 大白话测试结果

## 注意事项
- **不要**把逆推做成剧情摘要抄写
- **不要**一上来就学句子，忽略结构
- **不要**只看前几章，不看后文回收就下结论
- **不要**把“我很喜欢”直接等同于“我已经学会了”
- 逆推的目标不是复制样本，而是**找出它稳定有效的排布规律，再变成自己的东西**。

把大纲里的当前阶段先串成一条“糖葫芦”式大情节链，再从大到小拆成章纲和场景；连载时再滚动预排未来 2—3 章短章纲。**细纲不是从一句台词、一场互动往外猜；要先定这一段的大框，再一个一个框攻破。卡文时先分清：是情节闭合没想清，还是场景表述卡住。**

## 适用情况
- 已有粗纲，但一落正文就脱纲
- 知道这一卷要写什么，不知道每章写什么
- 连载中常常写到一半才发现前后不接
- 卡文时分不清自己是没想清剧情，还是不会写场景
- 伏笔、配角、章尾钩子老是忘
- 每天更新前只知道大方向，不知道接下来 2—3 章具体怎么落
- 脑中有纲，但一写到下一章就空白

## 输入
- 当前卷 / 当前阶段目标
- 已有大纲或节点列表
- 必须推进的主线 / 副线
- 已埋伏笔、可用配角、零碎灵感
- 预计章节数（可选）

## 步骤
### 1. 先把这一段串成“糖葫芦”
不要先写第 1 章台词，先列出这阶段 5—10 个大节点：
- 事情怎么开始
- 中间至少哪 2—3 次局势升级
- 最大转折是什么
- 这一阶段最后局势会变成什么

写成一句链条：
`因为____，主角不得不____；中间先后遇到____；最后____。`

### 2. 先定阶段终点，再倒推所需节点
写一句：
`这一阶段结束后，主角会____，局势会____。`
细纲的工作，就是把这句拆成若干章。

### 3. **切忌从小到大，必须从大到小**
顺序固定：
1. 大阶段目标
2. 大情节框
3. 每章主题
4. 每章场景
5. 场景里的具体动作 / 对话

如果连“大框”都没有，就别先抠某章某句台词。

### 4. 给这一段先套一个大框
用最简单的起承转合或目标—波折—结果框一遍：
- 起：这一段为什么启动
- 承：中间靠什么推进、加压
- 转：哪一下真正翻面
- 合：阶段结果和余波是什么

**先有大闭环，再拆小闭环。**

### 5. 再排每章“这章最主要发生什么”
每章先只写一句主题，不先写细节。
例如：
- 第 1 章：主角得知必须接这个任务
- 第 2 章：第一次行动受阻
- 第 3 章：旧伏笔回咬，局势升级
- 第 4 章：阶段性翻盘，抛出下一轮问题

### 6. 每章再拆 3—5 个场景
每个场景至少写清：
- 谁在场
- 这场要解决什么
- 冲突点在哪里
- 场景结束后局面变了什么

尽量用 `因为 / 所以 / 结果` 串，少用“然后”。

### 补充：极简三句话章纲
如果当前不想写完整细纲，就先给每一章写 3 句：
1. 这章主角要做什么
2. 中途被什么打断 / 局势怎么变
3. 结果是什么 / 章尾钩子是什么

模板：
- 这章主角为了____先去____。
- 中途____，导致____。
- 结果____；章尾____。

**脑中有纲不等于纸上有纲。**  
只要经常卡在“下一章第一句怎么写”，就先把这三句写出来。  
如果连这三句都写不出，说明卡的不是文笔，而是章纲还没立住。

### 补充：连载执行版短章纲（2—3 章）
如果是日更或周更，不必每次都把整卷细到头。每次更新前，先把未来 2—3 章各写成 2—4 句：
- 这章主角要做什么
- 会被什么打断
- 结果局势怎么变
- 章尾留什么钩子

这样既能防卡文，又留得出调整空间。就算这一章临场写偏了，后面两章也还有余地修正。

模板：
- 第 N 章：主角为了____，先去____；中途____；结果____；章尾____。
- 第 N+1 章：____
- 第 N+2 章：____

**先保证接下来 2—3 章清楚，再边写边校正，比一口气把十几章写死更稳。**

### 7. 给每章打 4 类标记
- **【伏笔】** 这章新埋了什么
- **【待回收】** 前文什么到这里该回
- **【小高潮】** 这章最值钱的一下
- **【章尾钩子】** 下一章具体还有什么可看

### 8. 控制单章负载
普通网文章里，一章尽量只扛 1—2 个主要情节任务，最好别超过 3 个。  
一章里事情太多，读者会觉得仓促，作者也最容易脱纲。

### 9. 卡文先诊断，不要盲写
#### A. 卡情节
表现：
- 只知道开头和结果，中间不知道怎么走
- 每一章都像“然后发生了……”
- 写着写着偏纲，因为中间闭合没想清

处理：
1. 回到当前大情节框
2. 重新写这段的起 / 承 / 转 / 合
3. 补出最少 2—3 个中间波折
4. 看哪个问题还没阶段性闭合

#### B. 卡场景
表现：
- 明明知道这章要发生什么，就是写不顺
- 一卡就卡在场景、氛围、动作、表述
- 越写越嫌弃，连带情节都看不顺

处理：
1. 先别改大纲，先给当前场景写一句功能
2. 补 4 个要素：地点、人物站位、目标、冲突
3. 先列 3—5 个动作 / 感官关键词，哪怕很粗
4. 把漂亮描写和精修后置，先把场面跑通

**卡情节要回大框，卡场景要回场面。不要混治。**

### 10. 脱纲时回查两件事
- 这章还服务当前大情节框吗
- 这条临时冒出来的新灵感，是不是比原定节点更值钱

如果不是，就留到备选区，不要硬塞。

### 11. 写完细纲后，反看回收链
检查：
- 哪些伏笔 2—5 章内要回
- 哪些配角到这阶段该起作用
- 哪个章尾钩子可以连成同一个大悬念

## 输出格式
1. 当前阶段目标
2. 糖葫芦式大节点链
3. 大情节框（起 / 承 / 转 / 合）
4. 每章主题
5. 每章 3—5 个场景
6. 每章的【伏笔】【待回收】【小高潮】【章尾钩子】
7. 未来 2—3 章滚动短章纲（可选）
8. 卡文时属于卡情节还是卡场景
9. 下一步修正方案

## 快速示例
### 输入
- 阶段目标：男女主从互相试探进入第一次正面冲突

### 输出
- 糖葫芦：发现异常 → 试探失败 → 配角泄密 → 正面交锋 → 关系恶化但绑定更深
- 大框：
  - 起：女主发现男主早就知道某件事
  - 承：两人互相试探，配角不断加压
  - 转：关键秘密被意外捅破
  - 合：第一次正面冲突爆发，下一阶段正式开局
- 滚动短章纲：
  - 第 1 章：女主试探男主，确认他在隐瞒；章尾发现配角偷听
  - 第 2 章：配角误传消息，逼两人提前摊牌；章尾男主反问女主到底知道多少
  - 第 3 章：秘密半揭，第一次正面冲突爆发

## 注意事项
- **不要**把细纲写成漂亮台词合集
- **不要**从一个小场面倒推整段剧情
- **不要**一脱纲就怪自己没天赋，先看是不是大框没立住
- **不要**卡场景时反复重做整本大纲
- **不要**把未来十几章一次性写死，导致中途没有调整空间

把一个小桥段拆成“人物是谁 + 这场戏要干什么”，再往外发散。**小梗不是写完就完，它应该顺手加深人设、推进关系、服务主线，或者给后文埋一个更值钱的东西。**

## 适用情况
- 想到一个小场景，但不知道后面怎么接
- 同一个桥段只能想到一种写法
- 小甜梗、小日常写完像废戏
- 想让一个场景同时立人设、推进感情、服务主剧情
- 觉得“大梗难想，小梗也发散不起来”

## 核心原则
1. **先看人物，再看场景。** 不同身份、性格、处境，会把同一个场景写成完全不同的戏。
2. **先定功能，再定走向。** 场景要服务什么，决定它往哪里拐。
3. 小梗最怕“可爱但无用”。
4. 发散不是乱开分支，而是找最适合当前主线的那一版。

## 步骤
### 1. 先写死这场景的固定部分
用一句话写：
`谁和谁在什么地方，发生了什么小事。`

例如：
- 男主陪女主路过套圈摊
- 两个人在雨天共撑一把伞
- 主角在酒会上掉了耳环

### 2. 再写人物条件
至少写清：
- 这个人物是什么身份
- 性格底色是什么
- 当前和另一人的关系阶段
- 他最擅长或最别扭的地方是什么

**同一个桥段，霸总、学生、明星、刺客、社恐，写法都不一样。**

### 3. 给场景选一个主功能
从下面至少选 1 项，最好不超过 2 项：
- 让人物排忧解难
- 加深好感 / 拉近关系
- 展示某个苏点或技能
- 提供后文有妙用的物件 / 线索 / 人脉
- 让人物正式介入主剧情
- 扩展技能树，给后续主线服务
- 做一口轻松糖 / 小爽点

### 4. 按“身份差”至少发散 3 版
同一场景，分别按不同人物版本写：
- 如果他是高位强势型，会怎么处理
- 如果他是青涩别扭型，会怎么处理
- 如果他是聪明撩人型，会怎么处理

每一版都要回答：
- 他会做什么
- 另一人会怎么反应
- 局势会变什么

### 5. 再按“功能差”发散 3 个结果
同一个动作，可以把场景导向不同结果：
- 结果 A：解决眼前麻烦
- 结果 B：两人关系更近
- 结果 C：顺手挂上主线
- 结果 D：让人物被旁人认出 / 重新评价
- 结果 E：埋一个后面能派上用场的东西

### 6. 选最值钱的一版，不选最热闹的一版
比较每个版本：
- 哪个最贴人设
- 哪个最能让人物显得带感
- 哪个最能顺手推进主线
- 哪个最能留后续

**最热闹的不一定最好，最能继续供戏的才最好。**

### 7. 给选中的版本再补一个“后续钩子”
场景结束时，至少留下一项变化：
- 好感升温
- 新误会 / 新暧昧
- 得到一件后面有用的东西
- 某人重新认识了主角
- 主角因此被卷入更大的事

## 输出格式
1. 固定场景
2. 人物条件
3. 这场戏的主功能
4. 三个身份版本
5. 三个结果版本
6. 最终选中的版本
7. 场景后的局势变化
8. 后续钩子

## 快速示例
### 输入
- 场景：男主陪女主在套圈摊前停下

### 输出
- 版本 A（高位强势型）：男主面不改色把整摊玩具全买下，直接打包给女主
- 版本 B（青涩学生型）：男主前几次没套中，嘴硬安慰女主，最后靠最后三个圈险险套中最好那个
- 版本 C（撩人型）：女主没套中，男主从背后半抱着她手把手教，顺手推进亲密互动
- 版本 D（主线型）：老板家小孩认出男主是某领域大佬，请他签名合照，男主因此正式被卷入一条新剧情
- 最终选择：如果当前重点是感情线，选 C；如果当前重点是主线介入，选 D

## 注意事项
- **不要**把小场景写成和主线完全无关的可爱摆件
- **不要**所有人物都用同一种处理方式
- **不要**只发散动作，不发散结果
- **不要**场景结束后一切如常，什么都没变
- 好用的小梗，不是“写了个小插曲”，而是**借一个小场景，顺手把人物、关系或主线往前推了一步。**

先承认一件事：设定不是拿来卡死剧情的。**当设定和情节打架时，优先保住情节的流畅与合理，再回头修设定，让规则继续为故事服务。**

## 适用情况
- 写着写着设定和情节衔接不上
- 既有世界规则卡住人物行动
- 想改设定又怕前后不一致
- 为了守设定，只能让人物做违和的事

## 输入
- 当前卡住的情节
- 相关设定条目
- 必须保住的剧情结果
- 前文已经写死的规则
- 最担心的漏洞

## 核心原则
1. **设定为情节服务，不为阻碍情节存在。**
2. 不能随手推翻旧设定，但可以补条件、补层级、补代价。
3. 真正该优先保的是**情节合理性**，不是设定字面完整性。
4. 修设定时，尽量让新解释反过来增强剧情，而不是只补漏洞。

## 步骤
### 1. 先写这段剧情到底非成不可什么
用一句话写：
`这一段无论怎么修，都必须完成____。`

例如：人物顺利相遇、主角必须脱困、真相要在此处揭开、冲突必须升级。

### 2. 找出真正卡住剧情的设定点
不要笼统写“设定不对”。
具体指出：
- 哪条规则卡住了
- 它怎么卡住了
- 如果硬守，会让谁变得不合理

### 3. 把设定分成“核心不可动”和“解释可调”
#### 核心不可动
- 世界观根基
- 题材卖点
- 读者已反复确认的硬规则

#### 解释可调
- 触发条件
- 使用范围
- 代价
- 时间窗口
- 特例来源
- 信息误差

**先守核心，再修外围。**

### 4. 选一种修法
常用修法：
- **补条件**：原来不是绝对不能，而是需要满足某条件
- **补代价**：原来能做，但代价很高，所以之前没这么做
- **补层级**：不同人、不同地点、不同时期规则不同
- **补例外**：例外不是天降，要有身份、道具、前史或权力来源
- **改解释**：前文只是角色理解，不等于世界真相
- **改路径**：不改终点，只换到更合规的达成方式

### 5. 先修设定，再回查前文痕迹
修完后检查：
- 前文有没有一句能提前埋住这个修法
- 哪些地方需要补一笔提示
- 读者回看时能不能说得通

如果完全没有前文支点，就尽量选改动更小的修法。

### 6. 保证后果连贯
每改一条规则，都要补：
- 谁因此获得新限制/新成本
- 谁因此有了新机会
- 后面哪几场戏要同步调整

**不带后果的设定修补，最像补丁。**

### 7. 如果必须改情节，先保合理性
当设定实在救不动时：
- 宁可改剧情路径
- 也不要让角色突然降智、失忆、违背性格去硬过关

句式：
`如果只能二选一，优先保住____的合理性。`

### 8. 做“天帝权限”自查
作者当然能改规则，但必须回答：
- 为什么现在才暴露这条新规则
- 这条规则是不是只为这一次服务
- 删掉这条新规则，故事还能不能站住

如果新规则只救这一场，后面再也没用，说明修得还不够稳。

## 输出格式
1. 必须保住的剧情结果
2. 卡点设定
3. 核心不可动 / 解释可调
4. 推荐修法
5. 需要补的前文痕迹
6. 修后带来的后果
7. 如果仍要改情节，怎么改更合理

## 快速示例
### 输入
- 设定里结界不能被打破，但剧情需要主角进入

### 输出
- 必保结果：主角必须在此处进入结界
- 卡点：绝对规则写死
- 可调部分：触发条件与代价
- 修法：补条件——结界只有在某时辰、持某物时会短暂变弱；补代价——进入后会留下可追踪痕迹
- 回查：前文提前埋下时辰、信物或旧人提过的异常
- 后果：主角能进去，但会引出后续追杀

## 注意事项
- **禁止**为了救剧情，临时发明一次性外挂
- **禁止**把角色降智当成设定修补
- **禁止**推翻已反复确认的核心规则
- **禁止**修完这一场，却不管后文连锁反应
- 好的设定修补，不是把漏洞补平，而是**顺手把剧情推得更顺、更有代价、更像早就安排好**。

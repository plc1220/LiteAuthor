用大婚、拜堂、揭盖头、洞房等仪式场景开局，在一场戏里同时立主角、抛矛盾、塞设定、做第一重反转。**婚礼开场最值钱的不是“满眼红”，而是“喜事现场哪里不对，以及主角当场怎么处理”。**

## 适用情况
- 古言、穿越、重生、替嫁、先婚后爱、宅斗、宫斗、悬疑古言需要强开篇
- 想让读者一章内同时吃到仪式感、冲突和钩子
- 已经写成“一睁眼一片红”，但开头撞模板、没差异
- 开篇信息多，不知道怎么自然塞进场面

## 输入
- 题材 / 世界观
- 主角当前状态：正常 / 穿越 / 重生 / 失忆 / 时间循环 / 半知情
- 主角对婚事的态度：愿意 / 不愿意 / 被迫 / 将计就计 / 另有目的
- 这场婚礼最大问题
- 想立住的主角性格
- 前 1 章准备回收的第一重反转

## 核心原则
1. **婚礼开场 = 仪式感 + 异常 + 选择。**
2. 先让读者看到不对劲，再解释原因。
3. 主角必须在前 300—800 字内做出判断、提问、试探、拒绝、逃跑、装乖、动手等动作，**不能只会懵和震惊**。
4. 用婚礼道具、礼制、宾客反应和对白偷渡设定，不要先停下来讲背景。
5. **别把重点写成“红嫁衣很好看”，要写成“这场婚为什么危险 / 荒唐 / 不该发生”。**
6. 同样是婚礼开场，差异化不在词藻，在于：主角知道多少、想做什么、现场最异常的东西是什么。

## 先定四件事
### 1. 主角知道多少
- **全不知道**：穿越、失忆、误入，靠观察和套话补信息
- **半知道**：知道自己被迫成亲，但不知道对手真实目的
- **全知道**：重生、复仇、将计就计，重点写选择而不是认知
- **知道结果不知道过程**：时间循环、预知死局，重点写抢时间

### 2. 这场婚礼最大的不对劲
只选 1 个主问题：
- 嫁错人 / 新郎新娘身份错位
- 婚事背后有阴谋、逼婚、替嫁、冲喜
- 新郎已死、失踪、重伤，甚至是牌位 / 冥婚
- 婚礼里藏刺杀、灭门、兵变、谈判
- 婚礼现场有命案、鬼怪、怪谈
- 主角重生，必须当场拒婚或改命
- 主角被卷进时间循环，每次都死在婚后
- 喜堂只是局，真正目标是引主角入场

把它写成一句话：
`这场婚礼真正让读者想追问的是：为什么____？`

### 3. 主角第一反应
从下面选最贴人设的一类：
- 闹：掀盖头、退婚、发难
- 忍：先顺礼、先观察、先记人
- 试：问关键问题、验证身份、试探立场
- 骗：装失忆、装柔弱、装配合
- 逃：装晕、借乱脱身、提前藏后手
- 打：先挟持、先刺杀、先抢主动权
- 谈：立刻谈条件、换合作、改立场

### 4. 章尾钩子
- 婚礼被打断，但更大的局刚开始
- 主角刚拒婚，发现真正掌局者不是眼前人
- 主角以为逃出去了，却又回到婚礼原点
- 主角刚想反击，对方说出一句颠覆前情的话
- 主角当场选了阵营，代价立刻落下

## 用细节快速定场
优先从下面挑 **3—5 个**，不要全上：
- 视觉：盖头、喜烛、双喜红绸、凤冠、花轿、香案
- 听觉：唱礼、锣鼓、鞭炮、窃语、突然死寂
- 触觉：凤冠压头、嫁衣闷热、红盖头遮视线、刀尖贴颈
- 人物：媒婆、喜娘、丫鬟、宾客、父母、对面新郎
- 异常：空席、白幔、血迹、牌位、哭声、没人敢抬头、礼数错位

**推荐写法：正常婚礼细节 2—3 笔 + 一个异常细节 + 主角反应。**
这样既有气氛，又能立刻起冲突。

## 差异化写法
### 不要所有开头都从“睁眼看见一片红”写起
可替换切口：
- 先听见唱礼 / 嘲讽 / 哭声
- 先感到头饰沉、盖头闷、刀贴喉
- 先看到不该出现在婚礼上的东西：牌位、血、兵器、空棺、白幔
- 先写主角已经做出一个反常动作：掀盖头、拒婚、逃跑、刺杀、装晕

### 让“卖点”长在婚礼异常上
把题材卖点直接塞进第一场：
- 穿越：信息缺口和现挂反应
- 重生：前世记忆和当场改命
- 悬疑：婚礼命案 / 怪谈 / 诡异规矩
- 宅斗权谋：婚事即站队
- 无限循环：同一场婚礼反复开局
- 先婚后爱：婚前立场对撞，不先写甜

## 常见变体库
1. **信息缺口型**：主角刚醒，先从礼制和旁人对白里拼出自己要嫁谁  
2. **拒婚改命型**：主角已知婚后是死局，当场翻桌  
3. **洞房藏刀型**：婚礼表面圆满，实则刺杀 / 谈判 / 反水  
4. **身份错位型**：新郎不是该娶的人，或主角其实是新郎 / 替身 / 嫁牌位  
5. **婚礼悬疑型**：喜宴同时发生命案、怪事、失踪  
6. **循环逃生型**：主角多次死在婚后，开局一切动作都为保命  
7. **鬼婚残念型**：婚礼是亡者执念，重点在认知反转和情感回收

## 推荐节拍
1. 1—2 句定场：婚礼正在进行  
2. 抛一个异常细节  
3. 主角做第一反应  
4. 对话带出半步前情  
5. 第一重反转  
6. 章尾留下更大的问题

## 输出格式
1. 题材与婚礼背景
2. 主角知道多少
3. 这场婚礼最大的不对劲
4. 用来定场的 3—5 个细节
5. 主角第一反应与对应性格
6. 第一重反转
7. 章尾钩子
8. 开篇 300—800 字 beats

## 快速示例
### 输入
- 古言重生
- 女主重生回出嫁当日，知道自己婚后会被卷进夺嫡惨死
- 想写女主冷静、强硬

### 输出
- 知情程度：全知道
- 不对劲：表面是盛大婚礼，实则是前世悲剧起点
- 定场细节：唱礼声、红盖头、凤冠压得额角生疼、瑞王伸来扶她的手、满堂骤静
- 第一反应：掀盖头，当场拒婚
- 第一重反转：父亲先要打她，却被更高权势者拦下
- 章尾钩子：她以为只是在退婚，结果真正的夺局从此刻开始

## 注意事项
- **不要**把婚礼开场写成纯背景说明
- **不要**让主角只会震惊、哭、发呆，没有动作
- **不要**一口气塞 4 个同等重要的反转
- **不要**所有细节都只服务“华美”，不服务危机
- **不要**把同类题材都写成同一种“醒来见红”模板
- 婚礼开场真正好看的是：**仪式越盛大，异常越刺眼，主角选择越尖锐。**

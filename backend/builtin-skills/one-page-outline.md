用一页纸把“为什么写、在讲什么、谁来制造问题、主角会怎么变”写明白，再开文。**大纲不是写给别人看的文学作品，而是防止自己写着写着忘了最初想写什么。**

## 适用情况
- 刚有灵感，不知道怎么展开
- 三章后容易跑偏、卡文、写崩
- 想要轻量大纲，不想一上来做复杂文档
- 开文前需要先抓骨架，再补细节

## 输入
- 一句话灵感、桥段、角色形象或氛围
- 题材与预计篇幅
- 主角和核心关系
- 期望结局或想收回的核心情绪

## 步骤

### 1. 写灵感激发点
先写下这篇文最初打动人的东西，只要 1—3 行。

可以是：
- 一个角色形象
- 一个关键桥段
- 一句话
- 一种情绪或氛围

作用：卡文或怀疑自己写偏时，拿出来提醒自己“当初为什么想写这个”。

### 2. 用一句话写故事核
优先使用下面任一公式：
- 一个（什么样）的人，在（什么情况下），要完成（什么事）
- 一个（什么样）的人，经历（什么事），变成了（什么样）的人

要求：
- 只写一句话
- 必须包含主角、处境、目标或变化
- 不能写成设定堆砌

### 3. 把一句话拉成 3—5 个阶段节点
把故事核拆成几步，不要一上来写细纲。

每个节点至少回答：
- 这一阶段主角在做什么
- 遇到了什么问题
- 局势发生了什么变化

常见拆法：
- 按时间段
- 按地图 / 副本 / 学年 / 职业阶段
- 按“大问题被拆开的几轮推进”

### 4. 给每个节点配“提出问题的人”和“解决问题的人”
支线人物不是摆设。至少写清：
- 谁负责制造障碍
- 谁负责提供帮助、资源或新方向
- 这些人为什么要这么做

**人物功能越清楚，大纲越不容易只剩流水账。**

### 5. 写主角的起点和终点
至少写两栏：
- 起点：主角现在是什么样
- 终点：主角最后会变成什么样

再补：
- 哪个阶段让主角第一次改变
- 哪个阶段逼主角做出关键选择
- 哪个阶段完成真正转变

### 6. 用“逻辑链”检查节点
逐条问：
- 为什么是现在发生这件事
- 为什么是这个人来挡路或帮忙
- 如果删掉这一节点，故事核会不会塌
- 这一步有没有逼主角改变

答不上来，说明这一步还只是热闹，不是骨架。

### 7. 一页只留骨架，不写设定大全
一页大纲只保留：
- 灵感激发点
- 故事核
- 3—5 个阶段节点
- 关键支线人物功能
- 主角起点 / 终点 / 转变

背景设定、支线细节、世界规则、名词解释，后面再补。

### 8. 随手记灵感，但不强塞正文
把临时想到的台词、桥段、设定关键词记到便签里。

规则：
- 能服务故事核就收进来
- 不能服务就先放着
- **灵感仓库不是必须全部兑现的购物车**

## 一页模板
```markdown
# 灵感激发点
- 

# 故事核
- 

# 阶段节点
1. 开端：
2. 发展：
3. 中段转折：
4. 高潮：
5. 结局：

# 支线人物功能
- 人物 A：负责提出什么问题
- 人物 B：负责提供什么帮助
- 人物 C：负责制造什么关键转折

# 主角转变
- 起点：
- 中途第一次改变：
- 关键选择：
- 终点：
```

## 快速示例
### 输入
- 灵感：一个错过竞赛名额的女孩重生回中考前

### 输出
- 灵感激发点：如果能回到学生时代，她最先想改掉什么
- 故事核：一个被家庭和自卑拖住的女孩，在重生后要靠学习逆袭并改变处境
- 阶段节点：
  1. 重生归来，识别旧坑，定下目标
  2. 找到学习方法，第一次用成绩翻盘
  3. 争竞赛名额，开始改变周围人的看法
  4. 用奖学金和机会改变家庭处境
  5. 完成逆袭，进入更大舞台
- 主角转变：从被动讨好、只会忍，到主动规划、敢争敢选

## 注意事项
- **禁止**把一页大纲写成设定资料库
- **禁止**只有事件列表，没有主角转变
- **禁止**支线人物只贴标签，不写功能和动机
- **禁止**灵感激发点和故事核完全脱节
- 一页大纲的目标不是精致，而是**让自己始终知道这篇文为什么存在**。

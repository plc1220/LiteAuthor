把一句会被匆匆带过的内容，改成“首句钉住 → 中间填充 → 尾句落下”的拉长段。**中间填充只负责拉开阅读时长，不负责推进新的连贯剧情。** 适合慢节奏戏、情绪停顿、关键台词前后，或原段太仓促怕读者没接住时使用。

## 适用情况
- 某个剧情点写得太快，怕读者没看清
- 想让一句关键台词、一个眼神、一个决定更有停顿感
- 想照顾跳读读者：即使中间略过，也能把首尾重点连起来
- 不适合：耳光、爆炸、突袭这类本该短促的瞬间爆点

## 输入
- 原句或原段
- 这一段真正要保住的首句信息与尾句信息
- 当前 POV
- 想加重的情绪：犹豫 / 暧昧 / 压迫 / 观察 / 余味等
- 可用的现场细节

## 核心原则
1. **段落主体在首尾两句。**
2. **中间填充必须彼此独立，不要串成一段小剧情。**
3. **不要在填充里放关键情报、关键因果、关键台词。**
4. 可以用静态描写、视线扫过、触感、环境小声响来加分。
5. **默认控制在一屏内，让跳读读者能迅速把首尾接起来。**

## 步骤
### 1. 先钉首句和尾句
分别写清：
- 这一段开始时，读者必须知道什么
- 这一段结束时，读者必须接到什么

### 2. 中间只加“可跳读填充”
从下列里选 2—5 个：
- 视线扫到的静物
- 另一个人物的局部细节
- 风、光、温度、气味、声响
- 一闪而过的当下念头
- 手指、衣角、呼吸等微动作

### 3. 检查填充是否“各自独立”
每一笔都要能单独删掉，不影响首尾理解。  
**禁止**这样写：
- 连贯回忆
- 连续推理
- 新的因果链
- 会把读者注意力整段带跑的插叙

### 4. 保持“可跳读”
把中间内容写成不看也不影响理解的东西：
- 少放关键指令和关键答案
- 少重复同一句台词
- 少解释大道理
- 多写能增加画面但不改变主干的碎片

### 5. 句式安排
- 首句利落
- 中间可适当变长
- 尾句重新收紧，落回真正要说的话 / 要发生的动作

### 6. 最后做跳读测试
假设读者只看首句和尾句：
- 还能不能无缝接上
- 重要信息有没有丢
- 中间有没有一段会把注意力彻底带偏

## 输出格式
1. 首句
2. 填充碎片清单
3. 尾句
4. 改写后的整段
5. 跳读测试结果

## 快速示例
### 输入
- 原句：男主看了女主一眼，说：“跟我走。”

### 输出
1. 首句：他先看见池边那朵半开的莲。
2. 填充碎片：水面细碎的反光；女主袖口被风拂起一角；远处隐约的钟声；她始终低着眼没有抬头。
3. 尾句：他这才收回视线，对她说：“跟我走。”
4. 改写后的整段：他先看见池边那朵半开的莲，水光碎在石阶边，女主袖口被风轻轻拂起一角，远处隐约传来一声钟鸣，她始终低着眼，没有抬头。他这才收回视线，对她说：“跟我走。”
5. 跳读测试：只看首尾也能读通；中间没有关键新信息。

## 注意事项
- **禁止**拉长时另开一段连贯回忆
- **禁止**把伏笔、答案、反转埋在填充里
- **禁止**为了拉长反复重复同一台词
- **禁止**章章都用，容易拖沓

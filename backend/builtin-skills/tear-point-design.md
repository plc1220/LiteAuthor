围绕“情绪积压后必须靠眼泪宣泄”来设计泪点。先判断读者到底是在为不公、失去、被安慰、被理解还是被共情而哭，再安排积压和爆发。**泪点不是单纯写惨，而是让人物的痛处被持续按住，直到出现无法硬撑的宣泄口。**

## 适用情况
- 想写虐点、感动点、苦尽甘来
- 角色经历很惨，但读者只觉得套路，不想哭
- 先虐后爽时，前面的痛感不够扎实
- 安慰戏、治愈戏写不出爆发力

## 输入
- 角色当前最在意的东西
- 当前处境里最明显的不公 / 失去 / 压抑
- 角色是否有能力立刻反击
- 谁会造成伤害，谁会给出安慰或认可
- 这一段最终想落到：纯泪点 / 治愈 / 先虐后爽

## 步骤

### 1. 先选泪点类型
优先从下面选 1 个主类型，必要时混 1 个副类型：
- **委屈型**：核心是不公，痛点是“本该得到的认可、尊重、待遇没有得到”
- **治愈型**：前提先有委屈，爆发点是“终于被安慰、被理解、被认可”
- **失去型**：核心不是受欺负，而是“曾经拥有、即将得到，却最终失去”
- **共情型**：让读者代入角色的普遍需求，例如归属、安全、亲情、尊严
- **扭转型**：先把泪点压到谷底，再顺势反弹成治愈或爽点

### 2. 锁定“哭点到底伤在哪”
用一句话写清：
`角色想要____，却因为____得不到 / 失去____，而且此刻无能为力。`

**没有“无能为力”，委屈感就会转弱；如果角色当场能解决，情绪更容易变成爽点。**

### 3. 把痛感落成具体的不公或失去
不要只写“很惨”“很委屈”。改写成可见情境：
- 被辜负：付出很多，却被当成理所当然
- 被轻视：能力、感情、身份、忠诚不被承认
- 被苛待：在群体里遭受不平等对待
- 被夺走：关系、机会、归属、身体、未来被拿走
- 被误解：说不清、没人信、越解释越错

### 4. 做逐级累加，不要一步到顶
按“小刺痛 → 更深的不公 / 失去 → 最重的无力”排 3-5 级。每一级至少加重 1 项：
- 待遇更差
- 代价更大
- 围观者更多
- 能求助的人更少
- 自尊心被压得更狠

**同一种惨反复堆不叫升级，必须让痛的维度变多。**

### 5. 设计“硬撑”阶段
真正能戳泪点的，常常不是哭出来，而是还在硬撑：
- 逞强说没事
- 把求助咽回去
- 习惯性自我贬低
- 明明受伤还先照顾别人
- 被安慰时第一反应是不敢信

先让角色硬住，后面情绪才有崩口。

### 6. 选爆发方式
根据类型安排最合适的宣泄口：
- 委屈型：真相被看见、终于有人站在角色这边
- 治愈型：一句精准安慰、一次被维护、一次被认可
- 失去型：遗物、旧习惯、迟来的理解、差一步就来不及
- 共情型：读者熟悉的恐惧或遗憾被点中

**治愈型的安慰必须戳中具体痛处，泛泛说“没事了”通常不够。**

### 7. 想先虐后爽，就把泪点接到反弹
如果要转爽，按这个顺序写：
`不公 / 失去积压 → 情绪宣泄 → 有人站队 / 真相揭开 / 主角获得反击条件 → 爽点回收`

**不要在情绪刚起时立刻翻盘。先让读者真正难受一下，再给回弹。**

### 8. 检查是不是“只有惨，没有泪”
逐条检查：
- 痛点是不是人人都看得懂
- 角色是不是确实在乎这件事
- 情绪有没有逐级累加
- 爆发口是不是精准对准前文痛处
- 这一哭之后，关系或局势有没有变化

## 输出格式
1. 主泪点类型 / 副类型
2. 角色最痛的需求
3. 不公 / 失去来源
4. 3-5 级累加链
5. 硬撑表现
6. 爆发场景
7. 是否转治愈 / 转爽
8. 余波

## 快速示例
### 输入
- 女主从小被家里偏心忽视，努力考第一也没人夸
- 想写“终于有人看见她”的治愈泪点

### 输出
- 主类型：委屈型 + 治愈型
- 最痛需求：被认可
- 不公来源：长期偏心和否定
- 累加链：成绩被无视 → 功劳让给弟弟 → 生病没人管 → 还被说矫情
- 硬撑表现：嘴上说习惯了，私下把奖状折起来塞抽屉
- 爆发场景：班主任在家长会当众说“她一直很优秀，也一直没人替她说过一句公道话”
- 余波：女主第一次哭出来，后面自然能接被维护、被偏爱或反击家人的情节

## 注意事项
- **禁止**只靠旁白说“她好委屈”
- **禁止**无差别堆惨
- 委屈型和失去型要区分清楚
- 想写治愈，先把前面的委屈写准

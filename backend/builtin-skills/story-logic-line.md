围绕一条核心逻辑线整理剧情，让所有人物、细节、视角和支线都朝同一关注点聚拢。**小说不是把“现实里合理会发生的事”一股脑写出来，而是把最该被看到的事排成一条线。片段可以塑造看点，全文必须服务主要矛盾和人物变化。**

## 适用情况
- 明明每段都说得通，合起来却松散
- 写着写着跑偏，主角被配角或支线挤掉
- 不知道某个视角切换值不值
- 片段有爽点，全文没有线
- 总想给路人、配角补完整背景，结果越写越散
- 开头或章节里总觉得“注意力不集中”

## 输入
- 作品一句话卖点
- 要检查的范围：全文 / 卷 / 章 / 场景
- 当前阶段的主要矛盾
- 想突出的人物、关系或看点
- 当前梗概、细纲或正文片段
- 已知卡点

## 核心原则
1. **全文服务主要矛盾和人物变化。**
2. 单个片段可以帮忙立看点，但**看点必须长在线上**。
3. **文中逻辑 ≠ 现实常识逻辑。** 小说不需要替每个路人单开一生。
4. 非主角视角不是不能用，但它的**视线必须往主角或当前关注点聚集**。
5. 任何人物、细节、支线，如果不能加压、铺垫、转折、回收，就别硬扩写。

## 步骤
### 1. 先写一句“这段的意”
先用一句话钉死：
`这段存在，是为了让____发生 / 加深 / 暴露 / 失控。`

可写成：
- 让男女主关系更近一步
- 让主角看见更大的对手
- 让某个误会升级成真正矛盾
- 让反派第一次显出压迫感

**写不出这句话，说明这段还没立住。**

### 2. 画出当前逻辑线
按 4 格整理：
1. 当前缺口是什么
2. 这一段怎么加压或推进
3. 这一段的落点是什么
4. 它把读者往后面哪一笔账上挂

如果这一段的终点不能接到后文，只能自成一段热闹，多半就偏了。

### 3. 给每个场景标功能
逐场景或逐段标 1 个主功能：
- 加压
- 铺垫
- 转折
- 揭示
- 回收
- 聚焦人物
- 留钩子

同一场戏最好还有 1 个副功能，但**不要什么都想干**。

答不上功能的内容，优先删、并、后置。

### 4. 区分“文中逻辑”和“常识逻辑”
常见跑偏写法：
- 某个路人只是来施压或见证
- 但因为“他不可能无缘无故这么做”，就开始补他完整背景
- 补着补着，主角被拖进他的私人支线
- 全文主线被打断

处理原则：
- 只补到**足够成立当前行为**的程度
- 不把功能角色扩成新主角
- 真要扩写，必须证明他后面会持续改变主线

### 5. 检查视角的“视线集中点”
把视角当成真实视线处理，而不是标签。

每次切视角都问：
1. 这个人此刻第一眼会看谁 / 看什么
2. 这段视线是不是在向主角、关键人物或矛盾点聚集
3. 切过去后，读者理解有没有更集中，而不是更分散

如果新视角只是为了解释别人怎么想、替配角说理、顺便展示别人的生活，多半不值。

### 6. 做“点在线上”检查
把当前片段里的看点列出来，再检查它们是否真的挂在主线上：
- 这个看点是在加压主矛盾吗
- 它有没有改变关系、立场、局势或后续期待
- 删掉后，全文会不会更集中

**片段是点，全文是线。点不在线上，就只是孤立花活。**

### 7. 对支线做三选一处理
发现偏离主线的内容时，只做 3 种处理：
1. 删掉
2. 压缩成一句或一小段
3. 改写成主线的后果、障碍或助推器

不要一边说它无关，一边舍不得篇幅。

### 8. 输出重排方案
至少给出：
- 核心逻辑线一句话
- 每场戏主功能
- 应删或应压缩的枝杈
- 该把注意力收回谁身上
- 视角切换是否保留
- 重排后的顺序

## 诊断模板
| 内容 | 当前作用 | 是否服务主线 | 处理方式 |
|---|---|---|---|
| 场景 / 角色 / 视角 | 加压 / 铺垫 / 转折等 | 是 / 否 / 部分 | 保留 / 压缩 / 删除 / 改写 |

## 快速示例
### 示例 1：加班场景
- 这场戏的意：让男女主关系推进，并给后续舆论埋钩子
- 有效细节：男主纠结怎么送她回家；同事语音突然外放；坏掉的雨伞制造同行理由
- 无效扩写：顺手把某个同事的家庭纠纷单独写成半章
- 原因：同事只需要承担“放大误会”的功能，不该抢走主线

### 示例 2：非主角视角出场
- 目标：突出某个核心人物第一次正式出场
- 稳的写法：先用旁观者视角进入场面，再让视线一步步聚到核心人物身上
- 不稳的写法：旁观者刚出场，就去写中介、路人、其他配角各自的想法
- 原因：视线没有聚焦，注意力被拆散了

## 输出格式
1. 当前范围的核心逻辑线
2. 当前主要矛盾与人物变化
3. 场景 / 段落功能表
4. 视角集中点检查
5. 偏离主线的枝杈
6. 删改建议
7. 重排后的简版提纲

## 注意事项
- **不要**把“所有事都有原因”误当成“所有原因都要写”
- **不要**让路人和支线因为补背景而变成新主角
- **不要**为了显得复杂，到处切视角分散注意力
- **不要**把单场爽点写得很满，却和全文没有关系
- 真正的行文逻辑，不是“事情都合理”，而是**所有内容都朝同一条线发力**。

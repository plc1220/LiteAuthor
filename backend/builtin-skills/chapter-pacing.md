按章节篇幅控制切场次数和情节点密度，让一章只做一件最值钱的事。**以 3000 字网文章为基准：场景转换尽量不超过 3 次，核心情节点尽量不超过 4 个。** 章节越短，越要少切、少散、少解释。

## 适用情况
- 一章里信息很多，读起来却发散
- 章节总在换地点、换人物、换话题
- 3000 字左右短章写成流水账
- 章尾平收，读者记不住这章到底发生了什么

## 输入
- 本章预计字数
- 本章主目标（推进剧情 / 推进感情 / 落爽点 / 埋钩子）
- 候选场景和想写的事件
- 下一章准备承接的内容

## 步骤
### 1. 先定本章唯一主任务
先写一句：
`这一章结束后，局势必须发生什么变化。`

没有唯一主任务，就会什么都沾一点，什么都不够重。

### 2. 用“3000 字基准线”先卡密度
- **2000—3500 字**：建议只保 1 条主线，**最多 3 次场景转换，最多 4 个核心情节点**
- **3500—5000 字**：可略放宽，但仍优先控制在 2 条并行内
- **低于 2000 字**：尽量只保 1—2 个情节点

核心情节点指会改变局势的点，不是每一句对话。

### 3. 给每个场景写“变化值”
每个场景必须至少完成 1 项：
- 新信息
- 新阻碍
- 新关系变化
- 新决定
- 新结果

答不上来就删、并、挪。

### 4. 先排四拍，再填细节
短章优先按这四拍排：
1. 进入当前局面
2. 出现干扰或波折
3. 落下本章最值钱的变化
4. 在结果将爆未爆处收尾或留钩

### 5. 超量时只做四种处理
- **并场景**：能在一个地点完成的，不换地点
- **压缩成交代**：不值钱的过程一句带过
- **后置**：把次要波折挪到下一章
- **拆章**：如果两件事都值钱，就不要硬塞同章

### 6. 章尾优先停在“最值钱的一拍”
不要在高潮后立刻加长解释，把最值钱的动作、答案、对话或钩子留在章尾附近。

## 输出格式
1. 本章主任务
2. 推荐场景数
3. 核心情节点 1—4 个
4. 每个场景的变化值
5. 需要删除 / 合并 / 后置的内容
6. 推荐章尾位置

## 快速示例
### 输入
- 3000 字
- 主任务：写主角参加晋级赛并立住实力

### 输出
- 场景 1：赛前被轻视，立冲突
- 场景 2：第一轮直接赢，态度开始松动
- 场景 3：更强对手出场，章尾留“主角真正的底牌还没出”
- 删除：赛场外闲聊、观众长段背景、回家日常

## 注意事项
- **不要**一章同时硬推三条大线
- **不要**靠频繁切场制造“内容很多”的假象
- **不要**每个情节点都解释到底
- 短章最重要的不是“塞满”，而是**让读者记住这章到底发生了什么**

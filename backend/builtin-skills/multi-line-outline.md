搭大纲时，先分线，再分层。**每条线要知道“在讲什么、为了什么、什么时候推进”，每份提纲要知道自己负责哪一级信息。**

## 适用情况
- 大纲一做就乱，写着写着跑脱
- 有感情线、事业线、世界观，但总是混成一团
- 有大剧情，没有中间推进思路
- 容易卡文、后期乏力、越写越散
- 想写 30 万字以上长篇，但不知道主线怎么吊住

## 输入
- 题材与一句话核心卖点
- 预计字数
- 主角 / 官配 / 反派 / 关键配角
- 现有设定与主冲突
- 想要的主要体验（爽 / 甜 / 成长 / 悬念等）

## 步骤

### 1. 先列出故事里到底有几条线
至少分清：
- 主线：这本书最重要的矛盾或目标
- 感情线：为了甜、虐、拉扯还是关系确认
- 事业线 / 升级线：为了爽、成长、逆袭还是权力变化
- 世界背景线：为了规则、势力、谜团还是时代变化
- 个人成长线（可选）
- 其他副线（可选）

给每条线写一句话：
`这条线在讲____，存在的目的是什么____。`

### 2. 每条线都写“起点 → 变化 → 终点”
每条线至少写清：
- 初始状态
- 2-4 个关键变化
- 最终要到哪里

如果一条线只知道开头，不知道会怎么变，它迟早会丢。

### 3. 把大纲拆成不同层级的文档
不要用一份提纲承担所有工作。至少拆成下面几层：

#### 世界背景
只记录会影响剧情的规则、势力、历史、资源和限制。

#### 时间轴（真实顺序）
按故事里真实发生的先后顺序记录事件，专门防止因果混乱。

#### 大纲（叙述顺序）
只记大剧情和大转折，重点写前因后果。

检验标准：能看懂“谁在什么时候、什么地方、做了什么、为什么”。

#### 细纲
把大纲里的剧情点扩成能想象出场面的内容：
- 这场戏具体发生了什么
- 谁想达到什么目的
- 冲突怎么起
- 这场戏会把哪条线往前推一步

#### 章纲
规划一章里先写什么后写什么，有多少梗，卡在哪里，章尾钩子是什么。

#### 梗杂 / 人设
单独放核心梗、人设冲突、设定冲突、备选桥段、角色资料，避免挤进主提纲里。

### 4. 先用大纲检查因果，再用细纲检查可写性
大纲重点查：
- 前因后果顺不顺
- 转折是不是有理由
- 这段删掉后主线会不会更清楚

细纲重点查：
- 能不能想象出场景
- 这一场具体怎么演
- 谁和谁的关系发生了什么变化

### 5. 给每条线安排入场时机和交汇点
提前决定：
- 哪条线先出来
- 哪条线后出来
- 它们在哪些节点交汇、互相放大
- 哪一阶段主推爽，哪一阶段主推甜，哪一阶段主推悬念

**不要所有线一起抢开场。**

### 6. 长篇必须留一个“大矛盾”一直吊到完结前
完结前要始终有一个最大目标或最大问题吊着读者。

常见形式：
- 主角成神 / 登顶 / 复仇成功
- 男女主真正站到一起
- 真相揭开
- 最终敌人被解决

按预计字数给它分阶段推进，不要中途提前做完主线再硬拖。

### 7. 每写一章，先标本章服务哪几条线
每章开写前先写一句：
`本章主要推进____线，顺带推进____线。`

如果一章服务不了任何线，删掉或并入别章。

### 8. 线和线之间要有主次，不要平均用力
常见分工：
- 主线负责吊读者
- 感情线负责拉扯和回收情绪
- 事业线负责阶段爽点和地位变化
- 世界背景负责压力和舞台升级

不同阶段允许主次变化，但不能长期失踪。

### 9. 自检容易跑脱的地方
重点检查：
- 某条线有没有存在感太弱
- 某条线是不是太久没推进
- 世界观是不是在单独演戏
- 感情线和主线有没有互相打架
- 章纲是否只在记流水顺序，没有梗和钩子

## 输出格式
1. 故事线清单（每条线的目的）
2. 各线的起点 / 关键变化 / 终点
3. 世界背景要点
4. 时间轴
5. 大纲节点
6. 细纲展开方式
7. 章纲模板
8. 长篇主线推进表
9. 跑脱风险点

## 快速示例
### 输入
- 题材：校园言情
- 核心卖点：青梅竹马重逢后，一边竞赛逆袭，一边关系拉扯

### 输出
- 感情线：男主从一直偏爱女主，到因误会疏离，再到重新确认关系
- 事业线：男主竞赛天赋一路兑现，地位不断抬高
- 大纲节点：重逢 → 重新同队 → 误会加深 → 竞赛翻盘 → 关系确认
- 章纲提醒：每章至少推进一条主线，并留一个下一章必须接的钩子

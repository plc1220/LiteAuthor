围绕一个常见配置，先找它真正吸引人的核心，再用连续“假如”翻动外壳：身份、资源、公司关系、立场、强弱、功能。**翻新不是乱加噱头，而是保住原梗最勾人的发动机，只换掉最套路的那一层。**

## 适用情况
- 设定一看就老：总裁/明星、豪门、先婚后爱、校园死对头、真假千金
- 有萌点，但总觉得“太常规了”
- 想在不换题材的前提下把故事改得更聪明
- 一写常见配置就只会按现成套路走

## 输入
- 当前基础配置
- 这个配置最想保留的爽点或萌点
- 目标题材和频道
- 已有的主线方向
- 不想丢掉的关键关系

## 步骤
### 1. 先把原梗写成一句最俗的版本
例：
- 小明星被总裁看重，一路宠成影帝
- 真千金回家打脸假千金
- 校园死对头从互怼到恋爱

先把“最套版”写出来，才知道要改哪层。

### 2. 分清“核心发动机”和“表面外壳”
分别写：
- 核心发动机：读者到底为什么会点这个配置
- 表面外壳：最容易套路化的身份、职业、贫富、公司归属、立场安排

**只要核心发动机还在，外壳通常都能翻。**

常见核心发动机：
- 高频相撞
- 身份反差
- 资源差
- 公开/私下双重关系
- 一方先识货
- 一方被另一方卷入新世界

### 3. 一次只翻一个轴
优先翻这 5 个轴：

#### A. 身份轴
- 如果他不是明星呢
- 如果她不是大小姐呢
- 如果他不是老师而是门卫、助理、前台呢

#### B. 资源轴
- 如果总裁其实没钱呢
- 如果豪门其实只剩壳子呢
- 如果表面强势的一方在关键问题上资源最短缺呢

#### C. 所属轴
- 如果两人不在同一家公司或阵营呢
- 如果他们表面同阵营，实际服务不同利益呢
- 如果原本的“上下级”关系被改成“合作、监管、竞争”呢

#### D. 立场轴
- 如果他不是来捧她，而是来查她呢
- 如果她不是来求助，而是来拆台呢
- 如果最初的靠近带有误判、敌意或任务呢

#### E. 功能轴
- 如果原本该被照顾的人，反而先看见了对方的困境呢
- 如果原本负责“宠”的人，根本没有这个能力呢
- 如果原本的一号炮灰，可以被抬成长期反派呢

### 4. 每翻一次，都强制回答 4 个问题
- 他们会在哪里更自然地相撞
- 新版本的核心冲突是什么
- 新版本多出来的爽点、笑点或反差是什么
- 这个设定能连续产出哪些场景，不是一章就用完吗

答不出第四个问题，这个改法多半只是点子，不是能连载的设定。

### 5. 先保“必然相撞”，再谈新鲜
翻新后，优先检查两人是不是更容易自然地产生交集，而不是更依赖巧合。
**能靠工作、利益、舆论、身份自然撞上，就不要靠逛街偶遇。**

### 6. 最多保留 1—2 个大翻动
翻一个轴，会新鲜。
同时翻四个轴，常常会散。

优先选择：
- 一个负责新鲜
- 一个负责增强冲突
其余维持稳定。

### 7. 用“如果反过来”再逼一次脑洞
在版本已经能成立后，再追问：
- 如果强弱关系反过来呢
- 如果主动方和被动方反过来呢
- 如果原本的贵人其实自顾不暇呢
- 如果原本的小角色握着更关键的信息呢

这一步适合从“能用”改到“更带感”。

### 8. 最后检查是不是只换皮，没换戏
逐条问：
- 新版比旧版多了什么新的矛盾
- 有哪些老套路被直接避开了
- 关系推进是不是更自然了
- 删除这个“新设定”后，剧情会不会立刻退回原套路

如果不会，说明这个改动还不够深。

## 输出格式
1. 原始老梗一句话
2. 核心发动机
3. 可翻的外壳轴
4. 3 个“假如”版本
5. 每个版本的新冲突 / 新爽点 / 可重复场景
6. 最推荐版本
7. 不建议再叠加的噱头
8. 后续可接的 3 条剧情线

## 快速示例
### 输入
- 配置：总裁 + 明星

### 输出
- 原始老梗：总裁看中明星，一路把他宠成顶流
- 核心发动机：行业高低差 + 高频交集 + 被看见/被捧的幻想
- 假如版本 1：明星不是明星，而是总裁经常碰到的外卖员或门卫
- 假如版本 2：总裁表面风光，实际公司快破产，根本没钱宠人
- 假如版本 3：总裁不是来捧他，而是监管舆论、盯他的数据，结果越盯越上头
- 结果：同样是“总裁×娱乐圈”，但入口、冲突和爽点都变了

## 注意事项
- **禁止**为了新鲜硬塞一堆互不兼容的噱头
- **禁止**只改职业名，不改关系发动机
- **禁止**新设定只够支撑一个段子，撑不起连续剧情
- **禁止**靠巧合维持交集
- 翻新老梗的关键不是“从没人写过”，而是**读者一看就明白为什么这个版本更想点**

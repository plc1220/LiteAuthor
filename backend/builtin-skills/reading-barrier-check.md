先把“读者顺不顺”放在第一位。**新人文笔的最低要求不是高级，而是不制造阅读障碍：读者能快速看懂句子、认清主角、跟上剧情。**

## 适用情况
- 被说文笔青涩、看不懂、读着卡
- 读者总要回看上一句或上一章
- 前三章人太多、信息太乱
- 想用隐喻、暗示、诗意表达，但怕读者吃不进去

## 输入
- 要检查的章节或片段
- 题材与预计读者
- 本段读者必须知道的核心信息 1—3 条
- 当前最怀疑会卡读者的地方

## 核心原则
1. **先让读者看懂，再谈高级感。**
2. **必须让读者知道的信息，不能写成谜语。**
3. 不重要的细节模糊一点没关系，关键处不清楚就是硬伤。
4. 读者不是来做阅读理解的。

## 步骤
### 1. 先标出“本段必须知道的 3 件事”
写下：
- 这一段读完，读者必须知道谁
- 必须知道发生了什么
- 必须知道为什么这件事重要

后面的检查都围绕这 3 件事展开。

### 2. 做“30 字 / 3 秒”句子检查
逐句看：
- 30 字内的一句话，能不能在 3 秒内看懂
- 主语是谁，是否明确
- 代词“他 / 她 / 它 / 这 / 那”有没有歧义
- 句子里是否套了太多转折、插入语、喻体

看不懂就拆句、换直说、补主语。

### 3. 做人名和称谓检查
重点看前 3 章：
- 读者能不能迅速记住主角名字
- 单章是不是一口气塞了太多新名字
- 配角有没有必要此时出场并被记住
- 同一人物的称谓是否频繁切换到认不出来

**前三章优先让读者记住主角和关键关系，不要像发班级表。**

### 4. 做章节目标检查
每章都要能一句话说清：
`这一章里，谁为了什么，做了什么，结果怎样。`

如果作者自己都说不清这章在写什么，读者更难看明白。

### 5. 做因果与转折检查
看前后句、前后段之间有没有明确关系：
- 因为 A，所以 B
- 虽然 A，但是 B
- 先 A，后 B，结果 C

如果只能看懂单句，连起来却不明白，通常是中间少了逻辑桥。

### 6. 做“重点信息直达”检查
把读者必须知道的信息逐条核对：
- 有没有被埋在大段环境、心理、比喻里
- 有没有需要靠猜才能确认
- 有没有先给一堆枝节，核心反而后置

原则：
- 关键身份、关键关系、关键结果要尽量直给
- 细节气氛、次要背景可以后置

### 7. 做比喻和暗喻检查
逐条问：
- 这个比喻是否比直说更清楚
- 读者不理解它，是否会错过关键信息
- 这句是不是把简单意思写得更绕了

如果比喻只增加阅读负担，不增加信息和情绪，就删或改直白。

### 8. 做“停顿点”检查
找出读者最可能停住的地方：
- 一句读完要回看
- 一个名字分不清是谁
- 一段看完不知道场景里到底发生了什么
- 一章看完不知道重点是什么

把这些停顿点一个个改成顺滑点。

## 输出格式
1. 本段必须知道的 3 件事
2. 30 字 / 3 秒不过关的句子
3. 人名与称谓过载点
4. 章节目标是否清楚
5. 缺失的逻辑桥
6. 被写绕的重点信息
7. 应删改的比喻 / 暗喻
8. 一版更顺的改写建议

## 快速示例
### 输入
- 第一章出现 9 个名字、3 条支线、很多代词
- 读者反馈“没记住谁是谁”

### 输出
- 必须知道：主角是谁、当前处境是什么、第一章冲突是什么
- 问题：
  - 人名过多，配角出场顺序失控
  - 代词指代不明
  - 核心冲突被闲聊和背景说明盖住
- 修改：
  - 前置主角姓名和处境
  - 删掉可后置的配角介绍
  - 每句补清主语
  - 用一句话立住第一章冲突

## 注意事项
- **不要**把“读者没耐心”当成自己写不清的借口
- **不要**关键处靠猜，次要处反而写很满
- **不要**同时塞太多人名、关系和设定
- **不要**明知读者看不懂，还硬写成隐喻谜语
- 没有阅读障碍，不等于文一定好；但**有阅读障碍，读者通常走得很快。**

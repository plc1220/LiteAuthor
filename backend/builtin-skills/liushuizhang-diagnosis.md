先别急着修句子。**很多“又跳又水”的问题，不是文笔差，而是该重的没重、该轻的没轻：什么都想写，结果关键场景没真正展开，次要场景又占了篇幅。**

## 适用情况
- 读者有人说太跳，有人说像流水账
- 一写就越写越细、越写越长
- 明明有情节，读完却没有记住哪一场
- 场景很多，但转折交代生硬
- 关键戏没吃够，过场戏反而很长

## 输入
- 要排查的章节 / 片段
- 这一章的一句话目标
- 现有大纲或场景列表
- 作者最怀疑发水或跳脱的地方

## 步骤
### 1. 先把这一章压成一句话
按模板写：
`主角为了____，做了____，结果____。`

写不出来，先补目标和结果。

### 2. 判断是哪一类问题
#### A. 大纲空无一物
表现：没有明确目标、阻力、代价，只能写日常动作和聊天。

#### B. 只有一级纲，没有事件链
表现：只知道结果，不知道靠什么具体事件走过去。

#### C. 有事件，没有结果
表现：场景里发生了很多动作、对话、转场，但读完前后局势没变。

#### D. 修文补丁过多
表现：新内容加上去了，旧废话没删，章节层层叠叠。

#### E. **重轻失衡：什么都想写，关键场景却没真正展开**
表现：
- 每个场景都沾一下，但都写不深
- 重要转折只一两句带过，日常流程反而写很细
- 场景之间不断切，读者觉得跳
- 因为什么都写了一点，又像流水账

处理：
- 先分 **重场景 / 轻场景**
- 十个场景里，着力写好 3—4 个就够
- **重场景优先选**：展现人物性格、关系改变量、关键冲突、重大信息
- 其余场景用一句话、一个时间点、几句对话带过

### 3. 列一张“场景主次表”
逐场标 4 项：
- 这场最主要的功能是什么
- 不写它会损失什么
- 它是重场景还是轻场景
- 应该详写到什么程度

标成重场景的，至少满足 1 项：
- 让人物露底色
- 改变关系状态
- 推进主线局势
- 抛出 / 回收关键信息

### 4. 重场景要真正展开
重场景至少交代清楚：
- 人物为什么这么做
- 他怎么做
- 做完之后结果如何

**不要只有“发生了”，要写到读者能吃到变化。**

### 5. 轻场景要果断压缩
轻场景常用 4 种压缩法：
- 一句话总结：`接下来三天，他都在四处托人找线索。`
- 对话补叙：用人物聊天把中间过程带出
- 时间点跳转：`半个月后。`
- 三句话结段：超过预定篇幅时，用三句收掉，转入下一情节

三句话模板：
1. 这一段发生了什么
2. 留下了什么结果 / 情绪
3. 下一场戏从哪里接

### 6. 专查“既跳又水”的根因
如果同时被说跳和水，重点查 3 件事：
1. **是不是事无巨细都想写**
2. **是不是每个转折都没真正展开**
3. **是不是缺少吸引人的重场景**

只要 3 项里中了 2 项，就按 E 类重轻失衡处理。

### 7. 最后再修文字
顺序固定为：
1. 补目标
2. 补结果
3. 分重轻
4. 压缩轻场景
5. 展开重场景
6. 最后才润句

## 输出格式
1. 当前是否属于流水账 / 既跳又水
2. 问题类型：A / B / C / D / E
3. 这一章的一句话目标
4. 场景主次表
5. 应详写的 3—4 个重场景
6. 应压缩或删除的轻场景
7. 可直接使用的“三句话结段”
8. 修改后章节顺序建议

## 快速示例
### 输入
一章里写了主角出门、赴宴、聊天、回家、查消息、再赴宴，什么都写了一点。

### 输出
- 类型：E 类重轻失衡
- 重场景：
  1. 赴宴第一次正面对上反派
  2. 回家后主角做出关键判断
  3. 第二次赴宴关系彻底翻面
- 轻场景：
  - 出门路上
  - 重复寒暄
  - 无结果的查消息流程
- 三句话结段：
  `这场宴席没有得到答案，但主角确认了反派在撒谎。`
  `回府后他一夜未眠，第二天就改了原计划。`
  `等他再踏进厅堂时，这一次已不是来试探。`

## 注意事项
- **不要**把“写得细”误当成“写得扎实”
- **不要**十个场景平均用力
- **不要**让关键转折只剩一句交代
- **不要**舍不得删重复过场
- 真正的节奏，不是一直快，而是**该重的重，该轻的轻**。

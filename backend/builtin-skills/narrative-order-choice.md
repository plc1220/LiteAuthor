先判断故事本身的矛盾够不够强，再选叙事顺序。**矛盾越强，手法越简单；矛盾越弱，越需要结构帮它出彩。** 网文优先保证清楚和抓人，不要为了“高级”把时间线切碎。

## 适用情况
- 开篇拿不准该顺着写还是倒着写
- 想靠结构制造悬念
- 故事本身不弱，但开头不抓人
- 担心插叙、环形写法把网文写冷

## 输入
- 一句话故事
- 最大矛盾或最大悬念
- 题材与预计篇幅
- 是否网文连载
- 当前开篇设想

## 步骤
### 1. 先写最值钱的东西
写一句：`这篇文最该让读者立刻看到的是____。`
可能是：大矛盾、主角处境、重大后果、最强悬念。

### 2. 判断故事属于哪一类
- **矛盾很强**：身份冲突、追杀、夺权、复仇、重大危机本身就够抓人
- **矛盾一般**：设定有趣、人设可爱，但开场缺硬冲突
- **结构型短篇**：重点在巧、在回环、在后劲

### 3. 选顺叙
适合：
- 主线冲突本来就强
- 网文长篇
- 升级、夺权、复仇、强剧情

写法：
- 直接从主角入局写
- 先让读者看清关系、目标、阻碍
- 不靠花活，靠事件推进

### 4. 选倒叙
适合：
- 开篇先抛一个大后果更抓人
- 故事中段有能钩住人的灾难、追杀、婚变、翻车、重大秘密

写法：
- 先给结果或高压场面
- 立刻抛出“为什么会这样”
- 1—2 场后尽快回到主线起点

**网文最稳的复杂写法，通常就是“复杂剧情 + 倒叙开头”。**

### 5. 选插叙
适合：
- 补关键前史
- 放大当前情绪或误会
- 解释当前选择

规则：
- 一次插叙只解决一个问题
- 插完要让现在的局势变重或更清楚
- 不要边推进边频繁回头补课

### 6. 谨慎选环形叙述
适合：
- 短篇、出版文、单元故事
- 主题回环明显，开头和结尾能形成照应

不建议：
- 网文长连载
- 本身冲突已经很强的故事
- 作者还控制不好多层时间线

### 7. 做媒介校验
如果是网文，优先检查：
- 开头 1—3 章是否足够清楚
- 读者能不能马上知道主角要干什么
- 结构是不是在帮助悬念，而不是拖慢信息理解

如果结构一复杂就影响追读，宁可改简单。

### 8. 产出开篇方案
给出：
- 推荐叙事顺序
- 第一场该写什么
- 哪些信息后置
- 哪个问题负责钩住读者
- 最大风险点

## 输出格式
1. 故事最值钱的点
2. 推荐叙事顺序
3. 选择理由
4. 开篇 3 场安排
5. 后置信息
6. 不建议用的手法
7. 风险提醒

## 快速示例
### 输入
- 皇子身处险境，暗恋女山匪，后续有权谋和感情线

### 输出
- 推荐：倒叙开头
- 第一场：先写皇子被追杀或婚局将崩的高压场面
- 钩子：他为什么会和女山匪扯上关系
- 后置：相识经过、旧恩旧账
- 风险：不要倒叙太久，三章还回不到主线

## 注意事项
- **不要**为了显得高级乱切时间线
- **不要**把插叙写成补设定流水账
- **不要**在网文里滥用环形叙述
- 结构的任务是放大卖点，不是盖住卖点

先把题材壳撕开，找出这本书到底在卖什么，再用 **角色功能、可代入场景和前三章兑现** 把卖点落地。**跟风热题材能不能成，不在于题材词热不热，而在于作者有没有把那口味写对。**

## 适用情况
- 跟风热题材却写不对味
- 设定很多，但不知道正文该先写什么
- 前三章没抓住卖点
- 文案说得很香，正文读起来货不对板
- 明明写的是豪门、娱乐圈、先婚后爱、老男人、升级文，读者却吃不到该有的体验

## 输入
- 题材与一句话梗
- 目标读者
- 主角与关键关系
- 当前最想写的桥段
- 预计前三章内容
- 题材里必须兑现的体验

## 步骤
### 1. 先写商品句：这本书真正卖什么
填空：
- `读者点进来最想吃的是____`
- `这本书真正拿来卖的是____`

**不要只写题材名，要写体验。**
- 豪门不是卖“有钱”两个字，是卖高配庇护、资源碾压、被坚定选择
- 娱乐圈不是卖“出道”，是卖被看见、舞台高光、资源升级、舆论反转
- 逆袭不是卖“升级”，是卖评价权回到主角手里

### 2. 把题材壳和底层需求拆开
按 3 层写：
1. 表层题材：豪门 / 娱乐圈 / 穿书 / 追妻 / 升级
2. 底层需求：被偏爱、被庇护、被看见、翻盘、报应、改命
3. 具体兑付：公开护短、资源反转、名场面出圈、对手吃瘪、地位上升

**如果只剩题材壳，没有底层需求，这本书很容易写成“像这个题材”，但不好看。**

### 3. 列核心梗和必吃点
列 3—5 个必写体验，每个都要能落成场景。

示例：
- 被高配角色明确护短
- 被亲近关系压榨后狠狠干回去
- 舞台 / 能力第一次惊艳全场
- 曾经看不起主角的人主动改口
- 主角拿到更高一级的选择权

### 4. 场景优先选“读者容易代入”的事
优先选这类压力或愿望：
- 被亲戚吸血、被同事针对、被上司打压、被圈层看低
- 明明有本事却没人信
- 终于有人站在主角这边
- 被普通生活里的委屈精准补偿

少用这种会让读者出戏的桥段：
- 只为热闹而硬造的浮夸冲突
- 与题材卖点无关的无效狗血
- 读者很难共情的空中楼阁式刁难

**越接地气、越能和现实委屈对上，前期越容易让新读者代入。**

### 5. 给主要角色分“功能性设定”
不要只写性格标签，要写他们各自负责什么体验。

#### 主角
至少负责 1—2 项：
- 代入入口
- 情绪反应与吐槽
- 主动翻盘
- 成长和选择

#### 高配角色 / CP / 搭档
根据题材，通常负责 1—3 项：
- 解决危机
- 提供庇护或资源
- 安抚和托底
- 放大主角价值
- 在关键时刻公开站队

#### 反派 / 对照组 / 配角
负责：
- 制造压抑
- 贡献误判和反差
- 衬出主角或高配角色的功能
- 把卖点推到台面上

**角色的功能，前三章就要有基本显示。**

### 6. 题材特权必须写实、写具体、写到位
问自己：既然选了这个题材，它最值钱的那口到底怎么被看见？

#### 豪门 / 权势题材
要让读者看见：
- 钱和资源能怎么改现实
- 身份压制力
- 护短时的底气
- “说买就买 / 说压就压 / 说保就保”的自然感

#### 娱乐圈 / 成名题材
要让读者看见：
- 舞台或作品的正反馈
- 粉丝和行业反应
- 资源升级的门槛差
- 名场面出圈后的连锁反应

#### 升级 / 事业题材
要让读者看见：
- 难度差
- 竞争者差
- 升级后的实打实好处
- 下一层门槛真的被打开

**不要只给题材名词，不给题材体验。**

### 7. 用一条流程把卖点落到开文前
按这个顺序整理：
1. 选题
2. 100—200 字故事梗概
3. 人物基础设定 / 进阶设定
4. 核心梗 / 萌点 / 爽点梳理
5. 用第 4 步反推文名、文案、前三章概要
6. 试算前三章：卖点够不够、叙事顺不顺、会不会跑偏
7. 再正式开文

### 8. 前三章至少兑现 3 件事
1. **核心卖点第一次显形**
2. **主角或关键角色的功能性展示**
3. **一个具体、可感的题材场景**
4. **一个让读者继续追的钩子**

可直接检查：
- 读完前三章，能不能说出“这本书卖什么”
- 读完前三章，能不能感觉到关键角色有什么用
- 读完前三章，能不能吃到题材特权而不是只看到设定说明

### 9. 把跑偏内容删掉或后置
出现下面情况就要改：
- 写豪门，却先花大篇幅写和卖点无关的商战
- 写娱乐圈，却不写舞台、粉丝、名场面，只写冗长日常
- 写高配庇护，却让高配角色前三章毫无作用
- 写逆袭，却先连压很多章不给任何希望或小释放

原则：
- 不服务“卖什么”的内容，删 / 并 / 后置
- 题材主菜先上，副菜后补

### 10. 最后做“货对不对板”测试
逐条问：
- 文案承诺的口味，正文前几章有没有兑现
- 题材最值钱的体验有没有写到位
- 角色功能有没有亮出来
- 当前桥段是不是在放大卖点，而不是分散卖点
- 如果把题材壳拿掉，这本书还剩什么想看

## 输出格式
1. 题材壳
2. 底层需求
3. 商品句
4. 必吃点 3—5 个
5. 主要角色功能表
6. 题材特权场景清单
7. 前三章兑现清单
8. 该删 / 后置的跑偏内容
9. 最终开文方向

## 快速示例
### 输入
- 题材：豪门老男人
- 问题：前面总想写商战，读者不买账

### 输出
- 底层需求：被高配对象庇护和偏爱
- 商品句：卖的是“强大可靠的人明确站到主角这边”
- 必吃点：护短、资源撑腰、危机解决、偏爱例外
- 功能表：主角负责代入和情绪；男主负责托底、解决危机、提供高配偏爱
- 前三章：先写一次护短或资源碾压，再补身份和事业信息；商战放后置，只保留能服务庇护感的部分

## 注意事项
- **禁止**为了像热题材而写热题材
- **禁止**前三章全在解释设定，不交付卖点
- **禁止**角色只有标签，没有功能
- **禁止**题材最值钱的体验一直不写，只在文案里存在

先把这一场真正发生了什么讲清楚，再补描写。**场景乱，通常不是辞藻不够，而是事情顺序、主语和视角没站稳。**

## 适用情况
- 明明脑子里有画面，写出来读者却说乱
- 一场争执、对话、追逐或小冲突里，看不清谁先做了什么
- 一加表情、服装、环境描写就更乱
- 多人物同场时，主语频繁切换
- 同一段里既想写动作，又想写心理和背景，结果镜头打架

## 输入
- 场景目标
- 出场人物
- 当前视角
- 这场戏的起因、经过、结果
- 角色人设关键词

## 步骤
### 1. 先写一句“这场戏到底在发生什么”
格式：`谁，因为____，对谁做了____，结果____。`
先不用修辞，只求说清。

### 2. 先排最简骨架：起因 → 经过 → 结果
只保留最必要的动作节点。
- 起因：事情为什么开始
- 经过：人物做了哪些关键动作、说了哪些关键话
- 结果：局势具体变了什么

**先别急着补表情、服装、环境。骨架没顺，细节越多越乱。**

### 3. 锁定视角，删掉越权信息
逐句检查：
- 这句是当前视角人物能看到、听到、判断到的吗
- 有没有突然跳到别人的脑子里
- 同一段里有没有偷换观察位置

### 4. 逐句补主语，别让“他/她”打架
高危情况：
- 同段连续两个“他”指不同人
- 动作一多就省主语
- 先写 A，后一句忽然变 B，却没标明

处理方法：
- 人一多时，优先点名
- 一个动作结束，再接下一个人的反应
- 同类动作别并排堆

### 5. 按动作链写，不按脑内镜头乱跳
稳妥顺序：
`看到什么 → 做什么 → 对方怎么反应 → 局势怎么变`

不要写成：
`说一句 → 衣服细节 → 对方童年阴影 → 窗外天气 → 又回到动作`

### 6. 人物先后顺序要贴人设
先问：
- 谁更急
- 谁更强势
- 谁更爱抢话或先动手
- 谁会先退、先忍、先观察

例如：
- 暴脾气角色先拍桌、先抢话、先有大动作
- 温和角色更可能后接、劝阻、停顿后反应

**动作顺序一贴人设，场面自然会顺。**

### 7. 骨架顺了，再补细节
只补能帮读者看清局面的细节：
- 表情：说明态度
- 动作：说明关系强弱
- 道具/服装：只写会影响当前画面的
- 环境：只写会影响当前动作的

每补一笔都问：这笔是让画面更清楚，还是只是好看？

### 8. 最后做“三句复述测试”
写完后用三句复述：
1. 因为什么起了这场戏
2. 中间谁做了什么
3. 最后局势怎么变了

如果三句复述不出来，这场多半还乱。

## 输出格式
1. 场景一句话
2. 起因 / 经过 / 结果
3. 当前视角
4. 关键动作链
5. 主语易乱点
6. 人设决定的动作顺序
7. 必要细节 / 可删细节
8. 整理后的场景 beats

## 快速示例
### 输入
- 两人争执：A 暴脾气，B 温和
- 问题：写出来像两个人同时在说话，看不清谁先动

### 输出
- 场景一句话：A 因为怀疑 B 隐瞒事实，先发作质问，B 试图解释，争执升级
- 动作链：A 摔杯开口 → B 愣住 → B 解释半句 → A 打断逼近 → B 后退但不服软 → 争执结果落地
- 人设顺序：先写 A 的大动作和抢话，再写 B 的克制反应
- 可删细节：衣服颜色、无关环境描写

## 注意事项
- **禁止**骨架还没顺就堆大量细节
- **禁止**一个段落里频繁偷换视角
- **禁止**多人物同场时省主语省到看不清
- **禁止**为了“有画面”打断动作链
- 清楚永远比花哨更重要。

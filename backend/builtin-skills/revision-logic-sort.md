先把信息拆开，再改句子。**改文越改越乱，通常不是文字问题，而是信息顺序、段落功能和重复量出了问题。**

## 适用情况
- 改稿次数一多，越改越乱
- 明明每段都能看懂，合起来却别扭
- 信息太多，不知道先讲什么后讲什么
- 重写一段时，总觉得哪里重复、哪里又缺
- 重生、掉马、揭秘类设定一改就打结

## 输入
- 当前章节或片段
- 必须保留的关键信息
- 当前读者必须知道的前置信息
- 想保留的高光或钩子
- 原段落顺序（可选）

## 步骤
### 1. 先列核心信息点
把这段必须讲清的内容单列出来，不要直接改正文。
例如：
- 女主前世惨死
- 女主重生
- 女主准备开始打脸

### 2. 给每条信息标作用
只标最主要的一个：
- 起因
- 当前事实
- 人物动机
- 转折
- 结果
- 钩子

### 3. 给每段写 10—15 字大意
逐段概括，不润色，只写“这段在干什么”。
如果一段概括不出来，说明它本来就散。

### 4. 重排先后顺序
优先用这几种顺序：
- 起因 → 经过 → 结果
- 旧事实 → 当前反应
- 问题 → 解释 / 答案
- 目标 → 障碍 → 行动

问自己：
- 哪个信息必须先知道，后一句才成立
- 哪个信息可以后置，先卖悬念
- 哪个信息其实重复了前文

### 5. 查重、并段、删枝杈
重点查三类：
- 同一件事说了两遍
- 两段功能相同，可以合并
- 对当前逻辑没帮助，只会拖乱的补充信息

**信息一多就乱，先删重，再谈润色。**

### 6. 给段与段补逻辑桥
每一段都回答：
- 这一段为什么接在上一段后面
- 这一段结束时，读者新知道了什么
- 下一段是顺接、转折，还是递进

答不上来，说明段落顺序还没理顺。

### 7. 最后再回到句子
等顺序定了，再去修：
- 句内主语是否明确
- 指代是否打架
- 过渡词是否过多
- 哪些地方需要拆句或合句

## 输出格式
1. 核心信息点
2. 每条信息的作用
3. 段落大意表
4. 建议顺序
5. 重复 / 可删内容
6. 需要补的逻辑桥
7. 整理后的改稿提纲

## 快速示例
### 输入
- 重生文改稿，写着写着把“前世惨死”“重生醒来”“决定报复”搅成一团

### 输出
- 核心信息：前世惨死 → 现在重生 → 目标是打脸
- 建议顺序：先点出惨死结果，再落到重生醒来，再给出“这次不再忍”的目标
- 可删：重复感慨前世有多惨的段落
- 逻辑桥：从“意识到重生”过渡到“决定报复”的心理转折

## 注意事项
- **不要**一边改句子，一边继续往里塞新信息
- **不要**舍不得删重复段落
- **不要**把“读者现在必须知道”和“作者以后想交代”混在一起
- **不要**没理顺顺序就先做精修
- 改稿先理信息，再理文字。

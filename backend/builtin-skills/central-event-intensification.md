围绕一个中心事件，层层加码重要性、特殊性和情感砝码，把“普通事”扩成能撑住大纲的主线事件。**先用最朴素的一句话定义事件，再不断追问：为什么非它不可、为什么必须现在、为什么只有主角能做。**

## 适用情况
- 中心事件太普通，撑不起全文
- 有设定、有角色，但剧情主轴发虚
- 主角的行动目标很清楚，读者却感觉不够带劲
- 想写剧情流，但关键事件缺重量

## 输入
- 一句话中心事件
- 主角身份与当前处境
- 题材、背景、时代
- 反派或阻力来源
- 想给读者的核心感受

## 步骤

### 1. 先把中心事件压成一句最朴素的话
用最简单的句式写：
`谁必须去做什么。`

例如：
- 主角要过一条河
- 主角要把证据送回国内
- 主角要在比赛前找到失踪搭档

不要一上来就堆设定。先把骨架钉死。

### 2. 给事件补“为什么非做不可”
至少答清三件事：
- **为什么必须现在做**
- **为什么失败代价很重**
- **为什么别人做不了，偏偏轮到主角**

如果这三点说不清，事件就很难有分量。

### 3. 先加“重要性”
从下面挑 3—5 项加码，不必全用：
- **时间重要性**：有最后时限、错过就没机会
- **空间重要性**：地点本身危险、稀缺、难以替代
- **社会重要性**：影响一群人、一个行业、一个阵营
- **历史 / 政治重要性**：和时代、国家、制度、阵营格局有关
- **机会重要性**：情报、机会、通道只出现一次
- **成败重要性**：成功能改命，失败会失去关键东西

写的时候不要只说“很重要”，要写成具体后果。

### 4. 再加“情感砝码”
围绕同一个事件，再补 4—6 个会让读者更在意的角度：
- **渴望**：主角最想守住什么
- **矛盾**：想达成目标，必须舍弃自己珍视的东西
- **冲突**：有人公开阻拦、竞争、背刺
- **危险**：路上真的会死人、失去、暴露
- **悬疑**：地点、任务、过去真相存在未知风险
- **成败一线**：看起来只差一步，实则随时可能翻车

**大事件只有外部 stakes 不够，最好把私人欲望也绑上去。**
例如：不只是“过河逃命”，还是“赶回去见多年未见的女儿”。

### 5. 最后加“特殊性”
让这个事件不再像谁都能套的模板。优先从这几层里选：
- **人物特殊性**：主角的职业、身份、能力、旧史很特殊
- **任务特殊性**：要带回的东西、要完成的事有独一性
- **背景特殊性**：国家、行业、家族、世界规则让它更不寻常
- **配角特殊性**：接头人、盟友、叛徒、见证者各自藏有变数
- **地点特殊性**：目的地本身和秘密、传闻、禁忌绑定

特殊性不是“更复杂”，而是“别人一听就知道这是这本书的事”。

### 6. 设计“三次受阻”或“多轮尝试”
不要让主角一上来就直达终点。至少设计 2—3 轮受阻：
1. 第一次卡在外部环境
2. 第二次卡在人或规则
3. 第三次卡在更深的真相、背叛或代价

每次失败都要：
- 暴露新信息
- 抬高代价
- 改变主角下一步策略

### 7. 加一组“对比”
用对比放大事件分量：
- 主角和普通人对比
- 主角和竞争者对比
- 主角过去的自己和现在对比
- 同一灾难里，不同人选择的对比

对比越清楚，读者越能感到这件事到底重在哪里。

### 8. 输出成“中心事件加码表”
按下面格式整理：

| 层级 | 内容 |
|---|---|
| 中心事件一句话 |  |
| 为什么非做不可 |  |
| 时间 / 空间 / 机会 stakes |  |
| 情感砝码 |  |
| 特殊性 |  |
| 第一轮受阻 |  |
| 第二轮受阻 |  |
| 第三轮受阻 |  |
| 最终高潮 |  |

## 快速示例
### 输入
- 中心事件：小明要过河回国

### 输出
- 重要性：这是边境河道，午夜后巡逻线变化，错过窗口就再无生路
- 情感砝码：他答应女儿二十岁生日一定回家
- 特殊性：他是潜伏多年的密探，身上带着影响国家命运的技术资料
- 多轮受阻：
  1. 跟大部队过河，踩踏失败
  2. 托关系上船，船被人凿穿
  3. 接头人失联，国内还怀疑他是内鬼
- 高潮：他只能独自穿越“死亡之河”，既要活着回去，也要证明自己没叛国

## 注意事项
- **不要**每加一层就换一个新故事
- **不要**只堆设定，不改变代价和选择
- **不要**把“特殊性”写成百科补丁
- 一次只围绕**一个中心事件**加码，先把它做重，再扩别的支线

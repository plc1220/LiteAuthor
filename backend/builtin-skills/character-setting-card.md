先用一张轻量人设卡抓住角色的底色、原型和作用，不必一上来填几百个问题。**人设表不是为了堆资料，而是为了知道：这个人为什么会这样想、这样说、这样做。**

## 适用情况
- 想快速立主角或配角的人设
- 人物标签很多，但一落笔就飘
- 群像角色容易撞性格、撞功能
- 不想做超长问卷，只想抓关键点
- 明知角色要发挥作用，却说不清他怎么体现

## 输入
- 角色在故事里的定位
- 当前题材和世界环境
- 角色预计出场阶段或戏份
- 与主角或核心关系的联系
- 想让读者记住的第一印象

## 写法步骤
### 1. 先定角色定位
一句话写：`这个人进入故事，最主要是为了____。`

常见定位：
- 推动主线
- 制造冲突
- 提供支撑
- 形成反差
- 承接感情线
- 做对照组

### 2. 填基础信息
先写最基本的身份框架：
- 姓名
- 性别
- 出场年龄 / 年龄段
- 身份 / 职业 / 阵营

这些信息只负责回答“他是谁”，不负责回答“他为什么好写”。

### 3. 找一个原型人物
从动漫、影视、现实人物、熟人里找一个参照。

要求：
- 只借气质、反应方式和说话感觉
- **不要**照抄原型的完整剧情
- 原型只是抓手，不是枷锁

写成：`整体感觉接近____，但在____上不同。`

### 4. 写 3—5 个关键词
关键词分两类：
- **性格关键词**：别扭、善良、控制欲强、嘴硬、护短、迟钝
- **剧情关键词**：容易被误会、适合写感情戏、关键时刻可靠、会引爆冲突

**关键词必须能长出具体行为。**

### 5. 追问关键经历
写一件或两件真正塑造角色的过去。

要回答：
- 这件事怎样塑造了当前性格
- 遇到什么情况会被触发

句式：`因为过去____，所以现在一遇到____就会____。`

### 6. 设计一个特殊事件
给角色安排一个能集中体现人设的桥段。

要求：
- 最好能同时证明 2 项以上特质
- 可以是已经发生过的代表性旧事
- 也可以是未来正文里一定要写的重要场面

特殊事件的作用：
- 把“标签”变成“会发生的事”
- 让作者知道这个人该怎么出手、怎么出丑、怎么动人

### 7. 把关键词改成表现
不要只写“他很别扭”“她很善良”，改成：
- 他说话会怎么拐弯
- 遇事会先做什么
- 被误会时会怎么反应
- 对亲近者和陌生人有什么差别

**尽量在正文里表现，不要直接用作者旁白把关键词说出来。**

### 8. 检查角色在故事里的效果
最后回答四问：
- 这种性格放进当前剧情，会制造什么结果
- 他最容易和谁起冲突或化学反应
- 哪个关键词最适合用场面来证明
- 删掉这个角色，主线会缺什么

## 输出模板
```markdown
## 人物设定卡
- 定位：
- 姓名：
- 性别：
- 出场年龄：
- 身份/职业/阵营：
- 原型人物：
- 性格关键词：
- 剧情关键词：
- 关键经历：
- 特殊事件：
- 表现方式：
- 在故事中的作用：
```

## 示例
### 输出
- 定位：主角的老师型配角，负责制造反差温情和关键指导
- 姓名：赵钱孙
- 性别：男
- 出场年龄：20
- 身份/职业/阵营：数学老师
- 原型人物：整体气质接近龟毛傲娇型，但更关心学生
- 性格关键词：别扭、善良、智商高、情商低
- 剧情关键词：容易被误会、适合温情戏、关键时刻会替人兜底
- 关键经历：年轻时被老师拉过一把，因此对差生格外恨铁不成钢
- 特殊事件：偷偷去家访替学生说好话，被撞见后死不承认
- 表现方式：嘴上刻薄，行动上补救；公开冷脸，私下替人兜底
- 在故事中的作用：既能制造误会，也能在关键处给主角支持

## 注意事项
- **不要**为了完整而填几百项无用信息
- **不要**只有标签，没有经历和行为
- **不要**把关键词直接塞进正文，当成作者旁白
- **不要**原型一拿来就整套照抄
- 轻量人设卡的重点不是“全”，而是**准、能写、能落到场面**

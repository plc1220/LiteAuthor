先定隐藏信息，再倒推该用什么细节让读者自己拼出来。**不是所有伏笔都要等大反转才回收。很多好用的伏笔，其实是把人物来历、关系、动机、旧事和未明说的原因，先用一两个细节悄悄放出去。**

## 适用情况
- 想让一句话、一个表情、一个小物件里含很多信息
- 不想直接讲人物背景和关系
- 想通过称呼、语气、熟悉度、偏好等细节暗示隐情
- 想让读者先猜到一半，再在后文得到证实
- 已经想好真相或隐情，但前文还缺“顺手”的铺垫

## 输入
- 后文要揭开的真相 / 隐情 / 关系
- 这次想先放出几成信息
- 涉及人物的真实关系、立场、信息差
- 伏笔第一次出现的场景
- 希望读者先产生什么判断或怀疑
- 计划何时回收

## 核心原则
1. **先定真相，再写细节。**
2. 细节伏笔第一次出现时，必须**先服务当前场景**，不能只为以后存在。
3. 让读者能察觉“不对”，但不要立刻把答案摊平。
4. 信息可以分次释放，不必一次说尽。
5. 最稳的细节伏笔，往往看起来像顺手一笔，回头才发现分量很重。

## 步骤
### 1. 先把隐藏信息写死
写清：
- 真相到底是什么
- 谁知道，谁不知道
- 这条信息这次准备放出多少
- 这次放出去后，读者最该多知道哪一点

### 2. 先选“这次要露哪一层”
常见分层：
- 只让读者感觉哪里不对
- 让读者知道两人旧识，但还不知道旧到什么程度
- 让读者猜到动机方向，但不确认原因
- 让读者知道人物在隐瞒，但还不知道隐瞒什么

**别一次把谜底讲完。**

### 3. 从这些载体里挑一个最自然的细节
优先载体：
- **称呼**：不该知道名字却叫得太顺，关系远近和称呼不匹配
- **语气 / 停顿**：一句话里突然顿一下，或话说半截
- **表情 / 反应**：该平静时太敏感，该惊讶时不惊讶
- **熟悉度**：对某个地方、物件、规矩过分熟悉
- **偏好 / 习惯**：对某样东西有特别反应，暴露旧事或执念
- **顺手带出的信息**：一句看似平常的话，其实说明这个人知道得过多

### 4. 先给它找一个表面解释
让第一次阅读时，这个细节看起来说得过去：
- 像是口误
- 像是观察力强
- 像是巧合
- 像是情绪失控
- 像是习惯使然

没有表面解释，伏笔会太扎眼。

### 5. 让细节兼顾“当场功能”和“后续回收”
这笔细节第一次出现时，至少还要完成一项：
- 推动当前对话
- 增强当前气氛
- 立人物
- 放大关系张力
- 让某个场面更自然

**第一次只是伏笔，通常太硬；第一次先是戏，后面才会变证据。**

### 6. 信息多时，分 2—3 次放
同一条隐情可分次出现：
- 第一次：只露异常
- 第二次：露出更明确的熟悉感、偏好或立场
- 第三次：把前两次对上，完成回收

这样读者既能参与推理，也不会觉得作者在故弄玄虚。

### 7. 尽量让读者自己接上
优先用这些方式让读者推断：
- 表情和语气对不上台词内容
- 一句顺嘴话暴露关系深度
- 反复出现的小物件或小偏好带出旧事
- 不直接说原因，只给足够多的侧面证据

### 8. 回收时要对账
回收时别只报答案，要告诉读者：
- 当时为什么会这么称呼
- 为什么会有那个停顿或反应
- 为什么这个人会对某物那么熟
- 为什么那句平常话其实很不平常

## 输出格式
1. 要隐藏的信息
2. 这次准备先放出几成
3. 细节载体选择
4. 表面解释
5. 当场功能
6. 读者可能得到的判断
7. 后续还要补的两步
8. 最终如何回收

## 快速示例
### 输入
- 隐情：某配角以前侍候过关键人物，和主线旧案有关
- 本次目标：不明说，只让读者意识到她来历不普通

### 输出
- 载体：她顺嘴说出一句只有旧宫人或旧部才会知道的话
- 表面解释：像是见识广、消息灵
- 当场功能：让当前对话更有分量
- 读者判断：她不是普通侍女，过去可能和那位人物有关系
- 后续回收：再用表情、语气和旧物反应补两次，最后揭开真实来历

## 注意事项
- **禁止**只有答案，没有前文证据
- **禁止**为了隐藏把伏笔写到完全没痕迹
- **禁止**让角色为了埋伏笔突然失常
- **禁止**一次性放出过多解释，把读者思考空间全拿走
- 好用的细节伏笔不是“藏得深”，而是**第一次顺手，第二次可疑，第三次拍腿**。

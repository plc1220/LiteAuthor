先明确自己的具体问题，再用参考物拆“解决办法”，不要抄表面桥段。**参考物的真正价值，不是给同款道具和同款场面，而是帮忙看清：别人是怎么把同类问题处理顺的。**

## 适用情况
- 知道自己哪里有问题，却不知道怎么改
- 男主不够苏、反派太扁平、主角太强导致剧情走不动、感情线不甜、节奏发空
- 看完好文只会说“真厉害”，学不到可迁移的方法
- 想借鉴成熟作品，但又怕抄表面

## 输入
- 当前项目一句话
- 当前最想解决的**单一问题**
- 题材 / 频道 / 篇幅
- 可选参考物 1—3 个
- 当前文本或问题片段（可选）

## 核心原则
1. **先有问题，再找参考。** 不要漫无目的地扒文。
2. 参考物必须尽量同题材、同频道、同目标读者，完成度明显高于自己。
3. **只拆“问题对应的解决办法”，不复述整本剧情。**
4. **学机制，不学表面。** 把“紧箍咒”改名成别的手环，不叫学会；把它抽象成“与力量同步出现的克制/代价装置”，再写进自己的世界规则，才叫学会。
5. 参考物的意义在于**对比差异**。看清自己少了什么，比知道别人写了什么更重要。

## 步骤
### 1. 把问题缩成一句话
用最直白的话写：
- 我的问题是什么
- 读者现在会觉得哪里不对
- 我理想中的效果是什么

例：
- 男主配置不差，但像路人，没有苏感
- 反派会坏，但坏得很平，不招恨
- 主角太强，剧情容易一拳结束

**一次只拆一个问题。**

### 2. 选对参考物
优先选：
- 同频道、同题材、同受众
- 在这个问题上明显更成熟的作品
- 自己真的看过、记得住关键桥段的作品

避开：
- 题材和频道完全不对的样本
- 完成度和自己差不多、同样有问题的样本
- 只火在设定噱头、但正文也没解决你这个问题的样本

### 3. 只截取“问题相关段”
不要整本抄梗概。只整理和问题相关的这些信息：
- 角色 / 局面原始状态
- 问题第一次暴露在哪
- 作者用了什么办法处理
- 处理后读者得到什么感受
- 这套办法后面有没有重复兑现

### 4. 列出对方的解决链
按这个格式拆：
1. 先制造了什么局面
2. 放出了什么关键做法
3. 这个做法为什么有效
4. 后面怎么重复 / 升级 / 回收

可直接这样记：
`场面 → 做法 → 起效原因 → 后续兑现`

### 5. 总结骨干，不要停在表面
把参考物压成一句“底层机制”：
- 不是“写了某个法器”，而是“给强者一个克制和代价”
- 不是“后妈撕裙子”，而是“先伪善放松警惕，再一步步降待遇、加码打压”
- 不是“男主又帅又会打游戏”，而是“在对位主角低谷出场改局，并反复解决足够让人在乎的事”

**只有总结出骨干，参考物才开始变成自己的工具。**

### 6. 对照自己的缺口
至少回答：
- 我现在缺的是哪一步
- 我缺的是配置、出场时机、社会证明，还是递进和回收
- 我是不是只写了结果，没写解决过程
- 我是不是有同类桥段，但没有“足够让人在乎”的 stakes

### 7. 改写成自己的方案
把“骨干机制”翻译到自己的项目里：
- 放进自己的世界观、身份关系、题材基调
- 让手段贴合自己的角色动机
- 保留功能，不保留对方的具体道具、台词、桥段顺序

可用这个句式：
`我要学的不是____，我要学的是____；在我的文里，它会表现成____。`

### 8. 最后查抄袭风险
逐条检查：
- 关键道具是不是还是同类外观和功能
- 关键场面顺序是不是几乎没变
- 人物关系和冲突结构是不是过于一致
- 我改掉的只是名字，还是连机制落点都换过了

如果只是换皮，继续往上抽象一层再改。

## 输出格式
1. 问题一句话
2. 参考物名单
3. 每个参考物的解决链
4. 共同骨干机制
5. 自己当前缺口
6. 可迁移方案
7. 需要避开的表面相似点

## 快速示例
### 示例 1：主角太强，剧情走不动
- 参考结论：给能力加克制、代价、反制条件
- 错误学法：照搬别人的封印道具
- 正确学法：在自己的设定里写出与力量同步出现的诅咒、冷却、反噬或被针对机制

### 示例 2：恶毒女配不够饱满
- 参考结论：先伪善，再试探，再降待遇，再毁成果
- 错误学法：照搬别人具体宅斗桥段
- 正确学法：保留“程序性打压”这条链，让手段贴合自己的关系和背景

### 示例 3：男主不够苏
- 参考结论：让他在对位主角低谷时改局，并在多个场域反复解决问题
- 错误学法：只给高颜值和高配置
- 正确学法：补出场时机、社会证明、解决事情的频率和“恰到好处的撩”

## 注意事项
- **禁止**把参考物分析写成剧情摘要
- **禁止**一次同时研究五六个问题
- **禁止**把“换个名字”当成创新
- **禁止**只抄名场面，不拆它为什么有效

把同一时间内连续发生的几件事，统一挂在一个主体的感受和视线下去写。**动作想接顺，先别轮流切怪兽、女主、男主三个主体；先锁一个人，再让其他事件通过他的感官被读者接收。** 适合战斗、救场、追逐、突袭、摔倒等连续动作场面。

## 适用情况
- “这一瞬间发生了三件事”一写就乱
- 场面里人多、动作多、读者容易跟丢
- 总爱用“——”“然后”“与此同时”硬连动作，但画面还是碎
- 战斗、救场、受击、追逐写不顺

## 输入
- 场景概述
- 同一时段内发生的几件事
- 最适合承接镜头的主体
- 这一段最终要落到的结果

## 核心原则
1. **一个时间切片里，先只跟一个主体。**
2. 先写主体“感到什么”，再写主体“知道发生了什么”。
3. 其他人的动作，只能通过主体的视线、听觉、痛感、猜测进入。
4. 一个感知单元写完整后，再决定要不要切出去。
5. 能用主体体验串起来，就不要用清单式报动作。

## 步骤
### 1. 先选主体
优先选：
- 这一下最需要代入的人
- 受击者
- 被救者
- 做出关键判断的人

### 2. 按主体的真实接收顺序重排
常用顺序不是“客观时间顺序”，而是：
`突发刺激 → 身体反应 → 视线 / 听觉捕捉 → 迟到的认知 → 下一步动作`

### 3. 先写感觉，再写原因
例如不是：
- 怪兽撞了女主，女主飞出去

而是：
- 砰！女主身上一沉，整个人被狠狠掀了出去
- 四肢百骸一起疼起来
- 她勉强撑起眼皮，才看见对面蓄势待发的怪兽
- 原来刚才撞飞她的是它

### 4. 其他角色通过“被看见 / 被听见”进入
救场也别直接切到救人者视角，先让主体接收到：
- 没等到撞击
- 听见兵刃相击
- 睁眼看见有人挡在身前

### 5. 一个感知单元写完，再开下一单元
可按下面切：
- 受击单元
- 认知危险单元
- 等死 / 反击单元
- 救场揭示单元

每个单元内部只服务一个主体，不要来回摇镜头。

### 6. 必须切换时，先给边界
切出去前先完成当前主体的小段落，再明确：
- 谁抬头看见了谁
- 谁出声
- 场面位置怎么变了

## 反例
- 怪兽冲过来。
- 女主摔倒在地。
- 男主及时赶来救下女主。

问题：三个主体在同一拍里轮流占镜头，画面容易碎。

## 正写模板
1. 主体先受刺激 / 做本能反应
2. 主体感到什么
3. 主体看见或听见什么
4. 主体这时才明白发生了什么
5. 主体作出下一步反应

## 输出格式
1. 选定主体
2. 这一段的感知链
3. 改写后的场景
4. 若要切视角，最佳切点

## 快速示例
### 输入
- 怪兽冲来，女主摔倒，男主救场

### 输出
1. 主体：女主
2. 感知链：被撞飞 → 疼 → 看见怪兽 → 闭眼等死 → 没等到撞击 → 听见兵刃声 → 睁眼看见男主
3. 改写：
   - 砰！女主胸口像被重锤砸中，整个人高高抛了出去，重重摔在地上。
   - 剧痛一下子从四肢百骸漫上来。
   - 她勉强撑起身，视线发花，只看见前方那头怪兽再次低下头，正朝自己冲来。
   - 原来刚才撞飞她的就是这东西。
   - 她下意识闭紧眼，等着第二次撞击落下。
   - 可预料中的剧痛迟迟没来，耳边反倒炸开一声兵刃相击。
   - 她猛地睁眼，才看见一道身影横在自己面前。

## 注意事项
- **禁止**在一个时间切片里无标记连切多个主体
- **禁止**只会用“然后 / 与此同时 / ——”把动作串成清单
- **禁止**主体还没感知到，就先替主体宣布别人的完整心理和动作
- 动作接不顺时，先问：**这一拍到底该跟谁走**

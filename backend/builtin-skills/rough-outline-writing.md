先搭一份能保主线的粗纲，再把细节留给正文生长。**粗纲的任务是留住主线、理清逻辑、防止前后矛盾，不是提前把全文写一遍。**

## 适用情况
- 不列大纲就写飞
- 细纲一细就卡手
- 按章纲写不下去，临场发挥又容易崩
- 连载中需要根据反馈微调细节

## 输入
- 一句话题材
- 主角或 CP 人设
- 核心梗或主线目标
- 预计篇幅
- 当前最怕写崩的点

## 步骤
### 1. 先分阶段，不先分章节
把全文切成 3—5 个阶段即可。
常用分法：
- 懵懂期 / 成长期 / 收果期
- 没感情期 / 动感情期 / 确定关系期
- 起势 / 升温 / 爆发 / 回收

### 2. 给阶段之间放关键节点
每个阶段之间至少设 1 个会改变量的节点，例如：
- 表白
- 掉马
- 获奖
- 打脸
- 任务完成
- 旧事揭晓
- 关系破冰、决裂、和好

### 3. 每个阶段只写四样东西
1. 这阶段的主要目标
2. 大致走向
3. 重要配角和作用
4. 这一阶段结束后，什么变了
**只写功能，不写流水账。**

### 4. 先保主线，再放细节
对每个阶段都写一句：
`这一段的核心作用是让____。`
比如：
- 两人感情升温
- 主角拿到关键资源
- 世界规则第一次压到主角头上
如果一句话说不出来，说明这阶段很可能发散。

### 5. 区分不可变和可变
#### 不可变
- 主线方向
- 关键节点
- 角色核心选择
- 必须回收的伏笔

#### 可变
- 去哪里
- 用什么方式完成
- 具体互动
- 某一章拆成几章
**行为可以变，作用不能乱。**

### 6. 写偏时先看崩的是行为，还是主线
如果正文和大纲不一样，先别急着推翻。先检查：
- 这段虽然换了写法，是否仍然达成原定作用
- 感情、主线、逻辑有没有受损
- 只是场景换了，还是核心结果变了
如果只是行为变了，允许保留新写法。

### 7. 别把粗纲写成简略版正文
高危信号：
- 每一段都在记先做什么后做什么
- 细节写得太死
- 一动笔就发现正文不肯按纲走
粗纲只负责方向和节点，不负责替正文表演。

### 8. 连载时按反馈微调细节
允许根据读者反馈调整：
- 哪段放大
- 哪段压缩
- 哪个配角多一点或少一点
但**不要为了评论临时改掉主线承诺**。

## 模板
### 分阶段粗纲
- 阶段一：
  - 目标：
  - 走向：
  - 关键节点：
  - 配角作用：
  - 阶段结果：

## 输出格式
1. 分阶段清单
2. 阶段之间的关键节点
3. 每阶段的核心作用
4. 可变和不可变清单
5. 最容易写飞的位置
6. 微调建议

## 快速示例
### 输入
- 题材：校园甜文
- 主线：死对头从互看不顺眼到偷偷心动

### 输出
- 阶段一：互呛期。目标是立住关系张力；结果是两人被迫绑定。
- 阶段二：升温期。目标是让敌意里掺进在意；关键节点是共同背锅和第一次护短。
- 阶段三：确认期。目标是把嘴硬推到告白；关键节点是旧误会揭开。
- 可变：具体在哪一场活动升温。
- 不可变：两人从互呛到互护的主线方向。

## 注意事项
- **禁止**把大纲写成章节流水账
- **禁止**为了细节偏离，慌忙全盘推翻
- **禁止**正文一变，就以为大纲失效
- 粗纲越短，越要清楚这一段到底为什么存在

用“核心底色 + 套娃层次”设计反差人设。**好看的复杂人设，不是贴很多标签，而是让读者先误判、再理解、再心疼，并且每一层都能长出剧情。** 同一个缺点，既要能制造冲突，也要能翻出同源的优点和更深的伤口。

## 适用情况
- 角色只有一个标签，显得单薄
- 角色第一眼不讨喜，怕写成纯毒点
- 想写嫉妒、偏执、嘴毒、病娇、胜负欲强等高风险人设
- 想让人物通过“逐步了解”自然带出剧情和感情线
- 角色设定很多，但不知道哪些层次该先露、哪些后翻

## 核心原则
1. **所有层次都必须能被同一个核心底色解释。**
2. **缺点必须真实存在，不能硬洗白。**
3. **优点最好和缺点同源。** 例如：不安全感 → 细心；胜负欲 → 努力；嘴硬 → 自尊强。
4. **越不接地气的人设，越要落回普通读者能共鸣的情绪入口。**
5. **层次越多，越要考虑另一主角和主线能不能接住，避免头重脚轻。**

## 步骤

### 1. 先锁定核心底色
先写一句：
`这个人无论怎么翻，最不会变的底色是____。`

可用方向：
- 骄傲要强
- 缺爱敏感
- 克制守礼
- 护短重情
- 冷静利己
- 温柔但自卑

后面的表层标签、深层成因、极端反应，都要从这根上长出来。

### 2. 排出三层“套娃”
至少给角色排 3 层，不要一上来把所有解释全说完。

#### 第一层：第一眼标签
读者和其他角色最先看到的一层。
例如：
- 嫉妒心强
- 嘴巴刻薄
- 高冷难接近
- 过分争强好胜
- 软弱爱哭

这层的任务是**制造第一印象和最早冲突**。

#### 第二层：形成原因 + 同源优点
回答：
- TA 为什么会变成这样
- 这个缺点反过来长出了什么优点
- 这个优点能不能在关键场面里救人、救局、救关系

例如：
- 因为长期被比较，所以嫉妒、要强；也因此格外努力、自我要求高
- 因为害怕失控，所以嘴硬、刻薄；也因此做事细致、边界清楚

#### 第三层：更深的伤口 / 核心需求
继续追问：
- TA 最怕失去什么
- TA 最想证明什么
- TA 最深的匮乏是什么

常见方向：
- 缺乏安全感
- 害怕被抛下
- 过度渴望被认可
- 害怕自己不值得被爱
- 害怕一旦示弱就会失去位置

这一层决定角色为什么会反复在同一个问题上受伤，也决定后续成长线和感情线的根。

### 3. 每一层都拆成 A 面 / B 面
不要只写“这一层是什么”，要写它怎么让人不爽，又怎么让人心疼。

- **A 面：** 让人误判、反感、警惕的表现
- **B 面：** 让人理解、心软、被吸引的表现

示例：
- 第一层“嘴毒”  
  - A 面：说话刺人、容易得罪人  
  - B 面：只在私下发作，台面上仍守分寸
- 第二层“要强”  
  - A 面：见不得别人压自己一头  
  - B 面：愿意下苦功，不会把失败全怪别人
- 第三层“缺安全感”  
  - A 面：敏感、多疑、患得患失  
  - B 面：会格外留意细节，更容易看见别人受的委屈

**A 面负责冲突，B 面负责翻盘。两面都要真。**

### 4. 把层次落到具体行为
每一层都至少写 3 类行为：
- 台面上怎么表现
- 私下里怎么表现
- 压力来了怎么选

重点问：
- TA 是把情绪发向别人，还是发向自己
- TA 有没有底线
- TA 在最难看的时候还守不守住某种体面或公正

**同样是嫉妒，有人会害人，有人只会自己较劲。边界一变，读者观感就完全不同。**

### 5. 用对照组把层次打亮
给角色安排一个能放大 TA 问题的对照组：
- 更耀眼的人
- 更不公平的优势拥有者
- 更会表演的人
- 更符合世俗标准的人

然后问：
- 对照组会逼出 TA 哪个缺点
- 对照组又会反衬出 TA 哪个优点
- 如果没有这个对照组，TA 的层次还亮不亮

**很多“讨厌型”人设之所以能成立，不是因为 TA 本身完美，而是因为对照一摆出来，读者会看见 TA 的委屈、努力、底线和公正。**

### 6. 排“掀套娃”顺序
不要一次性解释完。按“先误判 → 再改观 → 再心疼”排场景。

至少回答：
1. 读者先看到哪一层
2. 另一主角或关键配角先误判什么
3. 哪件事让对方第一次改观
4. 哪件事揭开更深一层
5. 最终哪一场戏让前面的理解全部升级

可直接套用的顺序：
- **先给小缺点**
- **再露底线**
- **再露努力或代价**
- **最后翻出最深伤口**

### 7. 让层次直接长出剧情
每一层都问 2 句：
- 这一层会制造什么冲突
- 这一层会推动哪条线

常见长法：
- 第一层缺点 → 制造误会、对立、第一波不赞同
- 第二层成因 → 推出主线困局、竞争线、对照线
- 第三层伤口 → 推动感情线、成长线、关键选择

如果一个层次只存在于设定表里，长不出事件和选择，就删。

### 8. 用“评价翻转”回收层次
复杂人设最好配一条清晰的评价翻转链：
`看不顺眼 → 不赞同 → 理解 → 心疼 → 偏爱 / 信任 / 站队`

注意：
- 早期的不赞同最好是**温和、合理的误判**，不是无脑欺负
- 后期的改观必须来自**看见行为和代价**，不是作者替角色喊冤
- 翻转后要有实际后果：站队、维护、合作、心动、关系升级

### 9. 做毒点控制
高风险人设必须先排雷。

重点检查：
- 缺点是不是被写没了，最后只剩“被宠”
- 角色是不是只会作，不会承担后果
- 读者能不能理解 TA 的情绪入口
- 这个缺点有没有越过底线，导致再怎么解释都难救
- 更深一层是不是只是卖惨，没有影响选择

**怕读者讨厌就把缺点写淡，最后往往会写成空心角色。真正稳的写法，是保留缺点，但给出边界、成因和代价。**

### 10. 检查会不会头重脚轻
如果这个角色层次很多，再检查 3 件事：
- 另一主角是否有足够厚度接住 TA
- 主线是不是已经被这个人一个人抢完了
- 当前题材和篇幅能不能承载这么多层

扛不住时，优先：
1. 减一层
2. 缩对照组和支线
3. 把复杂人设放到“被逐步了解”的位置上，让剧情负责掀层

## 输出格式
1. 核心底色
2. 第一层标签
3. 第二层成因与同源优点
4. 第三层伤口 / 核心需求
5. 每层 A 面 / B 面
6. 台面上 / 私下 / 压力下的行为
7. 对照组设计
8. 掀套娃顺序
9. 评价翻转链
10. 由人设长出的剧情线、感情线、成长线
11. 毒点风险与控制办法
12. 是否会头重脚轻

## 快速示例
- 核心底色：缺爱又要强
- 第一层：嫉妒心强、说话带刺
- 第二层：长期被拿去和更耀眼的人比较，所以格外努力，也特别能吃苦
- 第三层：缺乏安全感，害怕自己一旦不够优秀就会被丢下
- 评价翻转：旁人先觉得 TA 小心眼 → 后来发现 TA 从不主动害人，只会自己较劲 → 再发现 TA 在关键处反而比更圆滑的人更公正、更细心 → 最后理解 TA 一直在拼命证明自己值得被留下

## 注意事项
- **禁止**把两个互相打架的标签硬缝成复杂
- **禁止**只有卖惨，没有真实缺点
- **禁止**只会旁白解释，不让角色做选择
- **禁止**把高风险缺点轻飘飘糊过去，最后写成只会被宠的纸片
- 真正好看的复杂人设，不是“设定很多”，而是**同一个人能一层层翻面，而且每一层都能推动故事**。

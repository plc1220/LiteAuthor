先把故事拆成三条对应的线，再写章纲。**情节不是想到哪写到哪，而是人物变化线和事件线一一咬合。三条线一旦对上，前后矛盾、感情突兀、时间乱跳会少很多。**

## 适用情况
- 感情线和剧情线总对不上
- 会写片段，不会排长期推进
- 前后情绪跳、关系跳、时间跳
- 做了大纲，落到章纲还是卡
- 想先稳住前20章

## 输入
- 一句话故事核
- 两个关键角色，或两条核心关系
- 当前起点状态
- 目标阶段结果
- 已知关键事件

## 步骤
### 1. 先写故事核
用最直白的话写：`谁从什么状态走到什么状态。`

不用追求好看，先追求自己看得懂。

### 2. 再写“这本书主要想表达什么”
只写1句，例如：
- 互相试探后建立信任
- 从利用到相爱
- 从师徒到对立
- 从盟友到生死绑定

### 3. 写 A 线：角色 A 的变化顺序
按节点列：初始态 → 第一次变化 → 第二次变化 → 阶段高潮态。

每个节点尽量写成“对角色 B / 对主线”的态度，而不是空泛情绪词。

### 4. 写 B 线：角色 B 的变化顺序
要求和 A 线能对照。追问：`A 变的这一格，B 对应变了什么？`

### 5. 写事件/时间线
把会触发变化的事件按时间排出来。每个时间节点都要回答：
- 发生了什么
- 这件事改了谁的态度
- 它怎样把两条人物线往下一格推

### 6. 做对应检查
删掉这类节点：
- 事件发生了，但谁都没变
- 人物突然变了，但没有触发事件
- 两条人物线推进速度完全失衡
- 时间跳了，关系却像没经过中间过程

### 7. 再拆前20章章纲
每章至少推进两样：
- 一小步人物线
- 一小步事件线

章纲模板：
1. 这章发生什么
2. A 线推进到哪
3. B 线推进到哪
4. 章尾留下什么期待

### 8. 写完20章后通读一次
重点查：
- 有没有细碎信息没交代
- 哪个伏笔可以提前回收
- 哪条线拖慢了整体节奏

再继续往后滚动写下一段章纲。

## 输出格式
1. 故事核
2. 主题一句话
3. A 线变化表
4. B 线变化表
5. 事件/时间线
6. 前20章章纲
7. 对应失衡点
8. 后续调整建议

## 快速示例
### 输入
- 宫廷感情线：女主入宫后与男主从互相利用到彼此试探

### 输出
- A 线：不屑 → 注意到对方 → 起疑 → 被吸引 → 动心
- B 线：利用 → 防备 → 谈条件 → 依赖 → 动心
- 事件线：宫宴初见 → 被调到身边 → 第一次合作 → 误会争吵 → 共同对敌
- 前20章：每章都对应一个关系变化点和一个事件推进点

## 注意事项
- **不要**先堆零散桥段，再反推人物变化
- **不要**只写一条人的情绪线，另一条全靠临场发挥
- **不要**事件很多，人物却原地不动
- 非言情也能用：把 A/B 线改成“主角内在线 / 主角外在线”，第三条仍是事件线

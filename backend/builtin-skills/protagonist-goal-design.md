先定最终目标，再倒推实现方法、阶段目标和几件必须发生的大事。**大情节不是写到哪儿算哪儿冒出来的，它通常要提前设计成阶段性高潮、拐点或反转。写正文时，当前情节始终朝最近的一件大事推进；分阶段时按目标的实现/失败切，不按地图切。**

## 适用情况
- 总在大情节卡壳
- 知道结局，不知道中间每一段为什么存在
- 副本、地图一直换，主线却没真正往前
- 中段温吞，人物长期没有明显收获或挫折
- 事件很多，但缺少阶段高潮和方向感

## 输入
- 主角最终目标
- 做不成的后果
- 实现这个目标的大致方法
- 已想到的大事、高潮或转折
- 预计篇幅

## 核心原则
1. **先定最终目的，再定实现方法，再拆阶段。**
2. 大目标要尽早让读者知道，并在后文持续作为潜在牵引线存在。
3. 每个阶段都要有清晰小目标；阶段结局必须是**实现、失败或失败后被迫转向**。
4. 大情节要提前列出来：反转点、加速点、重大选择点、阶段高潮。
5. 写正文时，当前段落都要服务于**最近的一件大事**。
6. **阶段按目标切，不按地图切。** 换地图只是结果，不是分段标准。
7. 如果大情节总卡，通常是因为：**大事没提前想清、前段铺垫不足、或节奏没踏准。**

## 步骤
### 1. 一句话钉死最终目标
写成：
`主角最终必须____，否则____。`

要求：
- 够具体
- 能对应结局
- 能解释全文为什么要继续写

### 2. 写清实现这个目标的主要路径
继续写：
`想做到这件事，主角至少要先____、再____、最后____。`

这一步是在找“实现方式”，不是在堆事件清单。

### 3. 先列 3—6 件必须发生的大事
优先选能改写局势的大事：
- 阶段性高潮
- 重大失败
- 关键反转
- 关系或阵营翻面
- 让故事突然加速的事件

每件大事都要回答：
- 它会改什么局
- 它会逼主角做什么选择
- 它把故事推向下一阶段的哪个方向

### 4. 给每件大事补前置条件
不要只知道“大事会发生”，还要知道：
- 发生前主角必须先拿到什么
- 前面需要埋哪些信息
- 需要哪些人物关系先到位
- 为什么偏偏此时发生最值钱

### 5. 按目标拆阶段，不按地图拆阶段
正确问法是：
- 这一阶段主角要完成什么小目标
- 完成后会得到什么资格 / 资源 / 认知
- 失败后会损失什么、被迫转去哪里

**不要**用“到了新地方”直接当新阶段。地图可以变，但阶段必须有目标差异。

### 6. 给每个阶段设置小目标和阶段结局
每阶段都写成：
`为了更接近最终目标，主角这一阶段必须先____。`

再补：
- 成功会怎样
- 失败会怎样
- 半成功半失败会怎样

这样阶段边界才清楚。

### 7. 当前剧情始终推向“最近的大事”
写每一段时只问一件事：
`这一段如何把人物、信息、关系或资源推向最近的一件大事？`

可推进的东西包括：
- 让主角更接近触发条件
- 让人物关系更适合爆炸
- 让误会更深 / 线索更全
- 让代价更高、倒计时更紧

### 8. 用“得失变化”检查节奏
如果一大段里都没有明显得失，通常就温吞了。
每个阶段至少要出现其一：
- 得到新资源
- 失去旧优势
- 认知被改写
- 阵营变化
- 地位变化
- 关系翻面

### 9. 卡壳时做倒查
如果卡在某个大情节，逐条反问：
- 这件大事到底要改写什么
- 前面还缺哪一个前置条件
- 现在的人物为什么必须走到这一步
- 是不是把“换地图”误当成“有推进”
- 当前阶段的小目标是否写清楚了

### 10. 反查最常见的跑偏点
- 只有最终结局，没有中途大事
- 阶段按地点切，导致一换地图就像另起炉灶
- 事件很多，但彼此没有因果
- 大事发生前缺铺垫，只能硬拐
- 很久没有明显收获或挫折，读者感觉在原地打转

## 输出格式
1. 最终目标
2. 做不成的后果
3. 实现路径
4. 3—6 件关键大事
5. 每件大事的前置条件
6. 阶段目标表
7. 每阶段的成功 / 失败结果
8. 当前最近的大事
9. 可能卡壳的地方

## 快速示例
### 输入
- 女主要功成名就

### 输出
- 最终目标：女主必须闯出名声，否则会一直被困在原来的位置
- 实现路径：先拿到入场资格 → 做出阶段成果 → 守住名声与资源 → 赢下关键竞争
- 关键大事：
  1. 拿到入场机会
  2. 第一次成果翻车
  3. 竞争对手反咬
  4. 阶段成果公开证明自己
  5. 最终定局
- 阶段切分：不是“去学校 / 去外地 / 去新公司”，而是“拿资格 / 保资格 / 出成果 / 守成果 / 赢终局”
- 当前写法：每一段都服务于最近的大事，而不是写到哪儿算哪儿

## 注意事项
- **不要**把阶段写成旅行日志或换地图记录
- **不要**只有大结局，没有中途反转和阶段回收
- **不要**让大事纯靠巧合触发
- **不要**让小目标彼此无关，做完就忘
- 好的大情节设计，不是事件越多越好，而是：**大目标清楚、大事提前想好、每一段都在推向下一次改局。**

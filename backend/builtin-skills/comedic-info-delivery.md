把必须交代的信息，塞进一个真的有笑点、但笑完会留下结果的场面里。先保信息，再做喜剧。

## 适用情况
- 纯交代信息太干
- 轻喜、都市、悬疑、恋爱文里需要边逗边推剧情
- 角色一说笑就把重点冲没
- 想写轻松场面，但又不能停主线

## 输入
- 本场必须让读者记住的信息
- 参与人物及关系
- 场面的表层气氛
- 本场之后要推进到哪里

## 核心原则
1. **先定信息，再包喜剧。**
2. **笑点必须长在信息上**，不是另起一段插科打诨。
3. **本场至少要有一个硬落点**：一句明确台词、一个关键动作、一个决定，把信息钉死。
4. **笑完要有后果**：行动、判断、关系变化，至少留一项。
5. **禁止**为了搞笑把人物写蠢。

## 步骤
### 1. 用一句话写死核心信息
填空：
`这一场就算只让读者记住一句，也必须记住：____。`

### 2. 给信息选一个喜剧外壳
优先从下面选 1 种：
- 误会
- 过度解读
- 人设反差
- 一本正经说怪话
- 第三人打岔
- 动作窘态

**只选一个主笑点。** 主笑点越单纯，信息越清楚。

### 3. 把场面拆成两层
- 表层：读者当下在笑什么
- 里层：读者实际接收了什么信息

如果这两层互不相干，重写。

### 4. 按“三拍”组织场面
1. **抛信息苗头**：先把核心问题说出来
2. **用喜剧反应放大**：误会、嘴炮、窘态围着它转
3. **用硬落点钉死**：回到正题，给出一句不能误读的信息

### 5. 给信息一次回弹
让另一角色做至少一种反应：
- 复述
- 质疑
- 利用
- 拒绝
- 顺着它做出下一步行动

这样信息才真正进入剧情，而不是只停在段子里。

### 6. 保持人物智商
喜剧成立的原因必须是：
- 信息差
- 性格反应
- 场合尴尬
- 语言双关

**不要**让角色突然听不懂人话。

### 7. 结尾一定接结果
本场最后至少落一项：
- 线索成立
- 新调查开始
- 关系推进/变僵
- 读者知道下一步要做什么

## 输出格式
1. 核心信息
2. 表层笑点
3. 里层信息
4. 三拍场面设计
5. 硬落点台词/动作
6. 笑后余波
7. 容易冲没重点的地方

## 示例
### 输入
- 必须交代：角色 A 一直梦见同一个男人，这不是偶然，是重要线索
- 气氛：轻松一点
- 参与人物：A 和嘴欠前辈

### 输出
- 表层笑点：前辈误会 A 总梦见男人是在纠结性向
- 里层信息：梦会重复，且对象固定
- 三拍：
  1. A 认真发问：“为什么我总梦见同一个人？”
  2. 前辈立刻歪楼，抓住“同一个男人”调侃
  3. A 打断笑闹，把重点掰回：“不是这个意思。我是说，梦境在重复，而且人也没变。”
- 硬落点：把“重复”和“固定对象”明确说出来
- 余波：前辈虽然嘴上继续损，但开始把这件事当成需要调查的异常

## 注意事项
- **不要**先写一大段笑料，最后才想起补信息
- **不要**让笑点比信息更复杂
- **不要**用连续插科打诨稀释重点
- **不要**让本场笑完什么都没变

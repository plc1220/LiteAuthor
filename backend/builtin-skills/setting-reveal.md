先区分“作者自己要知道的设定”与“读者此刻必须知道的设定”，再决定怎么交代。**复杂设定可以做表，但不要把设定表直接塞给读者。** 设定的目标不是显得宏大，而是让读者刚好看懂眼前这场戏，并愿意继续看下去。

## 适用情况
- 开篇总想先贴世界观、家族表、名词表
- 读者说看不懂，或反过来说一上来信息太多
- 机甲、星际、修真、系统、宫斗等题材设定很多
- 第一章总写成“原来这个世界是……”
- 有成熟题材基础，却又忍不住从头讲一遍

## 输入
- 当前场景目标
- 当前涉及的设定清单
- 读者眼下必须知道的因果
- 题材是否属于读者已熟悉的成熟类型
- 作者后台设定笔记（可选）

## 核心原则
1. **后台设定不等于前台交代。**
2. **禁止**在正文前直接贴设定表、家族谱、专有名词清单。
3. 当前冲突成立依赖的规则，必须讲明白。
4. 成熟题材先借读者公共认知，只补这篇文真正不同的部分。
5. 能放进场景里交代的，不要改成独立说明书。
6. 设定一旦出现，最好立刻在情节里演示后果。

## 步骤
### 1. 先分“作者知道”和“读者现在必须知道”
分别列两张清单：
- 作者后台要知道的：完整历史、完整势力图、全设定细节
- 读者当前必须知道的：这一场戏看不懂就会卡住的规则、身份、后果

**先保护读者当前理解，不急着展示作者准备了多少。**

### 2. 如果是成熟题材，先借公共认知
例如：
- 宫斗文默认有后宫等级、争宠、站队
- 机甲文默认有学院、机甲、等级差
- 修真文默认有境界、宗门、资源争夺

这时优先只交代：
- 这篇文的独特规则
- 会直接影响当前冲突的部分
- 主角和常规框架不一样的地方

不要把大家都懂的一整套背景再从头讲一遍。

### 3. 把设定分成三类
#### A. 现在必须直给
- 当前冲突成立依赖的规则
- 主角高光成立依赖的知识门槛
- 违反后会立刻出事的禁忌
- 当前人物身份差带来的直接后果

#### B. 先提名，后解释
- 后面马上会用到，但当前不必展开的势力、物件、历史
- 当前先让读者知道“有这么个东西”

#### C. 先别说
- 暂时不用也不影响理解的家族史
- 远期谜底
- 只会增加名词负担的细枝末节

### 4. 用“规则 → 场景 → 后果”交代必须信息
最稳的写法：
1. 先给一句规则
2. 立刻让人物碰到它
3. 明确违反或利用后的后果

例：
- 规则：未授权学员不能启动高阶机甲
- 场景：主角为救人强开机甲
- 后果：系统锁定、积分清零、可能退学

### 5. 禁止“文前列设定”，改成“文内慢慢交代”
高危写法：
- 正文前贴五大家族介绍
- 第一章先讲完整世界史
- 主角开口就是“原来这个世界是……”

替代写法：
- 让家族信息在宴席、争权、婚约、站队时顺手出现
- 让世界规则在第一次违规、第一次高光、第一次吃亏时出现
- 让身份差在他人反应和资源分配里直接显形

### 6. 每场最多抖 1—3 条关键设定
优先级只看当前戏：
1. 当前冲突
2. 当前身份关系
3. 当前必须知道的规则

超过 3 条，读者就容易从追剧情变成背名词。

### 7. 设定交代完，局势必须动
每次交代后至少发生一件事：
- 主角被逼做选择
- 局势升级
- 关系判断改变
- 爽点/危机落地
- 读者知道下一步会更麻烦或更精彩

如果交代完什么都没变，这段大概率该删或后置。

### 8. 给自己保留后台设定表，但别端给读者
后台设定表建议至少分：
- 势力
- 规则
- 历史节点
- 资源 / 道具
- 重要人物身份

这张表是作者写作工具，不是正文内容。只有真正上场的部分，才进入读者视野。

## 输出格式
1. 当前场景目标
2. 读者当前必须知道的设定
3. 作者后台设定 vs 当前前台设定
4. 必须直给 / 先提名 / 先别说
5. 采用的交代方式
6. 规则 → 场景 → 后果
7. 本场不该现在讲的设定
8. 抖完后局势如何推进

## 快速示例
### 输入
- 题材：架空宫斗
- 场景：女主第一次被推去侍寝

### 输出
- 必须直给：后宫等级、侍寝意味着什么、这次侍寝为什么危险
- 先提名后解释：皇后背后的母家、上一任宠妃旧事
- 先别说：整段朝堂斗争史
- 交代方式：嬷嬷训话 + 其他嫔妃反应 + 女主被迫做选择
- 效果：读者立刻知道这不是普通“被点名”，而是一次会改命的局

## 注意事项
- **不要**把作者笔记直接当正文
- **不要**因为怕说明味，就把关键规则藏没了
- **不要**在开篇就端上整套设定冷盘
- **不要**一场戏里让读者记十个新名词
- 好的设定交代不是“藏得巧”，而是**读者刚好能看懂，而且更想看下一场**

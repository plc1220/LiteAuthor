先把结局写清，再把终局拆成一串阶段目标，倒推每一级成立条件，最后再正向填充事件。**先反推，再正填；大目标难设计，就拆成眼前能写的小目标。**

## 适用情况
- 已有想写的结局或名场面，但中间剧情总发散
- 长文容易灌水，不知道每一段该往哪推进
- 感情文、升级文、中长篇都想让进度更稳
- 知道“最后要在一起 / 复仇成功 / 飞升登顶”，但不知道中间怎么排
- 长篇连载到中段容易疲软、卡文、烂尾
- 开文时只有几个名场面，常常边写边推后文

## 输入
- 结局一句话
- 主角开局状态
- 题材、篇幅、主要角色
- 已想到的名场面或关键桥段
- 想回收的主题、关系或反转

## 步骤

### 1. 先把结局写成可验证结果
至少写清 4 项：
- 主角最终得到 / 失去什么
- 主角最终变成什么样
- 关键关系最终变成什么样
- 外部局势最终变成什么样

不要只写“幸福了”“成长了”“赢了”。

### 2. 把结局拆成阶段目标
把终局往前拆成 4—8 个台阶，每一级都必须是**能被剧情验证的阶段结果**。

常用句式：
`终局 ← 上一步 ← 再上一步 ← 当前起点`

例如恋爱文：
- 结婚生子
- 结婚 / 决定共度一生
- 求婚 / 正式定下关系
- 稳定恋爱
- 表白成功
- 暧昧拉扯
- 逐渐熟悉
- 刚刚认识

例如升级文：
- 登顶
- 打赢最终对手
- 拿到核心资源
- 进入更大舞台
- 站稳当前层级
- 第一次证明自己

**把所有文临时当成“升级流”来拆，会更容易看清进度。**

### 3. 给每一级阶段目标补“为什么难”
对每一级都回答：
- 现在离这一级还差什么
- 谁或什么在阻拦
- 主角要付出什么
- 这一阶段最有趣、最有新意的事件可能是什么

大目标难设计，小目标相对可控。
例如“从朋友到暧昧”就比“写出一段伟大爱情”更好操作。

### 4. 从终局反推四类必要条件
结局要成立，前面必须具备什么？
- 身份 / 处境条件
- 能力 / 资源条件
- 认知 / 价值观条件
- 关系 / 站队条件

### 5. 给每个条件倒推前置事件
用固定句式写：
`要让____成立，前面必须先发生____，并让主角付出 / 获得____。`

前置事件常见包括：
- 初次相遇
- 拿到资格、门票或资源
- 关系破裂或重建
- 真相曝光
- 失去关键东西
- 一次关键失败
- 一个立场选择

### 6. 再按阶段目标正向填充
从开局往后写，不是直接奔结局，而是逐级推进：
1. 启动事件：主角为什么被推上这条线
2. 阶段目标 1：先拿到什么
3. 阶段目标 2：再失去或再升级什么
4. 阶段目标 3：局势怎么变大
5. 转折：原路线为什么走不通
6. 高潮：最后一级怎么兑现

每一级都写清：
- 阶段目标
- 触发事件
- 主要阻力
- 阶段结果
- 它如何顶出下一阶段

### 7. 反推角色为什么配得上这个结局
问自己：
- 能走到这个结局的人，开局底色应该是什么
- 哪些性格会帮助结局成立
- 哪些性格会阻碍结局成立
- 需要哪次打击、成长或认知变化，才能跨过去

**结局不是作者发奖状，必须是角色一步步走到的。**

### 8. 让所有重要情节都服务“阶段目标或终局”
给每段情节写一句功能定义：
- 铺条件
- 逼选择
- 改关系
- 增代价
- 埋伏笔
- 回收前文
- 推进到下一阶段目标

如果一段情节既不服务终局，也不服务任何阶段目标，优先删、并或后置。

### 9. 查情节之间有没有缺过渡
连着看关键节点，逐条问：
- 从 A 到 B 能说得通吗
- 某个转折是不是少了前置条件
- 某个行为是不是少了动机
- 某段关系变化是不是跳太快
- 这一阶段目标完成后，为什么会自然进入下一级

### 10. 最后做“删掉测试”
逐段检查：
`删掉这一段，终局或某一级阶段目标还能不能照样成立？`
- 能：说明这段可能无效
- 不能：继续问它到底支撑哪一层台阶

## 连载防崩补充
### 1. 先查是不是“写一截推一截”
出现以下情况，说明这本长篇高危：
- 开文时只有开头和几个名场面，结局自己也说不清
- 大纲只到当前章节，后面靠连载时临时推
- 之前几本都后期疲软、卡文、烂尾，常靠快速完结开新脱身
- 一到高潮附近就看不见接下来几章和最终收束

长篇连载前，至少补齐 3 样：
1. 结局一句话
2. 当前卷终点一句话
3. 从现在到卷终点的 3—5 个关键节点

**脑中有纲不等于有纲。能模糊想象，不等于能稳定连载。写下来。**

### 2. 高潮前必须知道高潮后怎么收
中期高潮最容易把作者推到墙角。写高潮前先答：
- 这场高潮之后，局势会变成什么
- 主角下一阶段要付什么代价
- 哪条线到这里阶段性结束，哪条线会被顶到更高一层
- 这个高潮怎样继续通向最终结局

如果答不上来，先别硬冲高潮。  
**不知道高潮后怎么收，中期最高点很容易直接变成崩点。**

### 3. 不要用“迅速完结并开新”解决结构病
快速开新可以暂时逃掉一本书的后期疲软，逃不掉写法本身的问题。  
如果一本又一本都靠“差不多就完”“下一本再写好”来止损，问题通常不是题材变了，而是终局和阶段台阶一直没钉牢。

### 4. 连载执行版最少要保住两层
- **长层**：结局、阶段目标、当前卷终点
- **短层**：从当前章节到卷终点最近的 3—5 个节点

长层负责不崩，短层负责不断更。  
两层都没有时，最容易出现“昨天还很顺，今天突然不会写了”的崩盘感。

## 输出格式
1. 结局定义
2. 阶段目标阶梯
3. 每一级的阻力与代价
4. 终局成立的必要条件表
5. 每项条件的前置事件
6. 正向阶段大纲
7. 需要补的过渡
8. 可删 / 可并 / 可后置情节
9. 终局回收点
10. 当前卷终点与连载防崩风险

## 快速示例
### 输入
- 结局：男女主结婚

### 输出
- 阶段目标阶梯：
  1. 刚认识
  2. 逐渐熟悉
  3. 暧昧
  4. 表白
  5. 恋爱稳定
  6. 求婚
  7. 结婚
- 阶段 2 的问题：怎么从“认识”走到“熟悉”？
  - 需要共同事件
  - 需要反复见面理由
  - 需要至少一次关系改观
- 阶段 3 的问题：怎么从“熟悉”走到“暧昧”？
  - 需要立场摇摆
  - 需要一次越界或误会
  - 需要比朋友更近、比恋人更远的互动
- 正向填充后，中段就不再是空白，而是一节节往上爬的过程

## 注意事项
- **不要**只有终局，没有阶段目标
- **不要**把阶段目标写成同义反复
- **不要**只反推结果，不正向填事件
- **不要**为了走流程，硬塞和阶段无关的桥段
- **不要**边写边推到连自己都不知道高潮后怎么收
- 长文最稳的做法，不是一直想“大结局多厉害”，而是**眼前这一级怎么有趣地跨过去**

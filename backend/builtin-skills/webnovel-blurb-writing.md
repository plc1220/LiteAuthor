先把文案写成一段“爽点片段”，不是剧情摘要。**读者看文案，不是来补课，而是来确认这篇文能不能给自己想要的一口。** 文案最稳的写法，是把主角当前处境、触发事件和最想看的回收，压成一个短而带画面的场景。

## 适用情况
- 文案像大纲摘要，信息多但不勾人
- 只会写设定，不会写场景和爽点
- 剧情流、感情流文案总写得一个样
- 文案前情过长，手机端前半段抓不住人
- 文案和正文调性不一致

## 核心原则
1. **文案卖的是一口场景化快感，不是完整介绍。**
2. **前 120 字先亮出最值钱的点。**
3. **先分清是剧情流还是感情流，再选模板。**
4. **结尾尽量停在爽点、关系点或悬念点上。**
5. **文案承诺的口味，正文前三章必须吃得到。**

## 步骤

### 1. 先写商品句
填空：
`这篇文最值钱的一口是____。`

例：
- 看一个被低估的人，怎么把评价权狠狠干回来
- 看两个关系本来不该失衡的人，怎么一步步真香

### 2. 先选主卖类型
- **剧情流**：主卖翻盘、逆袭、打脸、掉马、升级、事业高光
- **感情流**：主卖关系变化、拉扯、偏爱、追妻、真香、修罗场
- **梗 + 人设**：两种都有，但必须有主次

### 3. 按类型套模板
#### 剧情流模板
`主角原处境（普通/低谷） → 遭遇触发事件 → 局面发生变化 → 给出一个场景高潮 → 以爽点或悬念收尾`

常见写法：
- 先写主角之前多惨/多普通
- 再写什么事让她/他有了翻盘机会
- 最后停在“接下来她/他会变成什么样”的最爽一幕上

可选补强：
- 配角惊叹
- 炮灰破防
- 身份称呼变化
- 一个能让读者脑补后续的高光画面

#### 感情流模板
`主角与另一方的身份/旧关系 → 遭遇触发事件 → 关系开始失衡或转向 → 给出一个场景高潮 → 以感情爽点收尾`

常见写法：
- 先写两人原本是什么关系
- 再写什么事把两人推近、推远或推到失衡
- 最后停在追妻、嘴硬心软、掉马、心动暴露等最想看的那一刻

### 4. 把抽象设定改成连续画面
不要写：
- 重生
- 豪门
- 先婚后爱
- 追妻火葬场

要写成：
- 重生回被退婚前夜
- 嫁进侯府第一天就要和离
- 明明要离婚的人先把和离书撕了

**让读者看完能脑补剧情，不是只记住题材词。**

### 5. 结尾只留最值钱的一钩
优先停在：
- 打脸前一秒
- 关系失衡的一句台词
- 高光场面的结果
- 一个会逼读者点开的反问或后果

不要用平收句结尾，比如：
- “于是故事就这样开始了”
- “接下来她努力成长”
- “他们会发生怎样的故事呢”

### 6. 控制长度和语气
- 200—300 字最稳
- 轻松文可补 3—5 句小剧场
- 正剧别写成段子，沙雕文也别端成文言简介
- 文案语气尽量和正文一致

## 常用骨架

### 骨架 A：剧情流
`她原本____。`
`直到____发生，她被迫/决定____。`
`所有人都以为____，可她偏要____。`
`这一回，她要____。`

### 骨架 B：感情流
`他/她原本和____只是____。`
`直到____发生，两人的关系开始____。`
`一个想退，一个偏要逼近；一个以为只是利用，另一个却先失了控。`
`这一次，谁先认输还不一定。`

### 骨架 C：剧情+感情双卖
先用 2 句讲外部处境，再用 1—2 句把关系推歪，最后用一句爽点/关系点收尾。

## 输出格式
1. 核心卖点一句话
2. 主卖类型：剧情流 / 感情流 / 双卖
3. 主角原处境
4. 触发事件
5. 局面变化
6. 场景高潮
7. 收尾钩子
8. 标准版文案
9. 小剧场版（可选）
10. 建议删掉的信息

## 快速示例
### 输入
先婚后爱古言

### 输出
- 核心卖点：看一个一心想和离的女主，怎么把嘴硬男主逼到先低头
- 主卖类型：感情流
- 标准版文案：
  嫁进侯府第一天，她想的就是拿到和离书。所有人都当她软弱可欺，只有她自己知道，这一回她不是来忍的。可她刚把后宅搅明白，那个原本对她冷眼相看的男人，却先一步挡在她身前不肯放人。  
  “侯爷不是最想我走？”  
  他盯着她手里的和离书，半晌才道：“以前是。”  
  现在，他只想把它撕了。

## 注意事项
- **不要**把文案写成设定说明书
- **不要**只讲前情，不给场景
- **不要**把最值钱的一口藏到最后半句
- **不要**文案卖一套，正文前三章写另一套
- 好文案的标准不是信息全，而是**让对味读者马上想点开**

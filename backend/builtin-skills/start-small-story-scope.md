先把故事缩到能写清的那一层，再往外扩。**写不清一个房间、一个家庭、一个项目、一个小镇时，直接上王朝、天下、宇宙，最后很容易只剩空设定和烂尾风险。小而精，通常比大而广更稳。**

## 适用情况
- 设定越写越大，后面越来越难收
- 世界观很宏大，但正文落不到人和细节上
- 什么都想写，结果不知道先写哪条线
- 题材很抽象，缺乏生活感和落脚点
- 开篇就想把整个世界、所有势力、全部主题铺满

## 输入
- 题材与一句话故事
- 当前已经想到的世界观 / 势力 / 地图 / 主题
- 主角前期真正活动的范围
- 预计篇幅
- 作者最熟、最能写真的生活经验或知识范围

## 核心原则
1. **先小后大，先具体后抽象。**
2. 先找到一个能被看见、能被生活细节撑住的“岛”。
3. 更大的设定，必须由主线需要来决定，不由“看起来厉害”来决定。
4. 大世界也要通过小场域、小关系、小冲突让读者摸到。
5. **小岛都描述不清，就别急着画大地图。**

## 步骤
### 1. 找出当前最清晰的“小岛”
先回答：
- 主角前 3—10 章主要在哪活动
- 主角最先要解决的麻烦发生在哪
- 哪一层关系最先真正动起来

常见小岛：
- 一个家
- 一间公司 / 门派 / 班级 / 战队
- 一条街 / 一个村 / 一座城
- 一次项目 / 一桩案子 / 一场比赛
- 一个具体关系网

### 2. 把抽象命题改写成具体落点
如果当前设想太大、太空，先换成更能写的版本：
- “国家像孤岛” → 先写一座边城、一个驻点、一个家族
- “每个人的心都是孤岛” → 先写一个房间、一扇门、一次无法沟通的饭桌
- “天下大乱” → 先写主角所在那一小块地方是怎么乱到他头上的

### 3. 给“小岛”补 5—10 个可落笔细节
要求都能直接写进正文：
- 日常规则
- 人际关系
- 典型物件
- 本地习惯
- 压力来源
- 资源稀缺点
- 能立刻制造冲突的小事

**细节越能落地，世界才越像真的。**

### 4. 只保留当前阶段真正需要的大设定
把想写的大内容分成两类：
- **现在必须知道**：不写读者看不懂当前冲突
- **以后再开**：现在知道了也不影响主线推进

后者一律后置。不要在主角还没走出第一小岛时，就把全世界地图摊开。

### 5. 画“规模梯子”
按从近到远列 3 层：
1. 核心小岛：当前必须写清的范围
2. 下一圈：主线推进后会碰到的更大范围
3. 最外圈：只在后期需要时再展开的宏观层

每一层都写明：
- 何时打开
- 打开是为了什么
- 它会改变主角哪一个选择或局势

### 6. 检查大设定有没有真正压到主角头上
每条外扩设定都问：
- 它在近期章节里，会不会直接影响主角选择
- 它是不是当前矛盾的来源、障碍或后果
- 删掉它，现阶段剧情会不会更集中

如果答案大多是否，就先别写。

### 7. 用“小处见大”代替“直接讲大”
更稳的写法：
- 通过一个部门、一个家庭、一个帮派写整个体系
- 通过一个日常规矩写出世界秩序
- 通过一件小事写出大环境的压迫或风气

让读者先摸到一小块真实，再自然相信更大的世界。

### 8. 预防烂尾：每次扩张都必须回到主线
故事规模变大时，必须说明：
- 这次扩张让主角遇到什么新问题
- 它怎么抬高原本的主矛盾
- 它会不会顺带吃掉太多无关篇幅

只负责“看起来更厉害”的扩张，优先砍掉。

## 输出格式
1. 当前故事的一句话重心
2. 当前最清晰的小岛
3. 小岛细节清单
4. 现在必须知道 / 可以后置的大设定
5. 规模梯子
6. 应删或应后置的外扩内容
7. 更稳的开篇范围建议

## 快速示例
### 示例 1：玄幻升级文
- 原设想：一开头铺满大陆格局、宗门历史、诸天战力
- 调整：先收在主角所在的小城、家族和入门考核
- 原因：这些内容已经足够承载前期羞辱、资源争夺和第一次翻盘

### 示例 2：现实题材
- 原设想：直接谈时代、阶层、社会隔阂
- 调整：先写一个家、一间房、一次饭桌冲突
- 原因：抽象主题先落到具体关系上，读者才摸得到

## 注意事项
- **不要**把“规模大”误当成“格局大”
- **不要**开篇先端地图、设定、历史年表
- **不要**还没写清一个具体场域，就急着引爆全世界
- **不要**因为舍不得设定，把每一层都提前亮出来
- 真正稳的宏大，不是写得更广，而是**每往外扩一层，里面都站得住人和细节**。

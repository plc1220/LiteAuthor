先让读者跟着主角进局，再考虑给谁镜头。**前三章最重要的不是展示世界有多大、CP 有多神、配角有多复杂，而是让读者先记住主角、困境、目标和下一步期待。** 对大多数作者，尤其是新人，前三章尽量稳住主角视角。

## 适用情况
- 开头总想切主角、CP、反派、配角多条线
- 第一章刚把主角写出点感觉，就突然换场景换人
- 出场人物一多，读者就认不清谁是谁
- 想保住 CP 的神秘感、苏感、逼格
- 想让“主角不知道的事”自然变成悬念
- 想借别人眼睛写主角出场，但怕算乱切视角

## 核心原则
1. **前三章先让读者跟主角待够。**
2. **主角不知道的事，往往就是最好用的谜题。**
3. **重要角色的内心越早摊开，神秘感越快掉。**
4. 对新人来说，**字数≈印象深度**。先把主角写丰满，再谈多视角。
5. 借别人的眼睛拍主角可以，**但镜头必须仍然在拍主角**。

## 输入
- 前三章梗概
- 主角当前困境
- 主角当前最明确的目标
- 开篇计划出场的人物名单
- 读者现在必须知道 / 可以后置的信息
- 是否有 CP、反派或重要配角想提前亮相

## 步骤
### 1. 先定前三章唯一主跟随对象
填一句：
`前三章，读者主要跟着____进入故事。`

默认答案优先是主角。  
如果不是主角，必须说明理由：这一开场为什么不用主角，反而更能立住主线和期待。

### 2. 第一章先交付四件事
第一章优先让读者快速看清：
1. 主角现在在哪种处境里
2. 主角想先解决什么
3. 眼前最大的阻碍是什么
4. 这件事为什么值得继续看

**先让主角动起来，再补背景。**  
不要刚把主角写醒，就立刻切去写别人的夜店、会议室、回忆录。

### 3. 把开篇人物分成“必须上桌”和“可以后置”
只让会立刻影响主角当前选择的人先吃到完整出场。

- **必须上桌**：此刻能给主角压力、帮助、误导、机会的人
- **可以后置**：暂时只负责帅、苏、神秘、设定量很大的人

同一章里，尽量不要同时给太多人：
- 姓名
- 身份
- 关系
- 立场
- 功能
- 独立心路

读者不是在背人物表，而是在跟主角过第一关。

### 4. 把主角“不知道”的东西整理成钩子
列两栏：
- **主角已知**
- **主角未知**

把“未知”里最勾人的 1—3 项，直接当作前三章的悬念来源：
- 谁陷害了主角
- 这段关系到底是真是假
- 对方为什么这么做
- 眼前异常背后藏着什么

**不要急着替主角回答。**  
读者跟着主角查、猜、误会、推翻判断，本身就是期待感。

### 5. 每次想切 CP / 反派视角时，先过三问
#### 问 1：主角这边写够了吗？
如果主角刚立住一点印象，还没让读者沉进去，先别切。

#### 问 2：这次切视角是在加钩子，还是在泄底？
- 展示主角看不到的关键行动 → 可能值得切
- 单纯说明“CP 其实很喜欢主角”“反派其实另有苦衷” → 多半在泄底

#### 问 3：这次切过去，会不会多出一堆新名字和新场景？
如果会，让读者刚建立的代入断掉，这次就不值。

### 6. 前期尽量别写 CP 的完整心路历程
前期更稳的写法是：
- 只写 CP 的行为、停顿、语气、选择
- 让主角和读者一起猜
- 把“他为什么这样”留成关系悬念

适合延后到中后期再开的内容：
- CP 的脆弱
- CP 的自卑
- CP 什么时候动心
- CP 长篇内心剖白

**重要角色一旦被完整剖开，就不再神秘。**  
前期先保住距离感和期待，后期再回收，会更有力。

### 7. 想借别人眼睛写主角时，用“低信息镜头”
可用写法：
- 路人
- 一次性侍应生
- 守门人
- 暂时不会抢戏的小配角

用他们做 1—2 段镜头，专门拍主角的：
- 外貌
- 气场
- 动作
- 第一印象

然后立刻把镜头拉回主角。

**尽量不要**让重要配角或主角 CP 在开头承担这个功能。  
因为读者会立刻分心去想：
- 这个人是谁
- 他为什么这么看主角
- 他是不是喜欢主角
- 他后面要干什么

这叫信息分散，不叫高级。

### 8. 按“三章节拍”检查镜头是否稳
- **第一章**：主角入局，困境明确，抛出第一个问题
- **第二章**：主角继续处理，问题加压，未知扩大
- **第三章**：主角做出第一次有效应对或关键选择，并留下更大的钩子

这三章里，默认都以主角体验为主。  
如果必须切视角，**一章尽量只切一次，而且切完要能明显改变读者理解。**

### 9. 最后做开篇排雷
逐条检查：
- 读者前三章最先记住的是不是主角
- 主角的困境和目标是否清楚
- 有没有刚沉浸就被新场景打断
- 有没有同章塞进太多有名有姓的人
- 有没有过早摊开 CP / 反派的心路
- 主角不知道的事有没有被保留成谜题
- 借别人写主角时，镜头有没有跑去写观察者自己

## 输出格式
1. 前三章主跟随对象
2. 第一章四件事
3. 必须上桌人物 / 可后置人物
4. 主角已知 / 未知信息
5. 是否需要切视角
6. 若切，切给谁、为了解答什么
7. CP 内心应后置到哪里
8. 开篇混乱风险

## 快速示例
### 输入
- 开篇：主角穿来后发现原身自杀未遂，被经纪人辱骂
- 作者想在第一章后半切去写大佬 CP 在夜店里想“包养谁”

### 输出
- 主跟随对象：主角
- 第一章先交付：醒来 → 发现处境 → 找线索 → 知道自己被陷害 → 决定先自救
- 不建议立刻切 CP 视角
- 原因：主角线刚建立代入，切去新场景会打断进入情境；CP 的“有兴趣”也属于过早泄底
- 替代写法：先让 CP 只以电话、试镜名额、传闻或一次现实对撞的方式进入主角线，等关系积累后再开 CP 心路

## 注意事项
- **这不是绝对铁律**。老作者如果能在极短篇幅里立住角色、场景和情绪，可以破。
- 但对大多数作者，尤其是新人，**前三章先稳主角视角，几乎永远比乱切更安全。**
- **不要**把“第三人称”误写成“全知旁白四处飞”。
- **不要**为了显得结构复杂，牺牲最宝贵的开篇代入感。

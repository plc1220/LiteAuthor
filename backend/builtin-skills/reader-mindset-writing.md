先用作者思维把故事想透，再用读者思维决定哪些东西真的要写上台面。**不是放弃逻辑，而是把逻辑留在后台，把读者当下最想看的信息和情绪放到前台。**

## 适用情况
- 写着写着出现大量无效信息
- 害怕不解释就有 bug，于是把设定、动机、心理写太满
- 章节慢热、没劲、过于干巴或信息量失控
- 不知道该详写还是一笔带过
- 爽点或甜点前堆了太多说明，回收又太快

## 输入
- 当前章节、场景或细纲
- 本章必须推进的剧情 / 感情
- 本章最该让读者吃到的感受
- 作者担心不写会出 bug 的信息
- 角色的深层动机 / 背景设定

## 步骤
### 1. 先写“这章最值钱的一口”
填空：
`读者看这章，最该被满足的是____。`

可填：
- 爽点
- 甜点
- 压迫感
- 悬念
- 反转
- 情绪回响

**一章可以有多个功能，但主味最好只有一个。**

### 2. 把所有信息分成三层
#### A. 台前必须写
不写就会出现这三种问题之一：
- 看不懂
- 不动情
- 不觉得爽 / 紧张 / 有吸引力

#### B. 后台支撑
作者必须知道，但正文只需点到为止。
包括：
- 深层身世
- 完整心理链
- 世界观底层逻辑
- 配角完整前史

#### C. 可删信息
只是在解释、证明作者想过、展示设定完整度，**但不推动当前阅读体验**。

### 3. 先保逻辑，再缩上台量
对每条后台信息问 3 句：
- 这条现在不写，读者还能跟上吗
- 这条现在写出来，会不会拖慢当前最值钱的一口
- 这条能不能换成一个动作、一个反应、一个细节暗埋

能暗埋，就不要整段解释。

### 4. 用“读者入口”重排顺序
优先顺序通常是：
1. 小钩子 / 小异常
2. 当前冲突或期待
3. 必要铺垫
4. 真正想看的场面
5. 余波 / 下一章钩子

**不要一上来先把来龙去脉全讲清。**

### 5. 详写什么，略写什么
#### 详写
- 主角做选择的瞬间
- 爽点、甜点、反转前后的关键动作
- 关系变化真正落地的场面
- 会被读者反复记住的名场面

#### 略写
- 纯背景说明
- 重复证明同一性格的段落
- 作者自己很在意、读者当下未必想看的完整逻辑链
- 不改写局势的日常过程

### 6. 不要把“读者思维”误解成只要高潮
正确写法不是“把所有铺垫都砍掉”，而是：
- 先让读者看到值得追的钩子
- 再补最短路径的必要铺垫
- 把回收写值钱

如果每章都硬砸爽梗、甜梗，没有蓄力，读者一样会麻。

### 7. 检查本章有没有无效信息
逐段问：
- 这段推进了剧情、关系、情绪、期待中的哪一项
- 删成一句会不会更好
- 这段是不是只在解释人物为什么会这样，而不是让读者直接看到“他就是这样”

### 8. 作者版转读者版改法
先做两版：
- **作者版**：把因果、设定、动机想全
- **读者版**：只把当前最该看到的部分写上台面

然后输出 4 栏：
1. 必须保留
2. 压缩成一句
3. 改成动作 / 对话 / 细节暗示
4. 后置到更有戏的章节

### 9. 常见失误修正
- 男主太惨：不要先写满全部苦难史，先写最能让读者入戏的当下处境
- 人设太深：不要把所有人格剖面一次讲完，先让读者看到最带感的表现
- 设定太全：不要像给汉堡念配料表，先让人吃到那口味道
- 节奏太快太干：不是强行加说明，而是给当前事件加转折、反应和回响

## 输出格式
1. 这章最值钱的一口
2. 台前必须写
3. 后台支撑
4. 可删信息
5. 建议的出场顺序
6. 详写 / 略写分配
7. 需要改成暗埋的内容
8. 章尾钩子建议

## 快速示例
### 输入
- 情节：女主在男主最低谷时救了他，作者想把男主全部悲惨身世和心理创伤一次写清

### 输出
- 最值钱的一口：救赎感 + 关键相遇
- 台前必须写：男主当下有多狼狈、女主做了什么、这次援手为什么对他特别
- 后台支撑：父母问题、童年创伤、完整人格形成过程
- 改法：先写“他最黑的时候被她看见并拉了一把”，深层成因后面分批借事件和反应露出

## 注意事项
- **不要**把逻辑完整等同于全部写出来
- **不要**因为怕 bug 就把读者不想看的东西一次讲完
- **不要**把读者思维理解成只写高潮、不写铺垫
- **先让读者想看，再让读者看懂**

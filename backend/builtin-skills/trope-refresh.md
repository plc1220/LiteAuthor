把常见桥段改到“熟但不旧”。先保住读者真正想看的那口，再用反套路和微创新把实现方式、角色站位或过程拐半步。**优先做渐进式微创新，不要为了创新把爽点和逻辑一起拆没。**

## 适用情况
- 桥段很经典，但一写就像旧流程
- 作者对套路有排斥感，想反套路又怕写歪
- 只有设定新，事件还是老路数
- 开篇脑洞不错，中段开始全靠老梗平推

## 输入
- 基础套路一句话
- 想保住的核心快感
- 当前题材 / 人设 / 世界观
- 不能改掉的结果
- 作者最嫌弃这个桥段的点

## 步骤
### 1. 先写默认老梗版
按“通常会怎么发生 → 主角怎么赢 → 众人怎么反应”写出最常规版本。

### 2. 定核心不动点
写一句：
`这个桥段最值钱的不是表面流程，而是____。`

例如：
- 打脸：夺回评价权
- 拜师：主角得到跨阶段的助力
- 相遇：两个人的关系被强行接通
- 逆袭：从被压住到反压回去

**后面怎么改都行，核心快感不能丢。**

### 3. 先选创新强度
- **渐进式微创新**：读者还能认出熟悉口味，但具体展开新了
- **突破式创新**：连底层规则都改掉，读者几乎认不出原梗

**先从渐进式微创新开始。**
优先让“结果熟悉，过程新鲜”；不要一上来把所有规则掀翻。

### 4. 用四个角度翻新
#### A. 范式创新：换底层实现机制
问：
- 这件事非得按原来的世界规则发生吗
- 同样效果，能不能换一种更贴合题材的实现方式

常改的东西：
- 奇遇来源
- 力量获得方式
- 冲突触发规则
- 信息揭露方式

#### B. 产品创新：换主角真正拿到的东西
问：
- 主角得到的，非得是原版那件奖励吗
- 能不能换成另一种更有后续玩法的资源、能力或知识

常改的东西：
- 神功换成知识
- 宝物换成关系
- 传承换成规则漏洞
- 帮助换成工具或方法

#### C. 定位创新：换角色站位和读者预期
问：
- 谁是受益者，谁是旁观者，谁是高位者，能不能调换
- 如果读者以为主角是“被选中的人”，能不能让他先变成“成全别人的那个人”

常改的东西：
- 主角与配角功能
- 高人与徒弟关系
- 受害者 / 拯救者位置
- 谁才是场面里的真正主角

#### D. 流程创新：换达成结果的路径和顺序
问：
- 同样的结果，必须一步到位吗
- 能不能让事情多拐一道弯、先失败一次、先误判一次

常改的东西：
- 先后顺序
- 波折层数
- 误会与延迟
- 谁先出手、谁后补刀

### 5. 每个角度至少出 2 个版本
不要想到一个就停。
至少列：
- 最稳版
- 最怪但有戏版

再比较哪一版最适合当前书的题材和人物。

### 6. 选“最熟但最新”的版本
最后保留的版本要同时满足 4 条：
1. 读者能吃到原梗的快感
2. 展开方式比老梗多半步惊喜
3. 更贴人物，而不是硬反套路
4. 后续还能继续长，不是一锤子脑洞

### 7. 补足伏笔、代价和余波
创新版确定后，再补 3 件事：
- 前面要埋什么提示
- 主角要付什么代价
- 这一改法会带来什么后续新问题

**只有新设定，没有新后果，不算真的翻新。**

### 8. 避免假创新
出现以下情况就重做：
- 只是把名词换了，冲突没变
- 为了反套路，把读者最想看的回收删掉
- 所有人降智，只为成全“意外”
- 脑洞只新一章，后面接不上
- 创新点和人物、题材完全不匹配

## 输出格式
1. 基础套路
2. 默认老梗版
3. 核心不动点
4. 范式 / 产品 / 定位 / 流程四个角度的改法
5. 最终选择版
6. 需要补的伏笔
7. 代价与余波
8. 为什么这一版比老梗更值

## 快速示例
### 输入
- 基础套路：主角坠崖，遇高人，学成绝世神功
- 核心快感：主角在绝境中拿到跨阶段助力

### 输出
- 范式创新：不是真的坠崖遇人，而是在夺舍失败时，意识在体内遇见第二人格或残魂
- 产品创新：高人不教神功，改教来自未来的科学知识，主角开始用科学方法解决江湖问题
- 定位创新：主角跳崖没遇到高人，反而先救下另一个人，最后变成“主角以为自己找机缘，结果自己成了别人的高人”
- 流程创新：主角拿着线索反复试错，总差一步见不到高人，最后高人等得不耐烦，主动出来找人

## 注意事项
- **不要**把反套路理解成“什么都反着来”
- **不要**为了新而新，结果核心快感消失
- **不要**只会换设定名词，不会改冲突结构
- **先保住好看，再谈新**

先给这场戏排主功能，再把人设、关系、背景、冲突尽量压进同一句台词、同一个动作、同一个物件里。**一场戏不求功能越多越好，但至少别只干一件小事。**

## 适用情况
- 明明写了冲突，人物还是立不住
- 背景交代总要单独解释
- 人物出场很长，却没有记忆点
- 章节容易水，每场戏都像只推进半步
- 对白和动作很多，但信息密度低
- 关键长戏写得热闹，却没有真正改局势

## 输入
- 这场戏的一句话摘要
- 这场戏必须完成的主功能
- 希望顺手完成的副功能 1—3 项
- 参与人物的人设关键词
- 读者在这场戏后必须知道什么
- 下一场戏要接什么

## 核心原则
1. **一场戏先定 1 个主功能，再带 1—3 个副功能。**
2. **能靠台词、动作、物件、他人反应带出的信息，不要另起一段解释。**
3. 人设不要靠旁白夸，优先靠**说什么、怎么说、做什么、怎么做**。
4. 同一笔最好同时完成两件事以上。
5. **多功能不等于堆料。** 如果一句同时承担 5 件事却读不顺，就是失败。

## 常见可合并的功能
- 立人设 + 推关系
- 立人设 + 交代背景
- 推冲突 + 补设定
- 推情节 + 埋伏笔
- 交代处境 + 留下下一步期待

## 步骤
### 1. 先写主功能
用一句话填空：
`这场戏存在，是为了____。`

### 2. 再列副功能
从下面最多选 3 项：
- 人设
- 关系
- 背景 / 设定
- 冲突升级
- 伏笔
- 节奏转折

### 3. 给每项功能找最省字的载体
按优先级找：
- **一句台词**：身份、旧账、关系、立场
- **一个动作**：气场、态度、控制欲、轻慢、紧张、护短
- **一个物件 / 环境细节**：处境、职业、贫富、伤口、危险
- **一个他人反应**：威慑力、地位、魅力、压迫感

### 4. 用“台词 + 动作”做压缩
先各写一列：
- 这个人嘴上会怎么露底
- 这个人身体会怎么露底

优先组合成：
- 一句问话 + 一个轻慢动作
- 一句回击 + 一个站位变化
- 一句寒暄 + 一个不合礼数的细节
- 一句命令 + 一个别人立刻服从的反应

### 5. 删除重复解释
如果台词已经说明“熟社会”，动作已经说明“轻慢、不好惹”，就不要再补一句“他看起来很社会，很不好惹”。

### 6. 检查这场戏是不是至少改了两样东西
至少回答两项：
- 读者对人物的认知变了什么
- 场上关系变了什么
- 局势往下走了什么
如果只改了一样，而且改得很小，继续压缩。

### 7. 做过载排查
出现下面情况就减法：
- 每句都像作者在塞信息
- 人物说话不自然
- 一个动作被解释三遍
- 背景量大到抢走当前冲突

## 关键重场戏加码
如果这不是普通过渡戏，而是单元里的**重场戏**，再多查四项：
- 当前事件或案件有没有被真正改写结果
- 某个关键配角的定位有没有翻面：路人变伙伴、下属变盟友、怀疑者变认可者
- 主角的认知、信念或判断有没有前进一步
- 外部格局有没有变化：地位、站队、团队气氛、话语权

**重场戏最好一次改 3 项以上。** 如果只是场面大、台词多、吵得凶，但案子、关系、认知和格局都没动，这场戏就还不够重。

### 重场戏常见组合
- 事件解决 + 配角转正
- 事件解决 + 主角信念升级
- 事件解决 + 团队态度改观
- 公开对峙 + 新盟友形成 + 下一阶段目标明确

### 重场戏检查句
写完后补这句：
`如果删掉这场戏，哪几条线会一起塌掉？`

能答出越多，这场戏越值钱。

## 快速模板
- 表层事件：____
- 主功能：____
- 副功能：____ / ____ / ____
- 关键台词：____
- 关键动作：____
- 他人反应 / 环境细节：____
- 这场戏后读者会知道：____

## 示例
### 示例 1：一句话 + 一个动作立配角
目标：
- 立住“这人不好惹、很护短”
- 顺手交代他和姐夫的关系
- 补一个旧背景：之前有人来收保护费

写法：
- 台词：“姐夫，上次收你们保护费的那几个，还来吗？”
- 动作：他把姐夫让拿的绷带随手一丢。
- 回应：“那当然不敢了，都知道我是你姐夫。”

这三笔同时完成了：
- 人设：社会、轻慢、有压迫感、护短
- 关系：他护着姐夫
- 背景：以前有人来收保护费
- 气氛：这人一出场就带威慑

### 示例 2：差写法
- 先写他虎背熊腰
- 再旁白说他不好惹、很护短
- 再单独回忆上次有人来收保护费
- 最后再让他开口

问题：信息分散，字更多，力度还更弱。

### 示例 3：重场戏写法
目标：
- 解决当前主冲突
- 把原本不靠谱的同事写成正式伙伴
- 让主角从半信半疑变成真正确定自己的判断
- 顺手改掉团队对主角的看法

写法：
- 表层是一次公开问讯 / 对峙
- 中层让同事在关键处看出漏洞，第一次站到主角这边
- 主角顺势拿出前文准备的证据，彻底拿下事件结果
- 围观者态度同步变化，主角地位上升

这场戏值钱，不只是因为赢了，而是因为**事件、关系、认知、格局一起翻面**。

## 输出格式
1. 这场戏的主功能
2. 这场戏的副功能
3. 每项功能对应的载体
4. 压缩版场景 beats
5. 应删掉的重复解释
6. 这场戏后人物 / 关系 / 局势分别改变了什么
7. 如果是重场戏：本场同时改写了哪几条线

## 注意事项
- **不要**为了多功能，把人物写成信息播报器
- **不要**一场戏塞满设定却没有当下冲突
- **不要**每场都追求四五个功能，重要戏重一点，过渡戏轻一点
- **不要**把“吵得凶、说得多、场面大”误当成重场戏
- 好的一场戏多用，读者只会觉得“顺”，不会觉得“作者在安排”

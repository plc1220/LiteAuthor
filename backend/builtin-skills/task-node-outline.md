把长篇主线当成“送信”：从起点出发，经过若干驿站，最后把信送到终点。可以绕路，但不能流浪；可以写支线，但支线必须把信继续往前送。

## 适用情况
- 知道开篇和结局，不知道中间怎么走
- 无纲写到中后期容易崩
- 支线很多，写着写着偏离主线
- 每章都有内容，但整本书像在原地打转

## 输入
- 开篇状态
- 结局目标
- 主角的主线任务
- 预计篇幅
- 现有支线或灵感清单

## 步骤

### 1. 先定两个大支点
- **开篇**：世界背景、主角初始处境、故事起跑线
- **结局**：主角最终达成的地位、关系或结果

要求：
- 开篇和结局之间要有明显距离
- 结局必须是“送到哪儿”，不能只是抽象口号

### 2. 把中间拆成任务节点
把“从开篇到结局”拆成若干个必须到达的节点。

节点要写成可验证的任务，例如：
- 拿回家庭主动权
- 赚到第一桶金
- 结识关键盟友
- 通过关键考试
- 拿下第一次胜利
- 建立自己的团队

**节点不是主题口号，而是能完成 / 能失败的事。**

### 3. 给节点排顺序
逐个检查：
- 为什么必须先到这个节点
- 它怎么给下一个节点开路
- 删掉它后，主角还能不能顺利到结局

如果一个节点删掉后完全没影响，要么合并，要么删除。

### 4. 让每个节点都服务主线
每个节点都要回答：
- 它推进了主角哪一步成长、资源或处境变化
- 它怎么让主角更接近最终结局
- 它结束后，故事比之前多了什么新局面

### 5. 给每章也设起点和终点
每章至少写两句：
- 这章开头，主角在什么局面里
- 这章结尾，主角多完成了什么 / 失去了什么 / 知道了什么

**章节也是小驿站，不要写完一章还停在原地。**

### 6. 支线必须为节点服务
保留支线的标准至少满足一条：
- 帮主角完成当前任务
- 提前埋下后面节点需要的人 / 物 / 信息
- 展示完成任务的代价
- 放大主线结果
- 把主线冲突推得更难

不满足，就删、并或后置。

### 7. 允许绕路，不允许流浪
可以变化的是：
- 去哪个场景完成任务
- 用什么桥段完成任务
- 由谁来提供帮助或阻碍

不能变化的是：
- 主线方向
- 关键节点
- 最终送达的终点

判断标准：**这段写完后，主角有没有更接近下一个节点。**

### 8. 把支出去的“鱼刺”收回来
如果有配角线、小事件、小副本，后面必须回收到主线：
- 改变主角选择
- 提供关键帮助或阻碍
- 在高潮处回收前面的因果

不回收的鱼刺越多，正文越散。

### 9. 卡文时倒查最近的节点
当不知道下一章写什么时，先问：
- 现在离哪个节点最近
- 要到那个节点还缺什么条件
- 下一章该补条件、加阻碍，还是直接到站

## 输出格式
1. 开篇支点
2. 结局支点
3. 任务节点表
4. 节点之间的因果链
5. 本卷 / 本章起点与终点
6. 必留支线与作用
7. 可删除或后置的支线
8. 当前最近的下一个节点

## 示例
### 输入
- 开篇：女主重生回童年，家里一地鸡毛
- 结局：女主成为上市公司 CEO

### 输出
- 开篇支点：生活艰难，遍地极品，主角重新开始
- 结局支点：公司融资上市，主角获得事业和人生主动权
- 任务节点：
  1. 先拿回家庭主动权
  2. 赚到第一桶金
  3. 结识关键贵人
  4. 考上目标学校
  5. 创办工作室
  6. 公司融资上市
- 章节原则：每章都要回答“这封信今天往前送了没有”

## 注意事项
- **禁止**只有开篇和结局，没有中间驿站
- **禁止**支线写得热闹，却不帮主线送信
- **禁止**每章都有事发生，但整本书不前进
- **禁止**把节点写成模糊口号，比如“慢慢成长”“逐渐变强”
- 长篇最怕的不是绕路，而是忘了自己要把信送到哪里

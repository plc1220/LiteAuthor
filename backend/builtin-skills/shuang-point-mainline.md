先定全文到底爽在哪一件事上，再让所有桥段都往这一个落点回收。**不是每章都要写同一种情节，而是每章的爽感最后都要让同一条主线更强。** 单章都爽，整体却散，问题通常不在桥段，而在落点不统一。

## 适用情况
- 每个桥段单看都爽，连起来卖点不明
- 经营文、娱乐圈、升级文越写越散
- 一会儿爽生意，一会儿爽武力，一会儿又爽粉丝，方向乱跳
- 读者觉得热闹，但抓不住核心

## 写法步骤
1. 用一句话钉死主线落点：`这篇文最主要的爽点，最后要落在____上。`
   常见落点：店越开越大、实力越来越强、粉丝越来越多、地位越来越高。**只选 1 个主落点。**
2. 给主线选一个可见指标。生意看客流、营业额、分店；娱乐圈看粉丝、热度、资源；升级文看境界、排名、权限。指标越清，爽点越稳。
3. 把每个事件改写成 `事件本身 → 直接爽感 → 主线增量`。
   - 赶走混混 → 打脸解气 → 顾客更敢上门，店面客流上涨
   - 业务能力爆发 → 舞台很爽 → 热搜、资源、涨粉
4. 辅助爽点必须能**转化**。如果主线是粉丝增长，业务能力的爽要转成口碑和吸粉；如果主线是生意扩张，人情和打脸的爽要转成渠道、客流、信誉。
5. 每次高潮最好回收两层：当场解气 + 主线真正抬高。只有前者，没有后者，桥段就只是热闹。
6. 允许场景、对手、阻力变化，但**不允许主线落点乱跳**。可以今天靠产品赢、明天靠人脉赢、后天靠口碑赢，只要最后都让同一主线变强。
7. 每场写完做落点检查：
   - 这场赢，主线具体涨了什么？
   - 删掉这场，主线会不会几乎不受影响？
   - 这场是不是只让主角爽了一下，却没留下后果？

## 输出格式
1. 一句话卖点
2. 主线落点
3. 可见指标
4. 事件 → 爽感 → 主线增量表
5. 不能转化的桥段
6. 中后期如何继续加码

## 注意事项
- **禁止**把多个主线落点并列当同等主菜
- **禁止**每场都爽，但每场爽的方向都不一样
- **禁止**承诺事业升级，高潮却长期浪费在边角热闹上

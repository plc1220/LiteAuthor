先给每场戏写一句“这场戏改变了什么”，再决定保留、压缩、合并或删除。**小说不是把生活原样搬进来，而是只留下与写作目的有关的那部分。看似闲笔可以有，但必须暗地里推进因果。**

## 适用情况
- 章节松散，写了很多漂亮但没用的戏
- 对话很热闹，删掉后剧情却没变化
- 场景像流水账，不知道哪些该留
- 想让故事看起来自然、不那么“机械推进”
- 想写貌似跑题的轻松段落，但怕真跑题

## 核心原则
1. 每个场景至少完成 1—2 项：**推情节 / 立人物 / 改关系 / 埋回收**
2. 如果一个场景对情节发展没有具体作用，再精彩也要处理
3. **伪随意**不等于真跑题：表面放松读者，暗里仍要布因果
4. 场景结束后必须有变化：信息变了、关系变了、局势变了，至少占一项

## 步骤
### 1. 先写场景功能句
给每场戏写一句：
`这场戏结束后，____发生了具体变化。`

写不出来，就先别写正文。

### 2. 用四个问题筛一遍
- 这场戏推动了哪条主线或支线？
- 暴露了人物哪一面？
- 让哪段关系前进或后退了？
- 埋了哪个后面必须回收的点？

四问全弱，基本就是废戏。

### 3. 给每场戏做处理决定
- **保留**：直接改局势
- **压缩**：信息有用，但篇幅过长
- **合并**：功能能塞进别的场景
- **删除**：删掉后什么都不变

### 4. 想写“看似跑题”，先给它藏一把刀
貌似偏题的段落，至少暗藏一项：
- 伏笔
- 线索
- 信息差
- 关系微调
- 情绪缓冲后的反转准备

**如果只是气氛、好玩、文笔漂亮，但后文没有回收，就不是真正的伪随意。**

### 5. 把直给推进改成“自然推进”
不要每场戏都像作者在发任务。
可用做法：
- 让角色在日常里带出关键信息
- 用闲谈、路过、旁见、误会带出伏笔
- 先给读者一个放松点，再在后文回收

但要保证：
- 信息不是白给
- 偶然只负责开门，不能负责收尾
- 回收时能看出前面不是废笔

### 6. 检查场景的进出点
常见问题是开得太早、收得太晚。
优先改成：
- **晚一点进场**：从真正有变化的一刻开始
- **早一点离场**：效果达到就收，不在场尾徘徊解释

### 7. 做一次“删掉测试”
把场景整段拿掉，再问：
- 主线会不会受影响
- 人设会不会少一层
- 后文某个回收会不会失效

如果三个答案都是否，这段多半可以删。

## 输出格式
1. 场景列表
2. 每场戏的功能句
3. 保留 / 压缩 / 合并 / 删除 判断
4. 每场戏暗藏的回收点
5. 哪些是漂亮废戏
6. 哪些可以改成伪随意推进
7. 删改后的章节顺序建议

## 快速示例
### 输入
- 主角和朋友去夜市闲逛
- 作者想写轻松日常

### 输出
- 功能句：这场戏要让主角第一次注意到反派铺子的异常，并让朋友暴露“爱逞强”的缺点
- 处理：保留，但压缩纯闲聊
- 伪随意点：看似逛街放松，实则埋下异常气味、路人传闻和朋友抢着出头的习惯
- 场尾：在主角瞥见那块旧招牌时收住，不必继续把整条街都写完

## 注意事项
- **不要**因为场景写得顺手，就默认它有用
- **不要**把“像生活”误当成“像小说”
- **不要**让轻松段落彻底脱离主线
- **不要**每次都用旁白解释这段为什么重要，尽量靠后文回收证明
- 场景取舍的目标不是删到只剩骨头，而是**让留下来的每一场都值钱**

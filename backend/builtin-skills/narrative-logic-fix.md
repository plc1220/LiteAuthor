先把信息拆开，再决定这段到底要让读者看什么。**叙述逻辑混乱常见有两种根因：少写了“因为所以”“虽然但是”里的半句话，或者一段里同时想说太多，没有侧重点。**

## 适用情况
- 语序乱，前因后果拉不清
- 脑子里信息很多，写出来像一团线
- 每句话单看都懂，连起来却别扭
- 一写场景就忍不住把相关枝节全塞进去

## 输入
- 要梳理的句子 / 段落 / 场景
- 本段想达到的效果
- 现有信息点清单
- 当前最卡的地方

## 步骤
### 1. 先把信息点全列出来
不要直接改正文，先列：
- 事实
- 原因
- 情绪
- 动作
- 结果
- 背景补充

### 2. 只选一个主侧重点
先回答：
`这段最想让读者看到的是什么？`

常见答案：
- 人物很惨
- 人物很急
- 某件旧物很重要
- 两人关系变了
- 某个误会加深了

只选 1 个主重点，最多再留 1 个副重点。其余信息后置、压缩或删掉。

### 3. 用逻辑模板补全缺失半句
把信息点套进这些模板：
- 因为 A，所以 B
- 虽然 A，但是 B
- 先 A，再 B，结果 C
- 如果 A，那么 B

任何套不进去的地方，都要追问：
- 我是不是少写了原因
- 少写了结果
- 少写了转折
- 少写了谁在做什么

### 4. 按“起因 → 当前动作 → 结果”排顺序
稳妥顺序通常是：
1. 必要背景 / 起因
2. 当下最该看到的动作或反应
3. 结果 / 余波
4. 次要补充信息

不要一上来把所有联想、设定、回忆一起倒给读者。

### 5. 只保留服务侧重点的细节
逐条问：
- 这个细节有没有放大当前重点
- 删掉它，会不会更清楚
- 它应该现在写，还是后面再写

**信息很多不等于逻辑强。** 逻辑强，往往意味着只留最该出现的那几条。

### 6. 发现两条线硬融不进一段，就拆开写
如果两组信息各自都重要，但放一起打架：
- 拆成前后两段
- 或改成“主重点详写，副重点一句带过”
- 或后置到下一场 / 下一段

不是所有脑内联想都必须同一口气写完。

### 7. 最后做三级检查
- 句内：主语、因果、转折清不清
- 段内：每句是不是都在服务同一重点
- 场景内：这段放在这里，是不是推动了本场目标

## 输出格式
1. 信息点清单
2. 本段主侧重点
3. 信息点之间的逻辑链
4. 建议顺序
5. 应删 / 应后置的信息
6. 一版更顺的改写

## 快速示例
### 输入
- 人物放下一把旧伞，准备去见喜欢的人
- 作者脑中同时想到：伞的家族来历、下雨路上多狼狈、人物有多心急、两人过去纠葛

### 输出
- 若主重点是“心急”：
  - 保留：进门就乱放伞、鞋没换好、慌张动作
  - 压缩：旧伞来历
- 若主重点是“旧伞的重要性”：
  - 保留：伞的来历、它和两人关系的关联
  - 压缩：一路上的琐碎动作
- 若主重点是“人物狼狈”：
  - 保留：浑身湿透、发抖、打喷嚏
  - 压缩：告白前心理戏

## 注意事项
- **不要**把“我脑子里都知道”误当成“读者也该同时知道”
- **不要**少写逻辑桥，让读者自己脑补前因后果
- **不要**一段里并列塞进太多等重信息
- **不要**把所有枝节都当成主菜
- 叙述逻辑顺不顺，关键不在写得多，而在：**重点清楚，连接完整。**

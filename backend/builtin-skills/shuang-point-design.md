围绕“主角的个人价值如何被看见、被承认”设计爽点。**主角很强却写不爽，通常不是主角不强，而是事情太容易、对比不够、观众不够，或者中间少了让读者烦起来再松口气的波折。**

## 适用情况
- 打脸有结果没体感
- 主角很苏很强，写出来像空夸
- 反派一碰就碎，读者没来得及烦就结束了
- 主角总是轻松赢，但场面很平
- 不知道爽点该怎么递进

## 输入
- 这波最想给的爽感：打脸 / 逆袭 / 苏强 / 团宠 / 掉马 / 公开高光
- 主角这波最值钱的个人价值
- 否定者、见证者、承认者分别是谁
- 当前压制与最终回收
- 这波之后要改写的局势

## 核心原则
1. **爽核 = 主角的价值被看见并改局。**
2. **难题够难，主角够轻，观众够到位，爽感才会立起来。**
3. 读者要先对压制、误判、反派产生情绪，后面的回收才解气。
4. **一波三折是最简单也最好用的放大器，但每一折都必须真加码。**
5. 打赢之后必须有后果：改口、站队、升资源、抬地位、开下一轮更大的局。

## 步骤
### 1. 先写这波爽核
用一句话填空：
`这波真正爽在：主角的____，被____看见并承认。`

### 2. 写清主角到底强在哪
不要只写“很强”。优先落到具体价值：
- 业务能力
- 战力 / 解决问题能力
- 地位 / 资格 / 资源
- 判断力 / 临场反应
- 品格 / 扛事能力
- 别人做不到的独门本事

### 3. 先把事情变难，再把主角写轻
如果主角一出手就稳赢，场面很容易平。优先给难度：
- 纸面上主角比对方弱
- 规则或身份对主角不利
- 别人长期做不到的事，主角要当场做成
- 同类竞争者很多，主角要硬生生脱颖而出

然后再写“轻”：
- 别人折腾很久，主角几分钟解决
- 别人满头大汗，主角依然从容
- 别人还在怀疑，主角已经把结果做出来了

### 4. 一定安排见证者
至少让一种人看见：
- 反派 / 否定者
- 围观群众
- 懂行的强者
- 主角在乎的人

可用反应：
- 惊叹
- 改口
- 破防
- 嫉妒
- 真香
- 当场站队

**没有人看见的高光，爽感会打折。**

### 5. 不会写爽时，先用“一波三折”
最稳的基础公式：
`第一次挑衅 / 误判 → 主角先挡住 → 对手继续加码 → 主角彻底打回去`

好处：
- 先把读者对反派的烦躁感抬起来
- 让主角的强不只是“赢”，而是“越压越稳”
- 打完以后，读者会有明显松口气的释放感

### 6. 设计“一波三折”时，保证每一折都有变化
#### 第一折：先让人不爽
- 反派挑衅
- 主角被误判
- 主角当前处在不利位置

#### 第二折：让情况更烦一点
- 反派升级手段
- 围观者误会加深
- 主角短暂被挡、被拖、被质疑

#### 第三折：狠狠干回去
- 主角拿出真正价值
- 围观者集体改口或破防
- 反派丢脸、失势或失去资格

**如果第二折只是原样重复，读者会烦作者，不会烦反派。**

### 7. 让读者完成移情
爽感离不开站位。至少补一项：
- 主角在这件事上本来就该有的东西被压住了
- 主角本来就值得这份认可，却一直被否定
- 反派的阻拦确实碍事，妨碍了读者的期待兑现

读者一旦移情，哪怕文字很平，场面也会更爽。

### 8. 把“很强”改成可见场面
至少写清 5 件事：
1. 主角做了什么
2. 事情为什么难
3. 对手 / 困境为什么挡不住
4. 谁当场改口、闭嘴或破防
5. 这次之后主角多了什么资格 / 地位 / 选择权

### 9. 爽点必须改局
回收至少 2 项：
- 主角拿到资源 / 机会
- 反派丢脸 / 失势 / 受罚
- 围观者改站队
- 主角被更高层看见
- 下一轮更大的爽点被顶出来

## 输出格式
1. 这波爽核
2. 主角的具体价值
3. 难度设计
4. 见证者与反应
5. 一波三折的三回合表
6. 读者移情点
7. 最终回收和改局结果

## 快速示例
### 输入
- 主角是业务能力很强的明星
- 想写公开高光

### 输出
- 爽核：主角的现场救场能力，被观众和业内同时看见并承认
- 难度：舞台出事故，别人都救不了场
- 一波三折：
  1. 对手等着看笑话，观众不看好
  2. 故障扩大，主角被继续质疑
  3. 主角临时上台稳住全场，对手当场破防
- 改局：主角不只洗白，还拿到更高资源和认可

## 注意事项
- **不要**只写“主角赢了”，不写事情为什么难
- **不要**一波三折变成重复折腾
- **不要**只有主角高光，没有围观和反应
- **不要**反派弱得像纸，读者连烦都来不及烦
- **不要**这波赢完后，局势什么都没变
- 爽点真正值钱的地方，不是“主角赢了”，而是**读者先憋住，再看主角把场子狠狠干回来。**

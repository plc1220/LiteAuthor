先把已写章节压成章纲，再从章纲里反推主线、定位和逻辑断点。**不会先做大纲，不等于只能硬着头皮往前冲；更稳的办法，是先写一批草稿，再用章纲把结构扒出来。**

## 适用情况
- 不会先列大纲，只会先写几章找感觉
- 已经写了 3—30 章，却越写越乱
- 觉得问题很多，但说不清到底卡在哪
- 试写后发现题材、基调、主卖点变了
- 想重写前 3—10 章、文名或文案

## 输入
- 已写正文或章节摘要
- 当前想到的核心梗
- 预期结局或阶段目标（可选）
- 当前最困惑的问题

## 核心原则
1. **先总结，再判断。** 不要一边慌着写、一边凭感觉骂自己。
2. 章纲写的是**章节功能**，不是剧情复述。
3. 每 5—10 章要看一次起伏和段落功能。
4. 如果定位变了，**前 3—10 章要敢于重写。**
5. 逻辑问题不是“感觉怪”，而是 A 到 B 中间缺了步骤、动机、触发或过渡。

## 步骤

### 1. 逐章压成一句章纲
每章用一句话写清：
`谁因为什么，做了什么，局势变成了什么。`

再补 3 项：
- 本章主要功能：立人设 / 推剧情 / 推感情 / 埋伏笔 / 回收
- 本章主冲突
- 章尾钩子或下一轮问题

### 2. 给每章标起伏
可用任一套标法：
- **压 / 扬 / 平**
- **起 / 承 / 转 / 合**
- **铺垫 / 爆发 / 回收 / 过渡**

目的是看清：
- 哪几章在真正推进
- 哪几章只是在重复
- 情绪是不是连压太久、连平太久
- 转折是不是来得太突然

### 3. 每 5—10 章归并成一个大段
对每一段写清：
- 这一段主要在解决什么
- 推进了哪条主线
- 哪个冲突被建立或被回收
- 下一段会被顶出来什么

如果写不出来，说明这几章很可能只是流水账。

### 4. 反查“这本书到底在卖什么”
拿现有章纲回看：
- 更像甜文还是爽文
- 更像剧情流还是感情流
- 更偏沙雕还是正经
- 背景、核心梗、流行元素是不是在服务同一件事
- 当前文名、文案和正文是不是一回事

如果最带感的东西和原定位不一致，就要重新定主卖点。

### 5. 找逻辑断点
按顺序检查章节之间：
- 从这件事发展到下一件事，合理吗
- 某个情节和某个情节之间，是不是缺了过渡
- 人物为什么会突然这么选
- 冲突为什么在这里爆发
- 关系为什么会在这里变化

把问题具体写成：
- 缺动机
- 缺铺垫
- 缺中间事件
- 缺信息交代
- 缺情绪回收

### 6. 找真正该保留的前 3—10 章
试写的意义，是看哪条线最能撑住开头。
从现有章节里筛：
- 哪几章最能体现主卖点
- 哪个开场最能立主角、立冲突、立核心梗
- 哪些内容只是试手感，正式开文时应该删掉或后置

**试写稿不等于正式开文稿。**

### 7. 如果已有结局，再拿结局反查
补一句：
`这批已写章节，究竟有没有在往结局或阶段目标走。`

检查：
- 现有事件是否在为终局铺条件
- 哪些桥段很好看，但和终局无关
- 哪些章节虽然热闹，却没有把主线往前推

### 8. 给出修法，而不是只给结论
每个问题都落到动作：
- 删
- 并
- 前移
- 后置
- 重写
- 改定位
- 改文名文案
- 重写前 3—10 章

## 常用模板
### 模板一：先写几章再定方向
`试写 3—10 章 → 逐章写章纲 → 判断主卖点 → 删除跑偏内容 → 重写前 3—10 章 → 再定文名文案`

### 模板二：已写很多但越写越乱
`逐章压缩 → 标起伏 → 每 5—10 章归段 → 找逻辑断点 → 决定删并改 → 再继续往后写`

## 输出格式
1. 逐章章纲表
2. 每章功能与起伏
3. 5—10 章分段总结
4. 当前真正的主卖点
5. 文名文案是否跑偏
6. 逻辑断点清单
7. 需要删 / 并 / 前移 / 后置的内容
8. 建议保留并重写的前 3—10 章
9. 下一版大纲骨架

## 快速示例
### 输入
- 已写 8 章，本来想写甜文，结果最有戏的是打脸和退婚

### 输出
- 章纲显示：前 2 章在立穿越和处境，第 3—5 章爽点明显，第 6—8 章开始拖
- 主卖点判断：目前真正抓人的是退婚打脸，不是慢热日常
- 修法：删掉过长日常，把打脸和退婚前置；如果决定转爽文，文名文案也一起改

## 注意事项
- **不要**把章纲写成剧情摘要抄写
- **不要**只说“哪里不对”，不指出到底缺了什么
- **不要**试写完发现方向变了，还死保旧开头
- **不要**一口气大改所有问题，先抓最伤主线的 1—2 项
- 先写后扒大纲的核心，不是证明自己没计划，而是：**把已经写出来的东西，变成能继续往下写的结构。**

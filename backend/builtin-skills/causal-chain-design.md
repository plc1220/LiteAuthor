把关键事件从“发生了”写成“为什么会发生、接下来会怎样”。**读者天然会追问“为什么”和“然后呢”，因果链越清楚，故事越容易让人信服和上头。**

## 适用情况
- 情节像天上掉下来，显得巧合
- 角色突然做决定，读者跟不上
- 反转有结果，没前因
- 事故、误会、冲突写完像单个点，带不出后续

## 输入
- 目标事件
- 已知前情
- 相关角色的动机、弱点、秘密
- 想要的效果：宿命感 / 报应感 / 悬念 / 误会 / 悲剧 等
- 允许的巧合比例

## 步骤
### 1. 先只写结果
用一句话写：
`发生了什么`

例如：`主角出了车祸。`

### 2. 立刻往前追问“为什么”
至少追 2—3 层：
- 直接原因是什么
- 再上一层原因是什么
- 这个原因又从哪来

把链条写成：
`结果 ← 直接原因 ← 更早原因 ← 更深动机 / 旧账`

### 3. 每一层原因都标类型
给每个节点标注：
- 人物选择
- 关系影响
- 世界规则
- 环境条件
- 旧账 / 伏笔
- 偶然因素

**优先保证至少有一层来自人物自身。**
只有外力、没有人心，因果链会发空。

### 4. 让每一层都能落到戏里
好的原因，不只是作者脑内说明，而是能变成：
- 一个场景
- 一句关键信息
- 一个伏笔
- 一个先前做过的选择

如果某个原因只能靠事后旁白补课，多半太虚。

### 5. 再追问“然后呢”
事件发生后，继续写：
- 立刻后果
- 次级后果
- 更长期后果

不要把因果只做半截。
`为什么会发生` 负责可信，`发生后会怎样` 负责故事继续走。

### 6. 让因果顺便暴露人物
优先把原因写成角色的欲望、恐惧、习惯或缺点：
- 她接电话，不只是因为电话响了，而是因为那通电话来自她最放不下的人
- 他迟到，不只是堵车，而是因为他先去做了另一个更在意的选择

**最有用的因果，不只解释事情，还解释人。**

### 7. 巧合只负责开门，别负责收尾
可以用巧合让麻烦发生，但不要用巧合替主角解决麻烦。
开局偶遇、路上出事可以偶然；翻盘、脱困、真相落地最好靠前面埋下的因果。

### 8. 控制链条长度
通常 3—5 个节点最稳。
太短会显薄，太长会像写案情说明书。
只保留能抬高情绪、人物或后续剧情的那几层。

### 9. 最后做“剪断测试”
从链条里删掉任意一环，再看事件是否还能成立。
如果删掉也没影响，这一环就是废原因。

## 输出格式
1. 目标事件
2. 向前因果链
3. 每一层原因类型
4. 向后后果链
5. 最能暴露人物的一环
6. 需要前置埋下的伏笔
7. 巧合过重风险点

## 快速示例
### 输入
- 事件：主角出了车祸

### 输出
- 结果：主角出车祸
- 直接原因：她边开车边接电话
- 更早原因：来电人是突然失联多年的父亲
- 更深原因：她一直放不下当年的抛弃，所以明知危险也要接
- 立刻后果：车祸入院，错过原定见面
- 次级后果：对方以为她故意爽约
- 长期后果：旧家庭线和感情线一起被牵出来

## 注意事项
- **禁止**只有“事情发生了”，没有“为什么”
- **禁止**每个大事都只靠巧合推动
- **禁止**把因果全塞到事后解释里
- **禁止**把每一层都解释到死，悬念该留就留
- 因果链的目标不是复杂，而是**让读者觉得这件事非这么发生不可**

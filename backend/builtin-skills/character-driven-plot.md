先把主角“是什么样的人、为什么会变成这样、遇事会怎么选”想明白，再用事件去逼他选择。**剧情不是给人设配插图，而是性格在关系和压力里不断做选择的结果。**

## 适用情况
- 知道人设，不知道故事怎么走
- 情节像作者硬推，换谁当主角都差不多
- 主角性格写在设定表里，正文里却不发作
- 想扩充情节，但越扩越像流水账
- 写到关键节点时，人物反应总像作者代笔

## 核心原则
1. **先问主角性格，再问事件。** 不是反过来。
2. 要写清性格从哪来：经历、身边人、执念、缺口、底线。
3. 每个关键情节都要回答：**按这个人的性格，他会怎么处理？如何收场？**
4. 配角不是摆设，他们负责触发、放大、扭转主角性格。
5. 同一个人设要在“初见 / 关系 / 压力”三种场面里都能成立。
6. 故事一旦脱离人物选择，只剩作者安排，人物就会变成工具人。

## 步骤
### 1. 钉死主角的核心性格
先写四句：
- 他最突出的性格是____
- 这种性格受____影响形成
- 他最在乎____
- 他的底线是____

### 2. 决定怎么揭示这个性格
选一种为主：
- **开门见山**：一出场就让性格发作
- **由浅入深**：先露表层，后露根子

再决定：
- 需不需要配角来见证或刺激
- 是先用小事显性格，还是先用冲突显性格

### 3. 列关系触发器
把主角身边重要人物列成表：
- 谁会让他放松
- 谁会激怒他
- 谁会逼他暴露软肋
- 谁会改变他原本的处理方式

**关系一变，情节就会变。**

### 4. 用“事件五问”推情节
每来一个新事件，都按顺序回答：
1. 依照他的性格，他第一反应是什么
2. 他会主动选哪种处理方式
3. 这种处理会得罪谁、吸引谁、连累谁
4. 他会如何收场
5. 这件事之后，他的性格或关系有没有被进一步揭示、加强或改变

答不出来，说明事件还没长在人设上。

### 5. 用配角放大主角
同一个主角，在不同人面前要出现不同侧面：
- 对强者：是较劲、欣赏还是回避
- 对弱者：是保护、冷眼还是利用
- 对亲近者：是嘴硬、依赖还是逃开
- 对敌人：是逞狠、算计还是先稳后动

### 6. 让性格在三种场面里兑现
至少安排三次：
- **初见场**：让读者一眼认出这人是什么路数
- **压力场**：出事时他还会不会这么选
- **结果场**：他怎么善后，最见底色

### 7. 检查故事是不是“换主角也能写”
逐条问：
- 删掉这个性格，关键情节还成不成立
- 把主角换成另一个人，处理方式会不会完全不同
- 主要转折是不是由主角的选择引发
- 主角和周围人物发生联系后，性格有没有产生真实变化

如果答案大多是否，说明剧情还不是人设推动的。

## 输出格式
1. 主角核心性格
2. 性格形成原因
3. 关系触发器
4. 三个关键事件的“事件五问”
5. 性格在初见 / 压力 / 结果场的兑现
6. 关系或性格变化点
7. 当前最像作者硬推的情节

## 快速示例
### 输入
- 主角设定：平时温和克制，关键时刻很果断

### 输出
- 性格形成原因：早年长期替家里收拾烂摊子，所以习惯先稳局面
- 关系触发器：朋友惹祸时他会先护住人；恋人受辱时他会直接翻桌；面对强敌时反而更冷静
- 关键事件 1：朋友被追责，他先替对方扛下表面麻烦，再私下切断后患
- 关键事件 2：恋人被误会，他不吵闹，直接拿证据反打
- 兑现：前者见担当，后者见决断，性格和剧情一起成立

## 注意事项
- **不要**只给性格标签，不给事件反应
- **不要**让主角只在旁白里有性格，到了关键处却按作者需要行事
- **不要**把所有变化都交给设定说明，优先交给选择和后果
- **不要**让配角只负责捧人，不负责制造关系压力
- 真正的人设推动剧情，不是人物资料很多，而是：**这个人一出手，故事就只能这么拐。**

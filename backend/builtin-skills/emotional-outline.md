先写情绪，再写事件。**大纲不是先把世界观、伏笔、悬念排整齐，而是先确定读者最想被满足的那口情绪。** 如果一做大纲就开始修逻辑、堆设定、搞精妙反转，最后很容易越写越偏离文案承诺。

## 适用情况
- 瞎写时带感，一做大纲就变冷
- 文案很香，正文却开始跑去写悬疑、设定、世界观
- 知道题材热，但抓不住“爽、萌、甜、求而不得”到底在哪
- 大纲越完整，卖点越不明显

## 输入
- 题材与一句话文案
- 主角或核心关系设定
- 想给读者的核心感受：爽、甜、虐后补偿、白月光感、追妻火葬场等
- 当前想到的主要事件（可选）
- 当前最担心的跑偏方向（可选）

## 步骤
### 1. 先钉死一句“这篇文到底爽或萌在哪”
用最直白的话写：
`这篇文本质上是在看____。`

不要写题材词，要写体验。

例如：
- 不停让渣渣懊悔痛苦，主角越过越好
- 让得不到主角的人长期求而不得
- 让被低估的人拿回评价权

### 2. 先代入“最不满足”的状态
按题材选入口：
- **爽文**：谁怎么欺负主角、夺走什么、误会什么、压住什么
- **萌文 / 甜文 / 白月光**：谁想要却得不到、谁看不见主角的好、谁迟迟够不到
- **追妻 / 火葬场**：谁先错过、谁后面会痛、主角凭什么不回头

**先把“不爽、不满足、缺口”写出来，不要立刻想逻辑。**

### 3. 写“情绪账单”
分三栏列出：
1. 现在最憋屈的点
2. 读者最想看到的回收
3. 比“回到原样”更高一级的补偿

示例：
- 被偷东西还被反咬 → 当众抓现场 → 对方身败名裂，主角拿回更多更好的东西

### 4. 先排情绪波形，再排事件
先用情绪句写每一段，不用中性事件摘要。

草稿示例：
- 主角现在太惨了，先让读者彻底站他
- 对面更过分了，这里必须再加压
- 不能再忍，开始第一次回弹
- 这还不够，要让他们公开后悔
- 主角翻身后还要更耀眼，让旧人彻底够不着

**草稿阶段允许口语化、情绪化，先保证“味道对”。**

### 5. 再把事件挂上去
每个事件都回答：
- 这场戏服务哪一种情绪
- 它是在加压、回弹、兑现，还是抬高下一层期待
- 删掉它，核心情绪会不会更集中

如果一场戏只服务：
- 世界观说明
- 悬疑炫技
- 逻辑整齐
- 作者自以为聪明的巧劲

却不服务核心情绪，优先删、并、后置。

### 6. 先保“文案兑现”，再补“逻辑好看”
按顺序检查：
1. 文案承诺的情绪有没有出现
2. 主角的委屈或缺口是否够具体
3. 回收是否够解气、够心动、够补偿
4. 最后再补起因、因果、伏笔、设定说明

**逻辑是修边，不是主发动机。**

### 7. 做“跑偏排查”
出现以下信号就重排大纲：
- 明明卖虐渣，主线却跑成悬疑破案
- 明明卖白月光，主角却忙着和过去彻底切割
- 明明卖甜宠，前三章全在讲设定
- 明明卖逆袭，长时间只写受苦，不给任何小回收

### 8. 产出两版大纲
- **情绪大纲**：只写情绪波形和必写回收
- **事件大纲**：在情绪大纲下面补具体事件、因果和伏笔

先定第一版，再做第二版。

## 输出格式
1. 核心情绪承诺
2. 主角或读者当前最不满足的点
3. 情绪账单
4. 情绪波形
5. 必写回收场景
6. 应删或后置的跑偏内容
7. 事件大纲骨架
8. 逻辑补缝清单

## 快速示例
### 输入
- 文案：白月光题材

### 输出
- 核心情绪承诺：让错过主角的人一直求而不得
- 不满足：他们曾轻视、忽略、错判主角
- 回收：他们后悔、讨好、追不上；主角越来越耀眼
- 跑偏内容：大段悬疑主线、与白月光感无关的复杂阴谋
- 情绪波形：旧伤揭开 → 再次错判 → 主角发光 → 他们开始后悔 → 主角高不可攀
- 事件骨架：每个关键事件都服务“后悔却得不到”

## 注意事项
- **不要**一上来就找 bug、修设定、补世界观
- **不要**让事件比情绪更像主线
- **不要**只写主角很爽，要写读者为什么能爽
- **不是逻辑不重要，而是先把卖点抓住，再让逻辑为卖点服务。**

把一段平直剧情拆成‘目标 → 小钩子 → 2—3 个连贯波折 → 最大剧情点 → 余波’。**波折要一环扣一环，不是随手往路上丢麻烦。**

## 适用情况
- 相遇戏、日常戏、推进戏太平
- 只会平铺直叙：人物去了哪、见了谁、说了什么
- 明知道这一段要推进感情或主线，却写成流水账
- 两条事件并行后情节碎掉
- 章节里只有对话和心理，没有起伏

## 输入
- 这一段最原始的平铺版本
- 主角当前目标
- 时间 / 地点 / 在场人物 / 每个人当时在做什么
- 这一段想落到的最大剧情点
- 想顺手埋的下个事件

## 步骤
### 1. 先写‘最平’的版本
用一句话写清这段如果不加工，会怎么发生。

例如：
- 女主去闺蜜家，正好遇见男主
- 主角去酒楼吃饭，顺手教训恶霸
- 主角参加晋级赛，一路打赢对手

先看清最平的骨架，才知道该在哪加起伏。

### 2. 定这段的最大剧情点
先写一句：
`这段真正想落到的结果是____。`

它可以是：
- 男女主第一次有分量的交集
- 主角当众立威
- 一个旧矛盾被重新点亮
- 一个更大的事件开始启动

**所有波折都要往这个点推。**

### 3. 用四个点搭场景骨架
先列四项：
- 时间
- 地点
- 人物
- 当时每个人正在做什么

这四项先定好，波折才不会飘。

### 4. 先放一个小钩子
在最大剧情点之前，先给读者一个想继续看的小异常：
- 路上瞥见熟悉人影
- 听见一句奇怪的话
- 注意到一个不该出现在这里的物件
- 某配角反应异常

小钩子的作用不是解决问题，而是把读者注意力往后推。

### 5. 设计 2—3 个连贯波折
按下面顺序优先设计：

#### 第一折：打断原计划
让事情不能按最平的方式发生。
- 路上遇见异常
- 临时多出一个人
- 原本轻松的事出现阻碍

#### 第二折：放大冲突或旧矛盾
把事情从‘小麻烦’推到‘不得不处理’。
- 配角诉苦
- 旧敌挑事
- 误会升级
- 利益冲突露头

#### 第三折：把局势推到最大剧情点
让前两折的积累一起炸开。
- 主角出手
- 男主 / 反派看见关键一幕
- 晋级赛撞上真正强敌
- 一个看似小事带出更大的局

### 6. 每一折至少改变一个变量
为避免假波折，每一折至少改变 1 项：
- 对手变了
- stakes 变大了
- 被谁看见变了
- 主角的处理方式变了
- 情绪方向变了
- 下一步后果变了

如果三折只是‘又来一次麻烦’，说明还是平。

### 7. 顺手埋 1—3 个坑或包袱
可埋的东西：
- 一个被刻意点亮的配角
- 一个反复出现的小物件
- 一句当下说不明白的怪话
- 一个暂时没解释的异常反应

埋坑时只需要让它‘被看到’，不要当场解释透。

### 8. 多事件处理规则
**优先把当前事件处理完整，再在其中埋下下个事件的钩子。**

如果一件事写到一半，突然平切去写另一件同样重要的事，最容易碎。

常用写法：
- 先解决眼前冲突
- 在解决过程中看见另一条线索
- 用余波把读者带去下一件事

### 9. 让波折服务主线或人物
每加一个波折，都回答这句：
`这一下是为了推进主线、放大主角、还是改变关系？`

三者都不是，就删。

### 10. 可选增味器
#### 悬念
不要把一切一次讲明，给读者留推测空间。

#### 误导
先把注意力引到错误答案，再回收真正答案。

#### 章尾留人
可用这些方式收尾：
- 话说一半
- 见人不说谁
- 亲密不碰嘴
- 打架不出手
- 关键答案留到下一章揭

### 11. 反查这段是不是仍然平
出现以下情况就重做：
- 三折彼此无关，只是连加事故
- 只有大段对话和心理，没有实际变化
- 波折结束后局势没变
- 当前事件和下个事件并行平推，导致节奏发散
- 主角只是旁观，没有主动应对

## 输出格式
1. 平铺版本
2. 最大剧情点
3. 场景骨架
4. 小钩子
5. 第一折 / 第二折 / 第三折
6. 埋下的坑或包袱
7. 下一事件线索
8. 余波

## 快速示例
### 输入
- 平铺版本：女主去闺蜜家，正好遇见男主
- 目标：让男女主第一次有分量地相遇

### 输出
- 小钩子：去闺蜜家路上，女主透过车帘看见一个熟悉人影
- 第一折：进门后得知闺蜜心情不好，正在为家中庶妹算计发愁
- 第二折：女主陪闺蜜去花园散心，庶妹再次挑事，现场冲突升级
- 第三折：女主替闺蜜出头，当场反击，这一幕正好被男主看见
- 余波：男主对女主产生新判断；庶妹记恨；熟悉人影身份留到下一章

## 注意事项
- **禁止**三折彼此无关，只是连加事故
- **禁止**纯靠大段对话和心理把情节讲过去
- **禁止**同时平均推进两件大事
- **禁止**波折结束后什么都没变
- **禁止**为了曲折而曲折，结果主线被拧断

同一个故事，先别急着问“谁是主角”，先问“让读者先站在谁那边，效果最好”。**视角决定的不是代词，而是信息差、悬念方式、共情入口和爽点顺序。** 第一人称还是第三人称是后一步；先决定站在谁那边看。

## 适用情况
- 大纲没问题，写出来却差一口气
- 不知道该从主角、对手还是旁观者写起
- 同一故事想要悬疑感 / 反差感 / 心疼感 / 爽感，不知道视角怎么换
- 追妻、误会、复仇、悬疑类尤其卡切入

## 核心原则
1. **最好用的视角，不是信息最全的视角，而是信息差最有味道的视角。**
2. 视角一换，读者先知道什么、先误会什么、先心疼谁，都会变。
3. 开局视角要服务这本书最值钱的体验，不要只服务“讲清楚”。

## 输入
- 一句话故事
- 核心矛盾
- 候选视角人物 2—5 个
- 这本书最想给读者的感觉：悬疑 / 心疼 / 爽 / 反差 / 压迫 / 欢乐
- 当前准备的开篇情节

## 步骤
### 1. 用一句话写故事核心事件
填空：
`谁因为____，必须____，但眼下卡在____。`

### 2. 列出候选视角
常见候选：
- 主角
- 对手 / 反派
- 知情者
- 不知情者
- 受害者
- 旁观者 / 见证者

### 3. 给每个视角填五格
对每个候选人都写：
- 他现在知道什么
- 他不知道什么
- 读者跟着他，会先感到什么
- 哪些信息会被天然藏住
- 哪个桥段最适合从他这里起

### 4. 按想要的效果选视角
#### 要悬疑
优先选“不知道真相但会被卷进去的人”。这样读者能和角色一起查。

#### 要反差
优先选“最容易误判局面的人”。这样同一件事会更有戏剧味。

#### 要心疼
优先选“承受伤害、失去、被辜负的一方”。

#### 要后知后觉 / 火葬场
可选伤人者视角，让读者看他怎么慢慢意识到失去。

#### 要爽
优先选“最能感到压制与翻盘落差的人”。

### 5. 再定“知道到哪一步”
视角不是知道得越多越好。先写清：
- 读者现在必须知道什么
- 最好暂时不知道什么
- 哪个秘密该跟角色一起发现
- 哪个秘密适合让读者先知道，形成戏剧反差

### 6. 选切入场景
不要只按“时间最早发生的事”切。优先选：
- 角色第一次真正碰到核心矛盾
- 角色第一次看见最带感的关系 / 危机
- 一个能立刻暴露信息差的场面

### 7. 至少做两版对照
同一故事，至少试写：
- A 视角版本开头 100—300 字
- B 视角版本开头 100—300 字

再比较：
- 哪版情绪更准
- 哪版信息更有钩子
- 哪版更接近这本书真正卖的东西

## 常见判断
### 悬疑
- 犯人视角：偏“读者知道，主角不知道”的紧张
- 主角视角：偏“读者和主角一起查”的悬疑

### 追妻火葬场
- 女主视角：重委屈、心死、抽离
- 男主视角：重后知后觉、错判、反差

### 升级 / 逆袭
- 选最能感到“被压着 → 翻身”的那个人
- 如果主角太稳，可短暂借旁观者放大第一次高光，但尽快回主线视角

## 输出格式
1. 故事核心事件
2. 候选视角表
3. 每个视角的已知 / 未知 / 情绪效果
4. 推荐视角
5. 推荐切入场景
6. 需要先藏住的信息
7. A / B 版开头对比

## 注意事项
- **不要**把“信息最全”误当成“视角最好”
- **不要**只因为习惯，就永远用同一种视角切入
- **不要**为了炫结构频繁换视角，先把最该跟的人跟稳
- 好视角的标准不是复杂，而是**这个故事从这里讲，味道最对**

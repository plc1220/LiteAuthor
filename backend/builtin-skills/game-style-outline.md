把剧情文大纲拆成“玩家、地图、行动、解锁条件、奖励卡”。**先像做游戏一样设计可玩的路径，再把它翻回小说剧情；不要写到一个节点才临时想下一个节点。**

## 适用情况
- 剧情文有卖点，但大纲一做就散
- 爽点能想到单个场面，想不到连续推进
- 写完一个节点后，不知道下一个节点怎么接
- 主角升级太随意，奖励像天上掉的

## 输入
- 一句话题材
- 主角起始状态
- 主线目标
- 事业线 / 感情线 / 谜团线
- 可用世界场域

## 步骤
### 1. 先把主角当“玩家”
写清：
- 玩家一进场看到的是什么
- 开局为什么不能退出
- 第一时间最该体验到什么卖点

前三章至少给：
- 开局困局
- 第一位关键人物
- 第一个小爽点 / 小解锁

### 2. 先画地图，不要先堆事件
列出主角会活动的主要场域：
- 学校 / 公司 / 宗门 / 副本 / 片场 / 宫廷 / 城市片区

每张地图都写：
- 这里能拿什么
- 这里会遇到谁
- 这里会触发哪类冲突

### 3. 把主线拆成几条“可攻略路线”
至少分清：
- 事业 / 升级路线
- 感情路线
- 核心剧情 / 真相路线
- 可选支线

每条路线都写成可解锁的节点，不要只写结论。

### 4. 把奖励写成“卡”
把每个重要收获写成一张卡或一个成就：
- 资源卡
- 身份卡
- 关键盟友卡
- 情感节点卡
- 地图开启卡
- 情报卡

### 5. 给每张卡设置解锁条件
按这个句式写：
`主角做了什么 → 增加了什么值 / 触发了谁 → 才能拿到什么卡`

条件可以是：
- 练习次数
- 任务完成度
- 公开表现
- 信任度
- 声望值
- 关键选择

**不要让主角走两步就随手拿到 SSR。**

### 6. 让小行动累出大节点
安排链条：
`日常行动 → 小增益 → 偶遇 / 冲突 → 中等奖励 → 更大舞台`

这样后续节点才不是平地起高楼。

### 7. 控制奖励密度和稀有度
把奖励大致分层：
- R：小好感、小信息、小资源
- SR：阶段机会、关键助力、关系明显推进
- SSR：地位跃升、重大翻盘、核心感情确认、主线突破

同一阶段不要连发太多高稀有奖励。

### 8. 把感情线也挂到玩法里
不要让感情线单独飘着。
问自己：
- 哪个任务会逼两人靠近
- 哪张卡会同时推进事业和感情
- 哪个地图天然适合制造 CP 感

### 9. 反查大纲是否能持续“可玩”
逐条检查：
- 删掉临时灵感，这套地图和节点还能不能跑
- 每个阶段是否都有可解锁目标
- 主角下一步始终知道自己能去哪里、争什么
- 奖励是否越到后期越有分量

## 输出格式
1. 开局场景
2. 地图清单
3. 各路线节点
4. 奖励卡列表
5. 每张卡的解锁条件
6. R / SR / SSR 分布
7. 哪些节点同时推进感情和主线
8. 后续可持续性风险

## 快速示例
### 输入
- 题材：娱乐圈逆袭

### 输出
- 地图：练习室、综艺现场、品牌活动、剧组
- R 卡：影帝第一次注意到她
- SR 卡：护肤品代言、综艺导师邀约
- SSR 卡：蓝血资源、奖项提名
- 解锁链：练舞提升舞台表现 → 综艺露面 → 赞助商看中 → 拿到代言

## 注意事项
- **禁止**写成“想到一个高光就补一个节点”
- **禁止**奖励没有条件、没有代价
- **禁止**把地图当摆设，和剧情无关
- 好大纲像好游戏，主角永远知道下一步值得去哪里

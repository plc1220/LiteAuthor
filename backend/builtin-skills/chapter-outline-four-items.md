先把这一章压成四项：**事件、关键词、承前启后、伏笔**。**章纲的目的不是写漂亮摘要，而是让这一章只做这一章该做的事。**

## 适用情况
- 已有卷纲或细纲，但下笔前不知道本章重点
- 日更或高频更新，需要快速定章
- 常把上一章、下一章的内容写混
- 本章情绪和细节容易散
- 伏笔和章尾钩子老忘

## 输入
- 上一章发生了什么
- 这一阶段当前大目标
- 本章必须推进的内容
- 下一章准备接什么
- 可用伏笔（可选）

## 四件套怎么写
### 1. 事件
一句话写清本章最主要的事：`谁因为什么，遭遇了什么 / 完成了什么。`

要求：
- **只能写本章主事件**
- 一章写不完，可以拆成两章
- **不要**把两个章节的主事件揉成一章

### 2. 关键词
写 2—4 个词，限定这一章的细节和情绪焦点。

关键词可以来自：
- 场景：论坛、宴席、训练场、病房
- 情绪：压抑、试探、坚定、羞辱、庆功
- 关系：误会、站队、和解、对峙
- 氛围：公开、私下、混乱、冷场

关键词的作用：
- 限定这一章该写哪些细节
- 限定主角该处在哪种情绪里
- 限定读者这一章主要吃到什么感觉

### 3. 承前启后
各写一句：
- **承前**：上一章什么结果带进了这一章
- **启后**：这一章结束后，会把什么问题、局势或钩子抛给下一章

要求：
- 本章要照顾前后逻辑
- **不要**提前替下一章把戏演完
- **不要**为了补前情，把上一章重写一遍

### 4. 伏笔
写本章是否需要埋伏笔。

如果要埋，必须写清：
- 埋什么
- 读者现在看到的表面信息是什么
- 后面准备怎么回收

如果当前不需要伏笔，可以不埋。**伏笔不是每章都必须有。**

## 写法步骤
### 1. 先定本章边界
先删掉不属于这一章的内容。

检查：
- 这章最主要解决什么
- 这章结束后局势变了什么
- 哪些内容其实该留到下一章

### 2. 用“事件”锁主任务
把本章压成一句能执行的话，不写成空泛概括。

- 差：主角经历了一些波折。
- 好：主角在论坛上发现系统性抵制，情绪受挫后决定硬扛下去。

### 3. 用“关键词”约束镜头和情绪
每写一段都回头看：这段有没有服务关键词。

如果关键词是“公开、羞辱、硬扛”，那这一章就不该突然跑去大段写轻松日常。

### 4. 用“承前启后”检查衔接
确保这一章既不是凭空开始，也不是写完就断。

### 5. 伏笔按需添加
伏笔只埋对后文真的有用的东西。

### 6. 下笔前再问四句
- 本章最值钱的一下是什么
- 本章结束后局势变了什么
- 哪个细节最能体现关键词
- 有没有把下一章内容抢写了

### 7. 临近写作时再写章纲
章纲越细，越适合临近写作时使用。**不要过分提前把单章章纲写死。**

## 输出模板
```markdown
### 第 X 章章纲
- 事件：
- 关键词：
- 承前：
- 启后：
- 伏笔（可选）：
- 本章最值钱的一下：
```

## 示例
### 输入
- 上一章：男主拿到了电影配角机会，消息曝光
- 本章：要写他被原配角粉丝抵制
- 下一章：准备查是谁在背后带节奏

### 输出
- 事件：男主发现网上出现系统性抵制，情绪受挫后决定继续演下去
- 关键词：论坛、质疑、支持、坚定
- 承前：成为配角的消息曝光，引来旧粉圈不满
- 启后：主角察觉背后有人操盘，准备追查针对者
- 伏笔：几个账号发言口径异常一致，暗示有人带节奏
- 本章最值钱的一下：主角从被攻击的动摇，走到重新立住决心

## 注意事项
- **不要**一章里塞太多主事件
- **不要**拿关键词代替真实情节推进
- **不要**让承前启后变成剧情复述
- **不要**为了伏笔硬埋没用的东西
- **不要**把十几章单章章纲一次性写死
- 一个内容没写完，宁可拆章，也别把多个内容混写

先把争议梗拆成“梗本体、毒标签、真萌点”，再重排叙事顺序，强化优点，弱化毒点。**不要先给梗判死刑，先判断读者到底会因为什么弃文，又会因为什么留下。** 很多梗不是不能写，而是作者先把最刺眼的毒点顶到了读者脸上。

## 适用情况
- 想写常见但容易挨骂的梗：替身、插足、追妻、白月光、误会伤害、霸总、强取豪夺等
- 自己一想到这个梗，脑子里先冒出的全是骂点，已经不会写了
- 同样的梗，别人写得动人，自己一写就像喂毒
- 不确定该保留什么、该弱化什么

## 输入
- 一句话梗
- 最想保留的萌点或名场面
- 当前最担心的毒点
- 目标题材与读者
- 可参考的成功作品（可选）

## 步骤
### 1. 先分开三件事
分别写一句：
1. `这个梗真正发生了什么`
2. `读者最容易给它贴什么毒标签`
3. `读者其实可能会被什么吸引`

**梗本体不等于标签，标签不等于写法。**

示例：
- 梗本体：女主在压力下做了会伤害男主的选择
- 毒标签：故意伤害、白眼狼、恋爱脑
- 可用萌点：两难、委屈、误伤后的追悔、补偿

### 2. 同时列优点和毒点
至少各列 3 条。

优点常见来源：
- 心疼感
- 求而不得
- 命运感
- 身份错位
- 误会反转
- 补偿感
- 打脸感

毒点常见来源：
- 道德标签过重
- 主观恶意太强
- 角色降智
- 伤害尺度过界
- 补偿太轻
- 回头成本太低

### 3. 判断毒点到底从哪来
不要只写“这梗有毒”，要把“毒”具体化。

常见来源：
- **标签先行**：读者先看到“小三、替身、渣攻”这类词，先入为主
- **主动恶意太强**：像故意作恶，不像两难、误判或被迫
- **信息顺序错误**：先暴露毒点，后补苦衷
- **伤害尺度过界**：超过目标读者容忍线
- **补偿失衡**：受害者没拿回东西，加害者代价太低
- **作者站位错误**：一味硬洗白，反而放大反感

### 4. 重写站位和叙事顺序
优先采用下面的改写方向：
- 把“故意”改成“被迫 / 误判 / 两害相权 / 信息不全下的错误选择”
- 先写角色失去什么、怕什么、被逼到什么地步，再写他做了什么
- 先给读者看委屈、代价、挣扎，再给标签
- 少重复最刺眼的毒标签，多写处境、行为和后果
- 给角色保留一条底线：会愧疚、会付代价、会补救、不会主动踩到底线外

### 5. 明确“弱化什么，强化什么”
按表输出：
- 要弱化的：标签、主动恶意、降智细节、无谓羞辱
- 要强化的：被逼无奈、代价、心疼点、求而不得、公开见证、补偿回收

### 6. 安排补偿和见证
至少回收两层：
1. **情绪回收**：委屈被看见、误会被澄清、立场被理解
2. **结果回收**：道歉、站队、关系反转、资源回拨、身份翻盘、反派付代价

**只有解释，没有补偿，读者不会买账。**

### 7. 如果有参考文，拆“为什么它能写成萌点”
不要抄结论，要拆方法：
- 它先让读者站谁
- 它先暴露优点还是先暴露毒点
- 它怎样回避最刺眼的标签
- 它让角色付了什么代价
- 它用什么场面把补偿做实

### 8. 给出三种结论
- **可直接写**：毒点可控，萌点足
- **必须改写后再写**：梗能用，但要换站位、顺序或尺度
- **建议放弃**：核心吸引力太弱，或已越过目标读者容忍线

## 补充：立场判断与高风险替代

### 1. 先定读者站位，不做空泛的性转测试
先写清：
- 这段梗默认让读者站谁
- 谁是主角 / 女主角 / 被偏爱者 / 受害者
- 读者此刻最想维护谁、看谁赢

**很多“毒不毒”不是抽象公平问题，而是站位问题。**  
同一事件，主角不同、叙事焦点不同、目标读者不同，毒感会完全不同。

性转测试只在一种情况下有参考价值：
- 性别换了以后，**主角位、被代入位、被心疼位、被评价位** 也保持一致

如果只是把性别一换，却把叙事站位也换了，得到的不是有效判断。

### 2. 先抽离剧情功能，再决定要不要保留这个高风险梗
对任何容易挨骂的设定，先问：
`它到底在剧情里负责什么功能？`

常见功能：
- 证明人物曾深深爱过、留下旧伤
- 制造关系隔阂或信任危机
- 给后续追悔 / 补偿 / 破镜重圆找理由
- 推动男女主慢慢靠近
- 制造牺牲感、亏欠感、命运感
- 让主角暂时陷入道德或情感困局

如果只是“感觉这样更刺激”，而说不出功能，多半可以删。

### 3. 先列 2—3 个低毒替代，再决定要不要硬写
如果高风险梗只是手段，不是卖点本体，就先找替代：
- “为前任留下靠近心脏的伤”  
  → 改成任务、家人、旧战友、陌生人救援留下的伤；  
  → 或保留前任，但弱化“理想化前任”强度，改写成旧事遗留，而不是现成白月光
- “为了利益和别人滚床单”  
  → 改成假订婚、公开站队、利益绑定、暧昧误会、被迫同行等更低毒手段
- “刚和男主在一起，心里其实还在选男配”  
  → 改成旧执念未散、误把依赖当喜欢、尚未真正确认关系
- “必须靠出轨 / 睡过 / 肉体背叛来制造现实感”  
  → 改成价值观冲突、立场选择、资源交换、名声代价、公开失信

**如果 2—3 个替代方案都能完成同一功能，就优先选更稳的。**

### 4. 新人或控场不稳时，别把高风险梗顶在开局
很多争议梗不是绝对不能写，而是：
- 一上来就写，读者还没站稳主角
- 补偿、解释、回收都很后面
- 作者控不住尺度和站位

这时高风险梗就会直接变成劝退点。  
**如果当前能力还不足以“在毒点外面裹糖”，先换设定，不要硬喂。**

## 输出格式
1. 梗本体
2. 读者会先看到的毒标签
3. 这个梗真正可用的萌点
4. 主要毒点与来源
5. 目标读者站位
6. 剧情功能
7. 应弱化的部分
8. 应强化的部分
9. 推荐叙事顺序
10. 必写补偿与见证
11. 低毒替代方案
12. 改写后一句话梗
13. 是否建议保留这个梗

## 快速示例
### 输入
- 梗：女主伤害了男主

### 输出
- 毒标签：白眼狼、故意背刺
- 可用萌点：被逼到绝路后的误伤、后知后觉的心疼、补偿与追悔
- 改写：先写女主被逼到什么地步、失去什么、为什么两害相权只能这样选；伤害发生后立刻写代价、愧疚和补救；后续让男主和围观者看见真相
- 不要写成：女主明知故犯还理直气壮，后续轻飘飘一句“我也很痛苦”就翻篇

### 另一个常见高风险例子
#### 输入
- 梗：男主胸口的伤是为了前任留下的，作者想用它写出“慢慢走出情伤”

#### 输出
- 目标读者站位：如果正文主卖当前 CP，读者默认应站现任关系
- 剧情功能：证明旧伤、制造情感门槛
- 低毒替代：伤改成旧任务或家人事件；前任改成过去关系节点而非理想化白月光；把重点放在男主如何处理旧伤、如何选择现任
- 结论：若当前篇幅和控场能力不足以快速给到现任关系的独特性与补偿，不建议保留原写法

## 注意事项
- **不要**把评论区的毒点总结当成唯一判断
- **不要**为了洗白硬改既有事实
- **不要**一边想写萌点，一边反复强调最刺眼的毒标签
- **不要**拿空泛的“性转公平”代替目标读者判断
- **读者的毒点不等于作者绝对不能写，但作者必须知道自己在卖什么、在避什么。**

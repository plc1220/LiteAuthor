先把一段故事拆成开篇、铺垫、高潮、结尾四段，再决定每段写多少、怎么写。**紧凑不是一直出事，而是每一段都知道自己在干什么。** 章节一旦没有作用，就很容易变成水章和跳订重灾区。

## 适用情况
- 大纲有事情，但读起来不紧凑
- 铺垫写太久，高光一闪而过
- 不知道某章节到底该干什么
- 单元剧情或感情线容易比例失衡

## 输入
- 这一段故事的核心矛盾
- 主角关系或主线目标
- 预计总字数或单元字数
- 目前想到的主要桥段

## 步骤
### 1. 先分四段
- **开篇**：相遇、入局、主问题出现
- **铺垫**：关系升温、伏笔、加压、为高潮蓄势
- **高潮**：最大爆发点、核心冲突正面兑现
- **结尾**：消解冲突、交代余波、落感情或阶段结果

### 2. 给每段写一句作用
按这个句式写：
- 开篇是为了____
- 铺垫是为了____
- 高潮是为了____
- 结尾是为了____

写不出来，就说明这一段还没想清。

### 3. 先定比例，再写细节
短篇可参考：
- 开篇：20% 内
- 铺垫：40% 左右
- 高潮：30% 左右
- 结尾：10% 左右

以 1 万字为例：
- 开篇约 2000
- 铺垫约 4000
- 高潮约 3000
- 结尾约 1000

长篇处理法：
- 把每个单元都当成一个小短篇
- 开篇可被前文承接吞掉
- 但铺垫、高潮、结尾通常不能省

### 4. 给每章标功能
逐章问：
- 这一章属于哪一段
- 这章在推进什么：关系、伏笔、危机还是结果
- 后面有没有必须用到它的地方

**没有功能的章节，优先删、并、挪。**

### 5. 做比例失衡检查
重点查两种常见问题：
- 阻碍写了很久，核心事件只用几页带过
- 日常、路途、重复拉扯太多，真正推进太少

如果前面已经写掉了 80% 篇幅，核心冲突还没爆，说明铺垫过长。
如果高潮刚起来就收，说明前面给太多、后面给太少。

### 6. 区分写法
- 开篇：快、清楚、尽早入局
- 铺垫：详略结合，别把每句对白都写满
- 高潮：必须详写，人物情绪、动作、现场气氛都要打满
- 结尾：回收代价，留余味，可详可略

### 7. 守住“无水章”原则
每章至少完成 1 项：
- 新信息
- 新关系变化
- 新阻碍
- 新选择
- 新结果

如果读者跳过这一章几乎不影响理解，多半就是水章。

## 输出格式
1. 四段结构表
2. 每段预计字数
3. 每章功能表
4. 比例失衡点
5. 该压缩的部分
6. 该扩写的高潮
7. 调整后的布局建议

## 快速示例
### 输入
- 男配上山抢女主，想写 3 万字

### 输出
- 问题：山下阻碍写了 2.5 万字，上山后的告白、冲突、反击只剩 5000 字
- 结论：铺垫过长，高潮被挤压
- 调整：压缩山下阻碍，把核心告白、对打、抉择留足篇幅

## 注意事项
- **不要**让同类桥段长期原地打转
- **不要**把最值钱的高潮写得最短
- **不要**每章都像过渡章
- 布局的目标不是平均分配，而是让笔墨跟着价值走

先把剧情流真正的“那根线”钉住，再让人物、事件、因果和好奇都挂在这根线上。**剧情流不是事件堆积，而是同一主轴在不同事件里的反复碰撞。**

## 适用情况
- 有很多桥段，但总觉得发浮
- 情节都围着“升级、夺宝、赢”打转，读者记不住为什么值得看
- 人物像工具人，换谁来当主角都差不多
- 故事热闹但主题不清，越写越散
- 关键事件删掉以后，前后居然没什么变化

## 输入
- 一句话故事梗
- 主角、反派、1—2 个关键配角
- 题材与世界背景
- 目前想到的主要事件 3—10 个
- 想给读者的核心体验：燃、爽、痛、悬念、宿命等

## 步骤
### 1. 先写“这篇文真正反复在审判什么”
用一句话写成：
`这篇文真正想反复碰撞的是：____。`

优先写价值判断或核心命题，不要只写表层目标。
- 差：主角夺宝、主角升级、主角赢第一
- 好：仁善在乱世里能不能立得住、勇气会不会得到回报、守护值不值得付代价、控制欲最终会毁掉谁

### 2. 分开“外层目标”和“里层主轴”
每篇剧情流至少写两层：
- **外层目标**：主角眼前要得到什么
- **里层主轴**：这件事为什么值得写，做成或做不成分别意味着什么

如果所有情节都只围着“得到本身”展开，故事就容易发浮。

### 3. 给关键角色各写“私人情绪驱动 + 一个错误”
至少给主角、反派、关键配角各写：
- TA 最本能的私人情绪是什么
- 这会驱动 TA 做什么
- TA 最容易在哪件事上判断错误 / 选错 / 错过

句式：
`因为____，TA 一遇到____就会____，也因此总会在____上犯错。`

**错误是剧情流的重要发动机。** 没有错误，人物很难把故事推歪、推深、推出代价。

### 4. 做“换人测试”
逐条检查关键事件：
- 换个性格完全不同的主角，这段还会不会几乎一样
- 如果会，说明事件还没长在人设上
- 继续补：这个人为什么偏偏会做这个决定

### 5. 用同一主轴筛事件
每个大事件都要同时满足 4 件事：
1. 有具体的事发生
2. 由前因逼出来
3. 和主轴发生碰撞
4. 留下新的好奇问题

可直接写成：
`因为____，发生了____，这次碰撞暴露了____，接下来读者会想知道____。`

### 6. 让同一主轴在不同场景里被反复审判
把主轴放进 3—5 类不同事件里：
- 关系事件
- 利益事件
- 危机事件
- 选择事件
- 代价事件

例如主轴是“勇气”：
- 求学时敢不敢冒险
- 求爱时敢不敢表白
- 投资时敢不敢下注
- 危机时敢不敢站出来
- 失败后敢不敢承认错误

这样事件之间既有共性，也不重复。

### 7. 把“好奇”当成剧情流底层情绪
每个阶段都至少安排一种“读者想继续看”的好奇：
- 猜到开头，猜不到过程
- 猜到过程，猜不到后果
- 以为知道真相，后来发现还有另一层
- 以为事件结束了，结果它还在改写后续关系和局势

**好奇不是故弄玄虚，而是让事件有意外的来由、走向或余波。**

### 8. 给主轴安排一次真正的代价
至少设计一处：
- 主角因自己的错误付代价
- 反派因自己的执念付代价
- 角色为了守住主轴中的某个东西，不得不失去别的东西

没有代价，主轴容易停留在口号。

### 9. 最后做删改检查
逐条问：
- 删掉这段，主轴还会被碰撞吗
- 删掉这段，人物关系或因果链会变吗
- 这段是不是只增加“获得”，没增加人物、主题或后果

答“不会 / 不变”的多半该删、并线或后置。

## 输出格式
1. 主轴一句话
2. 外层目标 / 里层主轴
3. 关键角色的私人驱动与错误表
4. 事件筛选表（事件 / 前因 / 主轴碰撞 / 新好奇）
5. 3—5 个主轴审判场景
6. 关键代价点
7. 可删或需改写的发浮情节

## 快速示例
### 输入
- 梗：主角在魔族入侵时隐藏实力出手

### 输出
- 主轴：力量该只用来成全自己，还是也要承担守护
- 外层目标：隐藏身份、解决危机
- 里层主轴：主角到底愿不愿意为比自己更大的东西出手
- 主角驱动：想保住清静，因此总想少管闲事；错误是总以为拖一拖也无妨
- 事件碰撞：魔族入侵、同门误判、强者见证、后续真相揭开
- 好奇：主角明明出手了，谁看见了？这次隐瞒会带来什么后果？

## 注意事项
- **禁止**所有情节都围着“我得到了什么”打转
- **禁止**人物只负责完成作者安排，不带私人情绪
- **禁止**只有事件，没有因果、余波和代价
- **禁止**把主题写成台词口号，不让事件去证明
